에이전트 팀을 사용하면 여러 Claude Code 인스턴스가 함께 작동하도록 조율할 수 있다.
한 세션이  팀 리더 역할을 하여 작업을 조율하고, 작업을 할당하며, 결과를 종합한다.
팀원들은 각각 자신의 컨텍스트 윈도우에서 독립적으로 서로 작동하며 서로 직접 통신한다.
단일 세션 내에서 실행되고 메인 에이전트에게만 보고할 수 있는 subagents와 달리, 리더를 거치지 않고 개별 팀원과 직접 상호작용할 수 도 있다.

연구 및 검토: 여러 팀원이 문제의 다양한 측면을 동시에 조사한 후 서로의 발견 사항을 공유하고 도전한다.
새로운 모듈 또는 기능: 팀원들이 각각 별도의 부분을 소유하며 서로 간섭하지 않는다.
경쟁하는 가설로 디버깅: 팀원들이 다양한 이론을 병렬로 테스트하고 더 빠르게 답에 수렴한다.
교차 계층 조율: 프론트 엔드, 백앤드, 테스트에 걸친 변경사항으로 각각 다른 팀원이 소융한다.

에이전트 팀은 조율 오버헤드를 추가하여 단일 세션보다 훨씬 더 많은 토큰을 사용한다.
팀원들이 독립적으로 작동할 수 있을 때 가장 잘 작동한다.
순차적 작업, 동일 파일 편집, 또는 많은 종속성이 있는 작업의 경우 단일 세션이나 subagent가 도 효과적이다.
