# 클래스패스(Classpath) 회고록

> Ex01_HelloPrint 실행 과정에서 발생한 질문들을 바탕으로 정리한 회고

---

## 1. 클래스패스란?

클래스패스 = JVM이 `.class` 파일을 찾기 위해 사용하는 경로 목록

- JVM이 클래스를 로드할 때, 클래스패스에 지정된 경로들을 순서대로 검색
- `패키지경로/클래스명.class` 형태로 파일을 찾아 메모리에 로드

---

## 2. 패키지가 있을 때 실행

### 문제
`java Ex01_HelloPrint` → `ClassNotFoundException`

### 원인
패키지가 선언되면 전체 클래스명(fully qualified name)으로 실행해야 함.

```
package ct.stage1.practice.ex;
→ 실행: java ct.stage1.practice.ex.Ex01_HelloPrint
```

### 실행 위치
클래스패스 루트 = `ct` 폴더의 상위 디렉터리

```
c:\dev\cs-study\     ← 여기서 실행하거나 -cp로 지정
└── ct\
    └── stage1\practice\ex\Ex01_HelloPrint.class
```

### 명령 예시
```powershell
java -cp c:\dev\cs-study ct.stage1.practice.ex.Ex01_HelloPrint ian
```

---

## 3. 패키지 없이 실행하려면?

### 방법 1: 패키지 제거
- `package` 선언 삭제 → 기본 패키지(default package)
- `ex` 폴더에서 `java Ex01_HelloPrint` 실행 가능

### 방법 2: 재컴파일 필수
패키지를 제거한 후 반드시 재컴파일해야 함.  
기존 `.class`는 패키지 정보가 바이트코드에 남아 있어 `java Ex01_HelloPrint`로 찾을 수 없음.

```powershell
javac Ex01_HelloPrint.java
java Ex01_HelloPrint ian
```

---

## 4. javac vs java 구분

| 명령 | 용도 | 예시 |
|------|------|------|
| javac | 컴파일 (.java → .class) | `javac Ex01_HelloPrint.java` |
| java | 실행 | `java Ex01_HelloPrint ian` |

- `ian`은 `main(String[] args)`의 `args[0]`로 전달되는 실행 인자
- `javac`에는 `.java` 확장자 필요, `java`에는 클래스명만 사용

---

## 5. 같은 디렉터리에 있는데 왜 ClassNotFoundException?

### 정상 동작
클래스패스가 설정되지 않았을 때 → 기본값은 현재 디렉터리(`.`)

### 원인: CLASSPATH 환경 변수
`CLASSPATH`가 설정되어 있으면 기본값(`.`)이 무시됨.

- CLASSPATH 미설정 → 현재 디렉터리에서 검색
- CLASSPATH 설정됨 → 그 경로만 검색, 현재 디렉터리 제외

### 해결
```powershell
java -cp . Ex01_HelloPrint ian
```
또는 CLASSPATH 환경 변수에 `.;`를 맨 앞에 추가

---

## 6. 클래스패스 지정 방법

| 방법 | 적용 범위 |
|------|----------|
| -cp / -classpath | 해당 `java` 명령 한 번에만 적용 |
| CLASSPATH 환경 변수 | 설정 시 모든 `java` 실행에 기본 적용 |
| 기본값 | CLASSPATH 미설정 시 현재 디렉터리(`.`) |

- `-cp`는 매 실행마다 지정해야 함
- 한 번만 설정하고 싶으면 CLASSPATH 환경 변수 사용

---

## 7. 복잡한 프로젝트에서 클래스패스

### 루트 하나만 지정
여러 패키지가 있어도 최상위 루트 디렉터리 하나만 지정하면 됨.

```
c:\my-project\     ← -cp c:\my-project
├── com\foo\Util.class
├── ct\stage1\...\Ex01_HelloPrint.class
└── org\lib\Parser.class
```

### 여러 루트가 있을 때
`;`로 구분하여 나열

```powershell
java -cp "c:\project;c:\libs;c:\external.jar" com.foo.Main
```

---

## 8. 동일한 클래스명이 여러 개 있을 때

### 패키지가 다르면 → 다른 클래스
`com.foo.Util`과 `com.bar.Util`은 별개. 전체 클래스명으로 구분.

### 같은 전체 클래스명이 클래스패스 여러 경로에 있으면
먼저 찾은 것이 사용됨. 클래스패스 순서가 중요.

---

## 9. 클래스패스는 환경 변수인가?

아니요. 클래스패스는 개념이고, 지정 방법은 세 가지:

1. 환경 변수 `CLASSPATH` (선택)
2. 명령줄 `-cp` (실행 시마다)
3. 기본값 `.` (CLASSPATH 미설정 시)

---

## 10. 요약 체크리스트

| 질문 | 답 |
|------|-----|
| 클래스패스란? | JVM이 클래스를 찾는 경로 목록 |
| 패키지 있을 때? | `java 패키지.클래스명` + 루트를 -cp로 지정 |
| 패키지 없을 때? | 재컴파일 후 `java 클래스명` (같은 폴더에서) |
| 같은 폴더인데 안 되면? | CLASSPATH 환경 변수 확인, `-cp .` 사용 |
| -cp 한 번만? | 아님. 매 실행마다 지정. 영구 적용은 CLASSPATH |
| 여러 디렉터리? | 루트 하나만 지정 |
| 같은 클래스명? | 패키지로 구분. 같은 전체명이면 먼저 찾은 것 사용 |

---

## 11. 실무에서는?

Maven, Gradle, IDE가 클래스패스를 대부분 처리함.  
직접 `javac`/`java`를 쓰는 경우는 학습·원리 이해용에 가깝다.
