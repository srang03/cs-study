Q1. `javac` 명령은 무엇을 하는가?
=> .java 파일을 컴파일해서 .class 파일인 자바 바이트 코드를 생성하는 명령어이다.
[정답] ✓

Q2. `java HelloWorld.class`로 실행하면 어떻게 되는가?
=> java에서는 .class 확장자를 붙이지 않는다. 올바른 실행은 `java HelloWorld`이다. 패키지가 있거나 CLASSPATH가 설정된 경우 클래스패스(-cp)를 지정해야 한다.
[1차 오답] JVM에서 바이트 코드를 읽어서 실행한다. — `java`는 클래스명을 인자로 받으며, `.class`를 붙이면 `HelloWorld.class`라는 클래스를 찾으려 해 `ClassNotFoundException` 발생
[정답] ✓ (수정 반영)


Q3. `main` 메서드가 `static`이어야 하는 이유는?
=> main 메서드는 프로그램의 진입점이고, 객체 생성 없이 바로 실행되어야 하기 때문이다.
[정답] ✓

Q4. 자바에서 링크(클래스 해석)는 언제 일어나는가?
=> 원래 C/CPP에서는 컴파일 타임, 링크 타임을 빌드 과정이고, 이후 런타임이 있지만, 자바에서는 컴파일 타임이 빌드 과정이고, 링킹은 런타임에 수행된다.
[정답] ✓

Q5. 파일명이 `Test.java`인데 안에 `public class Hello`를 선언하면?
=> 자바에서는 파일명과 클래스명이 일치해야 컴파일 에러가 발생하지 않는다.
[정답] ✓
