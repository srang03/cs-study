# 암호화 (Encryption)

SSL 및 암호기술 이해를 위한 기초 개념. 데이터 기밀성 확보를 위한 대칭키·비대칭키 암호와 무결성·부인 방지를 위한 디지털 서명을 다룬다. 암호화 키를 사용하는 이유는 기밀성 때문이고, 해시와 디지털 서명의 주목적은 무결성 및 부인 방지이다.

---

## 1. 대칭키 암호 (Symmetric Key Cryptography)

### 1.1 정의 및 동작 방식

대칭키 암호는 암호화와 복호화에 동일한 키를 사용하는 방식이다.

- 송신자와 수신자가 같은 비밀키(Secret Key)를 공유한다.
- 암호화: 평문 + 비밀키 → 암호문
- 복호화: 암호문 + 비밀키 → 평문

### 1.2 특징과 한계

| 구분 | 내용 |
|------|------|
| 장점 | 연산이 단순하고 빠르다. 비대칭키보다 전산 자원을 적게 사용한다. |
| 한계 | 키를 안전하게 교환해야 한다. 인터넷처럼 신뢰할 수 없는 채널에서 키 배포가 어렵다. |

> AES-128과 RSA-2048은 보안 강도가 비슷하지만, 키 길이와 연산량 차이가 크다. RSA는 큰 수 연산을 수행하므로 CPU·배터리 소모가 많다. 모바일 환경에서는 대칭키 사용이 유리하다.

---

## 2. 비대칭키 암호 (Asymmetric Key Cryptography)

### 2.1 정의 및 동작 방식

비대칭키 암호는 한 쌍의 키가 상호작용하는 구조이다. 한 키로 암호화하면 쌍을 이루는 다른 키로만 복호화할 수 있다.

| 키 | 용도 | 공개 여부 |
|----|------|-----------|
| 공개키 (Public Key) | 암호화, 서명 검증 | 외부에 공개 |
| 개인키 (Private Key) | 복호화, 서명 생성 | 본인만 보관 |

PKI(Public Key Infrastructure)의 근간을 이룬다.

### 2.2 대표 알고리즘

| 알고리즘 | 비고 |
|----------|------|
| RSA-2048 | 큰 수(소인수분해) 기반. 전산 자원을 많이 사용한다. |
| ECC (Elliptic Curve Cryptography) | 동일 보안 강도에서 키 길이가 짧다. 모바일·IoT에 유리하다. |

### 2.3 동작 예시

평문 65, 모듈러스 323, 공개키 5, 개인키 29인 경우(5와 29는 한 쌍, 5와 323은 공개, 29는 비공개):

```
65^5 mod 323 = 12   (암호화)
12^29 mod 323 = 65  (복호화)
```

큰 수에 대한 제곱·모듈로 연산을 통해 경우의 수를 늘리는 것이 핵심이다. `12^X mod 323 = 65`에서 X(개인키)를 찾으면 암호 체계를 무력화할 수 있으나, 충분히 큰 수에서는 계산이 현실적으로 불가능하다.

> 양자 컴퓨터: 빠른 연산으로 전통적인 비대칭키 암호(RSA, ECC)를 무력화할 수 있어, 양자 내성 암호(Post-Quantum Cryptography) 연구가 진행 중이다.

## 3. 디지털 서명 (Digital Signature)

### 3.1 정의 및 목적

계약서에 날인하면 계약이 성립한 것처럼, 디지털 환경에서 문서의 유효성·위변조 방지·부인 방지를 보장하는 기술이다.

| 목적 | 설명 |
|------|------|
| 무결성 | 문서가 전송 중 변조되지 않았음을 확인한다. |
| 부인 방지 | 서명자가 나중에 서명 사실을 부인할 수 없다. |

### 3.2 동작 방식

| 단계 | 동작 |
|------|------|
| 1. 서명 생성 | 원문의 해시를 계산한 뒤, 개인키로 서명(암호화)한다. |
| 2. 전송 | 원문 + 서명 + 공개키(또는 인증서)를 수신자에게 전달한다. |
| 3. 검증 | 수신자가 원문을 해시한 뒤, 공개키로 서명을 복호화하여 해시를 비교한다. 해시가 일치하면 위변조되지 않았다고 판단한다. |

개인키로 서명할 수 있는 것은 해당 공개키의 소유자뿐이므로, 공개키로 검증 성공 시 서명자 신원과 부인 방지가 확보된다.

### 3.3 SSL/TLS 인증서와의 관계

인증서 내부에는 서버의 공개키가 포함된다. CA(인증 기관)가 인증서에 디지털 서명하여, 브라우저가 해당 서버가 신뢰할 수 있는지 검증할 때 이 기술이 사용된다. (01.checksum_hash.md 4.2 참조)

## 4. 실무 활용

### 4.1 대칭키 vs 비대칭키 선택

| 상황 | 권장 |
|------|------|
| 대용량 데이터 암호화 | 대칭키(AES). 효율이 높다. |
| 키 교환, 인증, 서명 | 비대칭키(RSA, ECC). 키 배포 문제를 해결한다. |
| 하이브리드 | 실제 통신은 대칭키로 암호화하고, 대칭키 자체는 비대칭키로 암호화하여 전달한다. (SSL/TLS 핸드셰이크) |

### 4.2 비대칭키의 활용: 암호화폐 지갑

암호화폐 지갑의 본질은 비대칭키 쌍이다. 공개키(지갑 주소)는 공개되고, 개인키는 본인만 보관한다. 개인키를 분실하면 자산에 접근할 수 없으며, 유출 시 도난 위험이 있다.
