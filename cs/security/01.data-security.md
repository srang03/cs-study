# 데이터 보안: DLP와 DRM

## 개요

데이터 보안은 조직의 핵심 정보 자산을 보호하는 영역이다.
대표적인 솔루션으로 DLP(유출 경로에서의 탐지·차단)와 DRM(문서 자체 암호화·권한)이 있으며, 이 둘은 목적이 다르므로 상호 보완적으로 운용된다.

엔드포인트(PC·노트북) 보안을 이해할 때 DLP는 “데이터가 어떤 내용인지·어디로 나가는지”에 초점을 둔 통제이고, MDM(모바일 디바이스 관리) 등은 “기기 자체의 구성·앱·카메라”를 다루는 통제이다. 기능이 겹쳐 보일 수 있으나 역할이 동일하지는 않다(아래 [5. 엔드포인트 통제: DLP와 MDM](#5-엔드포인트-통제-dlp와-mdm) 참고).

## 1. DLP (Data Loss Prevention, 데이터 유출 방지)

### 1.1 목적

기밀 정보(개인정보, 소스코드, 재무 데이터 등)가 정책상 허용되지 않은 채널·상대로 유출되는 것을 탐지하고 차단하며, 필요 시 로그를 남긴다.

### 1.2 판단에 쓰는 정보

DLP는 보통 행위(채널·목적지) 와 내용·맥락을 함께 본다.

- 채널·맥락: USB 복사, 메일 첨부, 웹 업로드, 인쇄, 외부 도메인 등
- 내용 기반: 정규식·키워드·카드번호 검증(Luhn 등), 문서 파싱으로 추출한 텍스트
- 메타데이터·라벨: 민감도 라벨(예: Microsoft Purview 민감도 라벨), 저장 경로, 등록 문서 지문·해시 매칭 등
- 이미지·스캔 문서: 그림으로만 존재하는 글자는 OCR로 문자열을 만든 뒤 위 규칙을 적용하는 경우가 있다. Word·엑셀·텍스트 레이어 PDF 등은 파일 포맷 파서(텍스트 추출) 로 읽는 것이 일반적이며, 이를 OCR이라 부르지 않는다.

### 1.3 유형

실무에서는 흔히 엔드포인트와 네트워크(게이트웨이) 로 나누어 설명한다. 그 외에 클라우드(SaaS·메일·드라이브 API 연동) · 저장소 스캔(휴지기 데이터 발견) 을 별도로 두는 구성도 있다.

| 유형 | 설치·관측 위치 | 주요 차단·탐지 대상 |
|------|----------------|---------------------|
| Endpoint DLP | 사용자 PC(에이전트, 필요 시 커널 필터 등) | USB·외장 저장소 복사, 인쇄, 클립보드, 일부 메일·브라우저 전송 등 단말에서 나가기 직전의 경로 |
| Network DLP | 메일 게이트웨이, 웹 프록시·SWG 등 트래픽 관문 | SMTP, HTTP(S) 등 회사가 통제하는 출구를 지나는 메일·웹 업로드 |

역할 분리: USB에 파일만 복사하고 네트워크로 보내지 않는 유출은 네트워크 DLP만으로는 통상 탐지할 수 없다. 반대로 프록시를 타지 않는 직접 경로는 네트워크 DLP가 놓칠 수 있어 엔드포인트 DLP가 보완하는 경우가 많다.

### 1.4 엔드포인트 DLP의 동작(개념)

- 상시 서비스: 정책 동기화, 로그 전송, 사용자 알림
- 채널별 관문: 파일이 이동식 볼륨에 쓰이기 전 등을 파일 시스템 필터(예: Windows 미니필터) 로 가로채거나, 인쇄·클립보드·특정 앱 연동 등으로 데이터를 검사
- 정책 엔진: 추출·분류 결과와 정책을 매칭해 허용·차단·경고 결정

배포는 Intune·SCCM·GPO 등 PC 관리 도구로 에이전트를 설치·갱신하는 방식이 일반적이다.

### 1.5 네트워크 DLP의 동작(개념)

클라이언트 트래픽이 메일 서버 앞단·프록시 등을 지날 때 본문·첨부를 디코딩해 검사한다. HTTPS 본문 검사는 조직 정책에 따라 SSL/TLS 검사(중간 복호화) 가 필요할 수 있으며, 법·프라이버시 검토가 따른다.

### 1.6 민감도 라벨과 Rights Management(참고)

민감도 라벨은 조직이 정한 분류(예: 일반·내부용·기밀)를 파일·메일 등에 붙이는 메타데이터이다. Microsoft 환경에서는 Microsoft Purview Information Protection(과거 Azure Information Protection 계열과 연속선상)에서 라벨·자동 라벨링을 구성하는 경우가 많다.

라벨에 보호(암호화) 를 결합하면 Rights Management(RMS 등) 로 문서 내용 접근이 권한과 연동된다. 이 경우 라벨 표시만 지운다고 내용이 공개되는 것은 아니다(정책·제품 구성에 따름).

### 1.7 운영 연동

DLP 이벤트는 SIEM으로 Syslog·CEF·API 등을 통해 전달되어 상관 분석·알림에 쓰이는 경우가 많다.

### 1.8 한계

DLP를 우회하거나 통제 밖 기기에서 반출되면, 암호화되지 않은 파일은 누구나 열람할 수 있다. DRM·접근 통제·교육·엔드포인트·네트워크 통제를 겹치는 다층 방어가 필요하다.

## 2. DRM (Digital Rights Management, 디지털 권한 관리)

기업 보안에서는 E-DRM(Enterprise DRM) 또는 문서 보안이라는 명칭으로 사용된다.

### 목적

문서 자체를 암호화하여, 유출되더라도 권한 없는 사용자는 열람할 수 없도록 보호한다.

### 작동 방식

1. 문서 생성/저장 시점에 파일 자체를 암호화한다
2. 사용자별로 차등 권한을 부여한다 (읽기, 수정, 인쇄 등)
3. 문서 열람 시 암호화·권한 서버에 권한을 조회한 후, 허가된 범위 내에서만 접근을 허용한다

권한 설정 예시:

- A 팀장: 읽기 / 수정 / 인쇄 가능
- B 사원: 읽기만 가능
- 외부인: 열람 불가

### 한계

사용자 경험(UX)이 저하될 수 있으며, 에이전트 설치가 필요하다. 권한 서버 장애 시 문서 접근이 불가능해지는 가용성 리스크가 존재한다.

## 3. DLP vs DRM 비교

| 구분 | DLP | DRM |
|------|-----|-----|
| 보호 초점 | 데이터 이동(채널·목적지) | 데이터 파일 자체 |
| 핵심 기술 | 내용·라벨 탐지 및 전송 차단 | 파일 암호화 및 권한 통제 |
| 유출 직후 | 암호화되지 않은 파일이면 열람 가능 | 권한 없으면 열람 불가 |
| 보안 분류 | 데이터 보안(관측은 엔드포인트·네트워크에 걸림) | 데이터 보안 |

## 4. 실무 적용: 다층 방어 (Defense in Depth)

실무에서는 DLP와 DRM을 함께 운용하여 다층 방어 체계를 구성한다.

```
[문서 생성] → DRM·라벨·암호화 (파일 자체 보호)
     ↓
[외부 전송 시도] → DLP 차단·감사 (경로 통제)
```

- DRM(및 민감도 라벨+보호): 유출되어도 권한 없는 주체는 열람하기 어렵다
- DLP: 비인가 채널로의 반출 시도를 줄이고 기록한다

단일 솔루션의 한계를 보완하기 위해 여러 계층의 보안 통제를 중첩 적용하는 것이 다층 방어의 핵심 원칙이다.

## 5. 엔드포인트 통제: DLP와 MDM

MDM(Mobile Device Management) 은 엄밀히는 “모바일만”이 아니라, Intune 등으로 PC·모바일을 등록하고 구성 프로필·앱·준수 정책을 원격으로 적용하는 기기 수준 관리이다.

| 구분 | DLP(엔드포인트) | MDM(및 유사 PC 관리) |
|------|-----------------|----------------------|
| 중심 질문 | “이 데이터가 민감한가, 어디로 나가는가?” | “이 기기가 조직 정책에 맞게 설정되어 있는가?” |
| 전형적 기능 | 채널별 유출 검사·차단, 콘텐츠·라벨 정책 | 등록·인벤토리, OS 업데이트, Wi-Fi/VPN, 앱 배포, 카메라·스크린샷 제한 등 |
| 관계 | MDM으로 DLP 에이전트 배포, USB 저장소 사용 제한(OS 정책) 등을 할 수 있으나, 그 자체가 문서 내용까지 검사하는 DLP는 아니다 |

즉 “통제한다”는 점에서 겹쳐 보일 수 있으나, DLP는 데이터 중심(내용·채널), MDM은 기기·구성 중심으로 구분하는 것이 정확하다.

## 6. DLP·DRM과 자주 함께 등장하는 통제

| 항목 | 설명 | DLP·DRM과의 관계 |
|------|------|-------------------|
| BitLocker 등 디스크 암호화 | 분실·도난 시 디스크 읽기 어렵게 함 | 유출 경로 차단(DLP) 과 목적이 다르다. 상호 보완 |
| 이동식 저장소 정책 | GPO·MDM 등으로 USB 사용 자체 제한 | DLP의 “내용 검사 후 차단”과 병행 가능 |
| SIEM | 로그 수집·상관분석 | DLP 탐지·차단 이벤트 연동 |
