# 체크섬과 해시 (Checksum and Hash)

SSL 및 암호기술 이해를 위한 기초 개념. 데이터 오류 검출의 기본인 체크섬에서 보안의 핵심인 암호학적 해시 함수까지 다룬다.

---

## 1. 체크섬 (Checksum)

### 1.1 정의 및 동작 방식

체크섬은 네트워크 통신이나 저장 장치에서 데이터 전달 시 발생한 우연한 오류(손상) 여부를 확인하기 위해 사용되는 가장 기본적인 방법이다.

- 전송할 데이터의 각 단위를 더하여(Sum) 검사값을 생성한다.
- 일정 자릿수를 정해두고, 해당 범위를 넘는 자리 올림(Carry)은 버리거나 모듈로(Modulo) 연산으로 자릿수를 고정한다.

### 1.2 사용 목적과 한계

| 구분 | 내용 |
|------|------|
| 목적 | 통신 노이즈 등 단순 오류 검출 → 통신 신뢰성 확보 |
| 한계 | 악의적인 변조 시 체크섬 값까지 함께 조작 가능 → 보안 목적에는 부적합 |

---

## 2. 암호학적 해시 함수 (Cryptographic Hash Function)

해시 함수는 임의 길이의 데이터를 입력받아 고정 길이의 결과값(해시값, 다이제스트)을 출력하는 함수이다. SSL을 비롯한 IT 보안 전반에서 데이터 무결성(Integrity) 확보에 필수적으로 사용된다.

### 2.1 기본 특징

| 특징 | 설명 |
|------|------|
| 고정 출력 길이 | 입력이 1바이트이든 100GB이든, 동일 알고리즘은 항상 같은 길이의 결과를 반환한다. |
| 눈사태 효과 (Avalanche Effect) | 입력값의 1비트만 변경되어도 출력 해시값은 완전히 달라진다. |
| 단방향성 (One-Wayness) | 해시값으로부터 원본 입력값을 역산(복호화)하는 것은 수학적으로 불가능하다. |

### 2.2 암호학적 보안 요건

출력 길이가 고정되어 있으므로, 입력값이 무한하다면 비둘기집 원리에 의해 서로 다른 입력이 같은 해시값을 갖는 해시 충돌(Collision)이 이론적으로는 반드시 존재한다. 따라서 암호학적 해시는 "충돌이 없다"가 아니라 "충돌을 찾는 것이 현실적으로 불가능하다"를 만족해야 한다.

| 요건 | 설명 |
|------|------|
| 충돌 저항성 (Collision Resistance) | 동일한 해시값을 생성하는 서로 다른 두 입력을 찾는 것이 계산상 불가능해야 한다. |

---

## 3. 대표적인 해시 알고리즘

> SHA-128은 공식 규격 명칭이 아니다. 일반적으로 SHA-256 이상의 SHA-2 계열을 사용한다.

| 알고리즘 | 상태 | 비고 |
|----------|------|------|
| MD5 | 사용 금지 | 과거에 널리 사용되었으나 치명적인 충돌 취약점 발견. 보안 목적 사용 금지. 단순 파일 무결성 확인용으로만 간혹 사용. |
| SHA-1 | Deprecated | MD5 이후 표준이었으나 취약점 발견. 대부분의 웹 브라우저와 SSL 인증서에서 지원 중단. |
| SHA-2 계열 | 현재 표준 | SHA-224, SHA-256, SHA-384, SHA-512 등. SSL 인증서, 블록체인 등에서 널리 사용. |

---

## 4. 해시 기술의 실무 활용

### 4.1 패스워드 보호

사용자 인증 요소(지식·소지·생체·위치) 중 지식(패스워드) 기반 인증에서 해시가 핵심 역할을 한다.

| 기법 | 설명 |
|------|------|
| 해시 저장 | 패스워드를 평문이 아닌 해시값으로 변환하여 저장. DB 유출 시에도 원본 패스워드 노출 방지. |
| 솔트(Salt) | 원본 패스워드에 무작위 문자열을 덧붙인 후 해시. Rainbow Table 공격을 무력화하고, 사용자별 고유 솔트로 역추적을 현실적으로 불가능하게 만든다. |

### 4.2 데이터 무결성 검증 및 SSL/TLS

| 활용 | 설명 |
|------|------|
| 전자서명 (Digital Signature) | 송신자가 데이터의 해시값을 구한 뒤 개인키(PKI)로 서명하여 전송. 수신자는 공개키로 검증하여 위변조 여부를 확인한다. |
| SSL/TLS 인증서 검증 | 브라우저가 서버 인증서가 신뢰할 수 있는 CA에서 발급되었고 위조되지 않았는지 검증할 때, 전자서명(해시 + PKI) 기술이 사용된다. |

### 4.3 블록체인

| 개념 | 설명 |
|------|------|
| 해시 체이닝 (Hash Chaining) | 각 블록에 이전 블록의 해시값을 포함시켜 사슬처럼 연결. `Hash(이전 블록 해시 + 현재 데이터) = 현재 블록 해시` 구조. |
| 위변조 방지 | 특정 블록을 조작하면 눈사태 효과로 해당 블록의 해시가 변경되고, 이후 모든 블록의 해시가 연쇄적으로 어긋나 과거 데이터 조작이 사실상 불가능해진다. |
