# Java Set과 비선형 컬렉션 학습

## 1. Set과 비선형 자료구조

Set은 List와 달리 비선형 구조로 객체를 관리하는 인터페이스이다. 대상 요소가 연속적으로 저장되지 않는 비선형 구조이며, 객체를 중복해서 저장할 수 없고 요소 검색 속도가 빠르다는 특징이 있다.

비선형 구조를 사용하는 이유는 검색 성능을 높이기 위함이다. 선형 구조(배열, 연결 리스트)는 전체 순회에 유리하고, 비선형 구조는 특정 요소 검색에 유리하다. 데이터량이 많거나 검색·중복 제거가 중요할 때 Set 같은 비선형 컬렉션이 성능상 유리한 경우가 많다.

---

## 2. Set 인터페이스와 구현체

Set은 순서를 보장하지 않거나(또는 정렬 순서만 보장), 중복을 허용하지 않는 비선형 컬렉션의 대표 인터페이스이다. 대표 구현체로 HashSet과 TreeSet이 있다.

### 2.1 Set 인터페이스 개요

Set 인터페이스는 중복 없는 요소의 모음을 표현한다. 삽입 순서를 보장하지 않으며, 구현체에 따라 정렬 순서를 유지할 수도 있다. List와 달리 인덱스로 접근하지 않고, 요소 존재 여부 확인·추가·삭제·순회에 초점을 둔다.

### 2.2 HashSet

| 구현체 | 내부 구조 | 특징 |
|--------|-----------|------|
| HashSet | 해시 테이블 | 조회(READ) 위주 연산에 유리, 삽입·조회 평균 O(1), 순서 보장 안 함, null 요소 1개 허용 |

내부적으로 `HashMap`을 이용한 해시 테이블로 요소를 관리한다. 조회·존재 여부 확인이 빈번한 경우에 적합하다.

### 2.3 TreeSet

| 구현체 | 내부 구조 | 특징 |
|--------|-----------|------|
| TreeSet | 트리(레드-블랙) | 갱신(UPDATE)·정렬 위주에 유리, 정렬 순서 유지, null 불가, Comparator로 사용자 정의 정렬 지원 |

정렬이 필요하거나 범위 검색·요소 갱신이 빈번한 경우 TreeSet을 고려할 수 있다.

### 2.4 HashSet과 TreeSet 선택

- 조회(READ) 위주·대량 데이터: HashSet — 삽입·조회 평균 O(1), 순서 불필요 시 유리.
- 정렬·갱신(UPDATE) 위주: TreeSet — 정렬 순서 유지, 범위 검색·Comparator 활용 시 유리.

## 3. 이터레이터와 순회

이터레이터(Iterator) 디자인 패턴을 통해 컬렉션 전체를 순회하며 데이터에 접근할 수 있다. 열거자를 통해 요소를 하나씩 넘겨받아 순회하는 방식이다.

비선형 구조를 이터레이터로 순회하는 것은 선형 구조에 비해 부하가 될 수 있다. 순회가 꼭 필요할 때만 사용하거나, 반복문과 이터레이터 사용 시점을 고려하는 것이 좋다.

---

## 4. HashSet과 멀티스레드

HashSet은 스레드 안전하지 않다. 멀티스레드 환경에서 여러 스레드가 동시에 같은 Set에 삽입·삭제·조회를 수행하면 동기화 문제가 발생할 수 있다.

`HashSet<String> set = new HashSet<>();`를 `Set<String> set = Collections.synchronizedSet(new HashSet<>());`로 변경함으로써 동기화된 래퍼를 통해 스레드 안전성을 확보할 수 있다.

---

## 5. 요소 중복 판정

Set 컬렉션은 요소 객체의 중복을 허용하지 않는다. 기본적으로 인스턴스(참조) 비교를 사용하므로, 내용이 같더라도 인스턴스가 다르면 서로 다른 객체로 판정된다.

인스턴스가 다르더라도 내용이 같으면 중복으로 처리하려면, 해당 요소 타입에서 `hashCode()`와 `equals()`를 오버라이딩하여 내용 기준 비교 로직을 재정의해야 한다. Set은 두 메서드를 사용해 중복 여부를 판단한다.

---

## 6. Map과 비선형 구조 (참고)

Map은 키(Key)와 값(Value)으로 구성되는 `Map.Entry` 요소를 비선형 구조로 관리한다. 값은 중복이 허용되지만 키는 중복을 허용하지 않는다. 키를 기준으로 값 조회가 가능하며, 값을 기준으로 찾으려면 엔트리 전체를 순회해야 한다.

---

## 7. 선형·비선형 혼합 활용

동일한 정보를 두 가지 자료구조로 나누어 관리할 수 있다. 예를 들어 선형 구조(배열, 리스트)와 비선형 구조(Set, Map)를 각각 사용할 수 있다.

- 선형 구조: 인덱스로 순차 접근이 가능한 구조는 전체 순회 시 이터레이터 없이도 순회하기에 유리한 경우가 있다. 전체 순차 접근에 적합하다.
- 비선형 구조: 검색 속도가 빠르다. 특정 요소 존재 여부나 키로 값을 찾는 연산에 유리하다.

초기 로딩이 다소 느린 것은 수용 가능한 경우가 많지만, 런타임에서의 성능 이슈는 큰 문제가 될 수 있다. 접근 패턴(전체 순회 vs 검색 빈도)을 고려해 선형 구조와 비선형 구조를 조합하여 선택할 수 있다.
