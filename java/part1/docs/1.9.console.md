# 콘솔과 버퍼

## 1. 콘솔

텍스트 기반으로 입력·출력을 수행하는 장치를 콘솔이라 한다.
CLI(Command Line Interface) 기반 HCI(Human Computer Interface)는 키보드 입력으로 구현된다.
키보드 입력 시 그 값은 메모리(I/O 버퍼)에 연속적으로 저장된다.
I/O 버퍼에서 바이트 또는 키 입력 단위로 처리한다.

키를 누르면 하드웨어 수준에서 스캔 코드가 발생하고, 이를 표준화된 키코드로 변환하여 장치에서 인식한다.

## 2. 버퍼

버퍼는 충격을 완화하는 완충 장치이다.
예: 영상 재생 시 데이터를 버퍼에 채워 두고, 어느 정도 채워지면 재생을 시작하면서 다른 쪽에서는 버퍼를 계속 채워 나간다.
또 예: 네트워크에서 실시간으로 데이터를 받다가 일시적으로 끊기는 경우에도, 버퍼에 쌓아 둔 데이터로 끊김 없이 재생할 수 있다.
데이터가 송수신되지 못하더라도 미리 쌓아 둔 데이터로 끊김 없이 보여 줄 수 있는 것이 버퍼링이다.

콘솔 입출력에서도 버퍼가 사용된다.
사용자가 키보드로 입력하면 입력한 정보가 I/O 버퍼에 쌓이고, CPU는 버퍼에서 데이터를 가져와 처리한다.
출력은 화면에 보여 주는 것을 의미하며, 키보드 입력을 화면에 보여 줄 때에도 output buffer가 존재한다.
출력(또는 입력) 후 버퍼를 비우는 것을 플러싱(flushing)이라 한다.

## 3. 콘솔 입력 키코드 값 읽기

버퍼에서 한 바이트를 읽어 정수(키코드)로 변환할 수 있다.

```java
    int keycode = System.in.read();
    System.out.println(keycode);
```

`read()` 메서드는 버퍼에서 한 바이트만 꺼낸다.
버퍼가 비어 있으면 반환할 수 없어 대기(wait) 상태가 된다.
사용자가 입력하면 한 바이트가 꺼내지고, `read()`를 다시 호출하면 버퍼에서 다음 바이트를 꺼낸다.
정수(`int`)로 받은 keycode를 `char`로 변환해 출력하면 해당 코드의 문자가 출력된다.
다만 UTF-8처럼 한 글자가 여러 바이트인 문자(예: 한글 '가'는 3바이트)는 `read()`가 1바이트씩만 읽기 때문에, 인코딩이 깨져 보일 수 있다.
콘솔에서 한글 등 다바이트 문자를 안전히 읽으려면 `InputStreamReader`(또는 `Scanner`)를 사용한다.

## 4. Java의 인코딩 방식

- JVM 내부: 문자열은 UTF-16BE로 표현된다.
- I/O(콘솔, 파일 등): 운영체제 기본 인코딩 또는 지정한 문자 집합(예: UTF-8)을 사용한다. 입출력이 일어날 때 바이트 스트림은 이 인코딩으로 버퍼에 들어가거나 나온다.
- UTF-8: 유니코드를 1~4바이트 가변 길이로 인코딩한다. 예: '가'는 3바이트(0xEA, 0xB0, 0x80)이므로, `System.in.read()`로는 1바이트(0xEA)만 읽혀 '가'로 복원되지 않는다.
- Modified UTF-8: 자바 일부 API(예: 직렬화, JNI)에서는 표준 UTF-8이 아닌 modified UTF-8을 사용한다. null 문자(0x0000)는 UTF-8에서는 1바이트 0이지만, modified UTF-8에서는 0xC0 0x80(2바이트)으로 인코딩된다.

JVM 내부 문자열 표현(UTF-16BE)과 입출력 시 사용하는 인코딩(I/O 인코딩)을 구분해서 이해하면 된다. 콘솔 `read()`/`write()`는 바이트 단위이므로 I/O 인코딩에 따라 동작한다.

## 5. Scanner

정수, 문자열 등을 한 바이트씩 읽지 않고 토큰 단위로 해석·형변환하기 위해 `Scanner` 클래스를 사용한다.
`Scanner`를 사용하면 입력 버퍼의 내용을 원하는 형(정수, 실수, 문자열 등)으로 파싱할 수 있다.
`Scanner`는 생성 시 스캔할 대상을 반드시 지정해야 하며, 키보드 입력을 읽을 때는 생성자에 `System.in`을 넘긴다.
입력 데이터를 해석할 수 없을 때(예: 정수 기대 시 문자가 입력된 경우) 예외가 발생하므로, 예외 처리 로직으로 처리해 줘야 한다.

## 6. Scanner로 한 행·토큰 읽기

키보드 입력은 버퍼에 문자가 순서대로 쌓인다(상세는 2절 참고).
한 행 단위로 구분할 때는 개행 문자 `\n`을 구분자로 사용한다. `\n`은 이스케이프 시퀀스로 표현되는 개행 문자이며, 눈에 보이지 않지만 문자로 존재한다.

- `nextLine()`: 개행 문자(`\n`)가 나올 때까지 버퍼를 읽어 한 행 전체를 문자열로 반환한다.
- `nextInt()`: 다음 토큰을 정수로 해석해 반환한다.
- `next()`: 공백(화이트스페이스: 탭, 공백, 엔터 등)이 나올 때까지 읽어 토큰 하나를 문자열로 반환한다.

한 글자(1바이트)만 읽을 때는 `(char) System.in.read()`를 사용할 수 있으나, 한글 등 다바이트 문자는 깨질 수 있다(3절 참고).

`nextInt()`로 정수를 받은 뒤 엔터를 치면 버퍼에는 정수 값과 개행 문자가 남는다. 이어서 `nextLine()`을 호출하면 남은 개행 문자만 읽히고, 그다음에 입력한 문자열은 읽지 못한다.
이 경우 정수 입력 후 버퍼를 비워 개행 문자를 제거한 뒤 `nextLine()`을 사용해야 한다.
