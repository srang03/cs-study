# 기본 언어 시작

## 1. 항, 식, 구문

피연산자(리터럴, 변수, 메서드 호출 등)와 연산자가 결합해 식(expression)을 이룬다.
식은 평가(evaluation)할 수 있으며, 그 결과 하나의 값을 가진다.
여러 식을 모아서 하나의 구문(statement)을 이루고, 구문이 고급어의 실행 단위가 된다.
구문은 기본적으로 위에서 아래로 순차적으로 실행된다.

## 2. 컴파일러와 인터프리터

- 컴파일러
  - 고급어 소스코드를 기계어 또는 바이트코드 등 목적 코드로 번역하는 프로그램이다.
  - 전체 소스코드를 목적 코드로 변환한 후 실행한다.
  - 성능 최적화가 용이하여 다수 언어가 채택하고 있다.
  - 컴파일 타임에 문법 오류를 잡을 수 있다.

- 인터프리터
  - 고급어 소스코드를 직접 실행하는 프로그램이나 환경을 의미한다.
  - 보통 한 번에 한 줄씩 실행한다.
  - 성능(속도)면에서 컴파일러보다 느리다.
  - 유연성 측면에서는 컴파일러보다 좋을 수 있다.

### 2.1 자바: 하이브리드

자바를 완벽하게 컴파일러 언어라고 분류하지 않는다.
작성한 `.java` 파일을 컴파일러를 통해 `.class`(바이트코드) 파일로 변환하고, JVM이 이 바이트코드를 해석·실행한다. 필요 시 JIT 컴파일로 기계어로 변환하여 실행한다.
이 때문에 하이브리드 언어로 분류할 수 있다.
