# 중요한 시점

자바 프로그램이 소스 작성부터 실행까지 거치는 네 가지 시점을 구분한다.

## 1. 코드 작성 시점

`.java` 파일로 소스 코드를 작성한다.

## 2. 빌드 타임

자바 컴파일러가 `.java` 파일을 `.class` 파일로 번역한다.
자바에서는 빌드 시점에 외부 라이브러리를 한 실행 파일로 묶지 않는다(C처럼 정적 링킹하지 않는다). `.class`에는 해당 클래스·메서드를 가리키는 심볼릭 레퍼런스만 있으며, 실제 결합(해석)은 JVM이 런타임에 수행한다.

## 3. 링크 타임

일반적으로 링커가 모듈을 결합하는 링킹을 수행하고, `.class` 파일 간 의존성을 파악해 연결한다.
자바에는 별도 링커 단계가 없다. 이에 해당하는 작업(클래스 해석, resolution)은 JVM이 런타임에 수행한다.

## 4. 런타임

JVM의 ClassLoader가 절차에 따라 바이트코드를 로딩한 뒤 메모리에 적재한다.
필요한 클래스를 찾지 못하면 빌드 타임 오류가 아닌 런타임 오류가 발생한다.
자바에서는 빌드 타임 안에 컴파일만 포함되고, 링크(클래스 해석)는 런타임 안에 수행된다.
