# 메서드

## 1. 메서드의 개념

메서드(함수)는 절차적 흐름을 가진 여러 구문을 `{ }`로 묶어 사용하는 단위 코드이다.
클래스를 구성하는 요소 중 하나이다.

- 프로그램의 시작점은 `main` 메서드이다. `main`의 시작과 끝이 프로그램의 실행 범위와 같다.
- 여러 메서드는 호출을 통해 서로 연결된다(바인딩).

## 2. 문법 특성

메서드는 반환 타입, 메서드 이름, 매개변수 목록으로 선언한다.
접근 제어자·수식자를 붙일 수 있다.

```java
public static int add(int a, int b) {
    return a + b;
}
```

- 호출자와 피호출자: 메서드를 호출하는 쪽이 호출자, 호출당하는 쪽이 피호출자이다. 호출 시 인수를 넘겨 피호출자 메서드의 매개변수에 대응한다.
- 반환: `return`문으로 반환 시점과 반환값을 지정한다. 반환 타입이 `void`이면 `return`문 없이 메서드 끝에 도달해도 제어가 호출자로 돌아간다. 반환 타입이 `void`가 아니면 `return`문으로 값을 반환해야 한다.

## 3. 멤버 변수와 식별자 검색 순서

멤버 변수(필드)는 메서드처럼 클래스를 이루는 멤버이다.
인스턴스 변수·클래스 변수를 아우르는 말이며, 클래스에 속한 모든 메서드가 접근할 수 있고 서로 데이터를 공유할 수 있다.
변수의 종류와 스택·heap 배치는 1.8 변수와 GC를 참고한다.

식별자 검색 순서는 다음과 같다.

1. 지역 스코프(해당 블록 내 지역 변수·매개변수)
2. 해당 메서드 바디 전체
3. 클래스 멤버(인스턴스·클래스 변수)까지 확장

## 4. 기초적인 메서드 설계 원칙

- UI와 기능은 분리한다.
- 재사용 가능한 단위는 메서드로 구현한다(DRY 원칙).

## 5. 이벤트 루프(메인 루프)

이벤트 루프는 `main` 메서드에서 사용자 인터페이스(메뉴 등) 출력과 사용자 입력을 반복하는 구조이다.
여기서 말하는 이벤트 루프는 CLI 프로그램의 메인 루프(메뉴 반복 구조)를 가리키며, GUI의 이벤트 디스패치 루프와는 구분된다.

- 메뉴 출력 → 사용자 선택 확인 → 선택에 따라 기능 수행 순서로 동작한다.
- 종료를 선택할 때까지 위 단계를 반복한다.
- 많은 CLI 응용 프로그램이 채택하는 일반적인 구조이다.

## 6. 매개변수 전달

Java에서는 인자 전달이 항상 값에 의한 전달(pass by value)이다.

- 원시 타입: 값 자체가 복사되어 전달된다.
- 참조 타입: 참조(주소) 값이 복사되어 전달된다. 즉, 참조값이 복사될 뿐이며, C의 포인터나 Call by reference와는 다르다. 다른 언어에서는 Call by reference를 쓰는 경우가 있으나, Java에서는 모든 인자가 값(또는 참조값)으로 전달된다.

호출 시 넘기는 값은 인수(argument, 아규먼트), 메서드에서 받는 변수는 매개변수(parameter, 파라미터)라 부른다. 같은 맥락에서 쓰이는 말이다.

- 매개변수는 스택에 할당되는 지역 변수와 같은 성격이다.
- `new`로 생성한 객체는 heap에 할당된다.
- `String`은 참조 타입이지만 기본형처럼 다뤄지는 경우가 많다는 점에 유의한다.

## 7. 메서드 호출과 스택 변화

메서드 호출 시 스택에 해당 메서드의 프레임이 쌓이고, 반환 시 LIFO 순서로 제거된다.

예: `main`이 `foo()`를 호출하면 `main` 프레임 위에 `foo` 프레임이 쌓인다. `foo`가 `return`하면 `foo` 프레임이 제거되고 제어가 `main`으로 돌아간다.
메서드 호출·반환에 따른 스택 동작은 1.8 변수와 GC를 참고한다.

## 8. 재귀 호출

재귀 호출은 메서드 내부에서 자신을 다시 호출하는 것이다.
반복문과 스택 구조가 결합한 형태이며, 비선형 자료구조에서 자주 사용한다.

- 호출이 반복될수록 스택에 프레임이 쌓이므로 호출 오버헤드가 있다.
- 종료 조건이 잘못되면 무한 재귀에 빠져 스택 오버플로우가 발생할 수 있다.

## 9. 다중 정의(Overloading)

다중 정의(오버로딩)는 같은 메서드 이름으로 매개변수 목록(개수·타입)만 다르게 선언하는 것이다.
반환 타입은 오버로딩 판별에 사용하지 않는다.

```java
public static int add(int a, int b) {
    return a + b;
}
public static double add(double a, double b) {
    return a + b;
}
```

다양한 타입에 대해 같은 기능을 제공할 때 자주 사용한다.
