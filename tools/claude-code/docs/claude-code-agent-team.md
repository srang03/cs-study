# Claude Code 에이전트 팀

> 실험적(experimental) 기능 — 기본 비활성화, 명시적 활성화 필요

## 개요

에이전트 팀을 사용하면 여러 Claude Code 인스턴스가 함께 작동하도록 조율할 수 있다.
한 세션이 팀 리더 역할을 하여 작업을 조율하고, 작업을 할당하며, 결과를 종합한다.
팀원들은 각각 자신의 컨텍스트 윈도우에서 독립적으로 작동하며 서로 직접 통신한다.

단일 세션 내에서 실행되고 메인 에이전트에게만 보고할 수 있는 subagent와 달리, 리더를 거치지 않고 개별 팀원과 직접 상호작용할 수도 있다.

## 활성화

환경변수 설정 또는 settings.json에서 활성화:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

## 아키텍처

| 구성요소 | 역할 |
|----------|------|
| Team lead | 메인 세션, 작업 조율 |
| Teammates | 독립 세션, 각자 컨텍스트 윈도우 보유 |
| Task list | 팀원이 claim/완료하는 공유 작업 목록 |
| Mailbox | 에이전트 간 메시징 시스템 |

역할 예시: Backend Agent, Frontend Agent, Test Agent, DevOps Agent

## 적합한 사용 사례

- **연구 및 검토**: 여러 팀원이 문제의 다양한 측면을 동시에 조사한 후 서로의 발견 사항을 공유
- **새로운 모듈 또는 기능**: 팀원들이 각각 별도의 부분을 소유하며 서로 간섭하지 않는 작업
- **경쟁하는 가설로 디버깅**: 팀원들이 다양한 이론을 병렬로 테스트하고 더 빠르게 답에 수렴
- **교차 계층 조율**: 프론트엔드, 백엔드, 테스트에 걸친 변경사항을 각각 다른 팀원이 소유

## Agent Team vs Subagent

| 항목 | Agent Team | Subagent |
|------|-----------|----------|
| 통신 | 팀원끼리 직접 통신 | 리더에게만 보고 |
| 조율 | Task list 기반 자기 조율 | 리더가 모든 작업 조율 |
| 컨텍스트 | 각자 독립 컨텍스트 윈도우 | 각자 독립 컨텍스트 윈도우 |
| 토큰 비용 | 높음 (팀 규모에 비례) | 낮음 (결과만 요약) |
| 적합한 작업 | 병렬 연구, 토론이 필요한 작업 | 빠른 단일 작업 |

## 주의사항

- 에이전트 팀은 조율 오버헤드를 추가하여 단일 세션보다 훨씬 더 많은 토큰을 사용한다
- 팀원들이 독립적으로 작동할 수 있을 때 가장 잘 작동한다
- 순차적 작업, 동일 파일 편집, 또는 많은 종속성이 있는 작업의 경우 단일 세션이나 subagent가 더 효과적이다
- 한 세션당 하나의 팀만 생성 가능, 중첩 팀 불가
- 세션 재개(resume) 시 in-process 팀원은 복원되지 않음
