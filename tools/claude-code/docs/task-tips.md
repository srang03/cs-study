1-2. Lazy Loading 
CLAUDE.md에 모든 정보를 넣으면 매 세션마다 수천 토큰이 낭비된다. 참조만 두고 상세 내용은 별도 파일로 분리하기
Claude는 필요할 때만 해당 파일을 읽습니다. 이게 바로 Lazy Loading입니다.

## 프로젝트 문서

1-3. 컨텍스트 윈도우 관리
핵심 원칙: 한 세션 = 한 피처
하나의 세션에서 여러 기능을 연달아 구현하지 마세요.
신선한 컨텍스트가 부풀어진 컨텍스트보다 항상 낫습니다.

```
**실전 워크플로우:**

필요한 것만 검색해서 가져오게
컨텍스트 윈도우 절약 + 충분한 맥락 유지

1. **Plan Mode**에서 전체 작업을 설계하고 단계를 나눈다
2. `/clear` 또는 새 세션을 연다
3. 플랜의 **첫 번째 단계만** 구현한다
4. 완료되면 다시 `/clear` → 다음 단계로

/clear 컨텍스트 완전 초기화
/compact 대화 내용 압축 (맥락 유지)
```

1-6. 무거운 작업은 스크립트로 오프로드
무거운 데이터 처리를 대화 안에서 시키면 컨텍스트가 오염됩니다. 클로드에게 스크립트 작성하게하고 결과만 돌려주세요.

| 상황 | 프롬프트 예시 |
| --- | --- |
| DB 마이그레이션 검증 | "10만 행 CSV 파싱해서 무결성 검증 스크립트 작성해줘. 결과는 summary.json으로" |
| 로그 분석 | "수백 MB 로그에서 에러 패턴 추출 스크립트 만들어줘. [report.md](http://report.md)로 정리" |
| API 응답 비교 | "v1과 v2 응답 차이 비교 스크립트 작성해줘. diff를 api-diff.json으로" |
컨텍스트 관리

2-1. 플랜모드 사용
plan mode 4 단계
플랜 모드에서 클로드에게 작업 설명
클로드가 계획을 제시 (어떤 파일 수정, 어떤 접근할지)
계획을 리뷰하고 피드백
만족 하면 Accept Mode 전환하여 실행

플랜 없이 바로 실행하면 클로드가 엉뚱한 방향으로 코드를 대량 수정해버리는 참사가 벌어질 수 있음

2-2. TDD 기반 스마트 코딩
핵심 루프: 작은 변경 -> 테스트 -> 린트 -> 커밋 -> 반복
1. 기능 하나 추가
2. 테스트 돌려서 확인
3. 린트/포멧 체크
4. 문제없으면 커밋
5. 다음 기능으로 넘어가기


2-3. Thinking 로그 읽기
클로드가 생각하는 과정을 보여주는 thinking 로그를 무시하지 않기
클로드가 잘못된 가정을 하고 있다면, 그 순간 escape로 중단하기
잘못된 가정 위에 쌓은 코드는 전부 쓸모가 없으므로 초반에 잡는게 핵심


2-4. 다른 AI에게 비평 받기
클로드와 작업하다 막히면 /export로 대화 내보내서 chatgpt나 gemini에게 보여주기
이 대화를 분석해서 클로드가 놓치고 있는 것이나 잘못된 접근 있으면 지적해줘

자동화 팁: 커스텀 스킬 만들어서 /review-with-gpt 같은 명령으로 이 과정을 원커맨드로 처리

2-5. 로컬 TOOO.md
프로젝트 루트에 TODO.md를 만들어 클로드에게 테스트를 추적하게 하기

하루 시작 - 할일을 todo 리스트에 체크리스트로 작성
클로드에게 TODO 첫번째 항목 부터 시작
에이전트 팀즈로 여러 테스크를 병렬 처리 가능
세션 종료시 TODO.md 업데이트 해줘 -> 진행 상황 자동 반영

여러 세션에 걸쳐 작업의 연속성을 유지하는 핵심 도구이다.
