# Claude Code 실전 팁

워크플로우 전체 프로세스는 [workflow.md](workflow.md), 기본 사용법은 [claude-code-basic.md](claude-code-basic.md) 참조.
이 문서는 그 외 실전에서 유용한 팁만 정리한다.

## 1. 컨텍스트 관리

### 한 세션 = 한 피처

하나의 세션에서 여러 기능을 연달아 구현하지 않는다.
신선한 컨텍스트가 부풀어진 컨텍스트보다 항상 낫다.

실전 워크플로우:
1. Plan Mode에서 전체 작업을 설계하고 단계를 나눈다
2. `/clear` 또는 새 세션을 연다
3. 플랜의 첫 번째 단계만 구현한다
4. 완료되면 다시 `/clear` → 다음 단계로

| 명령 | 동작 |
|------|------|
| `/clear` | 컨텍스트 완전 초기화 |
| `/compact` | 대화 내용 압축 (맥락 유지) |

### Lazy Loading

CLAUDE.md에 모든 정보를 넣으면 매 세션마다 토큰이 낭비된다. 참조만 두고 상세 내용은 별도 파일로 분리한다.

```markdown
## 프로젝트 문서
- API 스펙: @docs/api-spec.md
- DB 스키마: @docs/db-schema.md
- 코딩 컨벤션: @docs/conventions.md
```

폴더별 CLAUDE.md (`src/auth/CLAUDE.md`, `src/payments/CLAUDE.md`)로 분리하면 해당 폴더 작업 시에만 로딩되어 컨텍스트 bloat을 방지한다.

### 무거운 작업은 스크립트로 오프로드

무거운 데이터 처리를 대화 안에서 시키면 컨텍스트가 오염된다. 스크립트를 작성하게 하고 결과만 전달한다.

| 상황 | 프롬프트 예시 |
|------|-------------|
| DB 마이그레이션 검증 | "10만 행 CSV 파싱해서 무결성 검증 스크립트 작성해줘. 결과는 summary.json으로" |
| 로그 분석 | "수백 MB 로그에서 에러 패턴 추출 스크립트 만들어줘. report.md로 정리" |
| API 응답 비교 | "v1과 v2 응답 차이 비교 스크립트 작성해줘. diff를 api-diff.json으로" |

## 2. 작업 추적

### 태스크 관리 판단 기준

큰 단위 작업(세션 한 번에 끝내기 어려운 경우)에는 태스크를 나눠서 진행 상태를 관리하면서 커밋하면 효과적이다.
단, 작업 사이즈가 크지 않으면 태스크 관리는 오버헤드가 된다. 안 나눠도 충분히 할 수 있는 작업을 굳이 분리하지 않는다.

### TODO.md 활용

프로젝트 루트에 TODO.md를 만들어 작업 연속성을 유지할 수 있다.

1. 하루 시작 — 할일을 체크리스트로 작성
2. Claude에게 첫 번째 항목부터 시작
3. 세션 종료 시 "TODO.md 업데이트해줘" → 진행 상황 자동 반영
4. 다음 세션에서 이어서 진행

## 3. 품질 관리

### Thinking 로그 읽기

Claude가 생각하는 과정을 보여주는 thinking 로그를 무시하지 않는다. 잘못된 가정을 하고 있다면 그 순간 ESC로 중단한다. 잘못된 가정 위에 쌓은 코드는 전부 쓸모가 없으므로 초반에 잡는 것이 핵심이다.

### 다른 AI에게 비평 받기

Claude와 작업하다 막히면 대화를 내보내서 다른 AI에게 분석을 요청한다.

> "이 대화를 분석해서 Claude가 놓치고 있는 것이나 잘못된 접근 있으면 지적해줘"

### Mermaid 아키텍처 정리

CLAUDE.md에 Mermaid 다이어그램으로 아키텍처를 정리하면 Claude가 프로젝트 구조를 한눈에 파악한다.
별도 파일(`docs/architecture.md`)에 정리하고 CLAUDE.md에서 `@docs/architecture.md`로 참조하는 것을 권장한다.
