# Claude Code 워크플로우 가이드

---

## 핵심 원칙

| 원칙 | 내용 |
|------|------|
| 승인 게이트 | 계획 승인 전 코드 생성 절대 금지 |
| 주도권 | 아키텍처·기술 선택은 항상 개발자가 결정 |
| 기획/코딩 분리 | Plan 완전 확정 후에만 구현 단계 진입 |

---

## 워크플로우

```
Research → Plan → Review(반복) → Implement → Feedback
```

---

### 1단계. Research

목적: 기존 코드베이스 구조와 의존성 완전 파악

출력: `research.md` (구두 요약 불가, 반드시 파일로 저장)

프롬프트 패턴:
```
이 폴더를 깊이 읽고, 어떻게 동작하는지 매우 상세히 모든 세부사항을 파악해라.
끝나면 research.md에 상세 보고서를 작성해라.
```

필수 파악 항목:
- 기존 레이어 구조 및 관례 (ORM, API 패턴 등)
- 중복될 수 있는 기능·엔드포인트
- 건드리면 안 되는 영역

> 가장 비싼 실패: 기능은 동작하지만 주변 시스템을 파괴하는 코드

---

### 2단계. Plan

목적: 구현 전 전체 접근 방식 확정

출력: `plan.md`

프롬프트 패턴:
```
research.md를 기반으로 [기능명] 구현을 위한 상세 plan.md를 작성해라.
코드 스니펫, 파일 경로, 트레이드오프를 포함해라.
```

plan.md 포함 항목:
- 접근 방식 상세 설명
- 실제 코드 스니펫 (가정 기반 X, 실제 코드베이스 기반)
- 영향받는 파일 경로
- 트레이드오프

---

### 3단계. Review (반복)

목적: 개발자가 plan.md를 직접 검토·수정하여 방향 확정

반복 사이클:
```
plan.md 검토 → 주석/메모 추가 → Claude에 전달 → plan.md 업데이트 → 재검토
```

개발자 결정 사항:
- 채택할 접근 방식 선택
- 제거할 불필요한 코드/기능 명시
- 건드리면 안 되는 영역 명시 ("이 파일/함수는 수정 금지")
- 기술 스택 최종 선택

> 승인 전 구현 착수 금지 — 이 단계가 완전히 끝나야 4단계 진입

---

### 4단계. Implement

목적: 확정된 plan.md 기반으로 실제 구현

프롬프트 패턴:
```
plan.md를 기반으로 전부 구현해라.
각 작업 완료 시 plan.md에 완료 표시해라.
모든 작업이 완료될 때까지 멈추지 마라.
매 단계마다 typecheck를 실행해서 새로운 오류를 만들지 마라.
```

구현 중 규칙:
- plan.md 외 범위 임의 확장 금지
- 기존 패턴·관례 준수 (ORM, 레이어, 네이밍)
- 완료 체크리스트를 plan.md에 실시간 반영

---

### 5단계. Feedback

목적: 결과 검토 및 다음 사이클 진입 여부 판단

- 예상과 다른 동작 → 새 Research 또는 Plan 사이클로 복귀
- 추가 기능 → 1단계부터 재시작 (기존 plan.md에 덧붙이기 금지)

---

## 프로젝트 유형별 시작점

### 기존 프로젝트
```
Research → 레이어별 코드 템플릿 파악 → Skill/Agent 정의 → Plan → Implement
```

### 신규 프로젝트
```
Skill/Agent 정의 → Design Doc (Plan Mode) → Design Doc 승인
→ Phase 분리 → Phase별 Plan Mode → Phase별 Implement
```

> Design Doc 완성 전 구현 착수 금지

## 문서 관리 규칙

### 문서 역할 분리

| 파일 | 역할 | 수정 주체 |
|------|------|-----------|
| `research.md` | 코드베이스 분석 결과 (읽기 전용) | Claude (생성), 개발자 (열람) |
| `plan.md` | 구현 계획 + 진행 체크리스트 | 개발자 + Claude |

### 문서-코드 동기화 문제 방지 규칙

문서는 코드를 복제하지 않는다
- ❌ 함수 전체 코드를 문서에 붙여넣기
- ✅ 파일 경로 + 함수명 + 변경 의도만 기록

문서는 다른 문서를 참조하지 않는다
- ❌ `plan.md`에서 `research.md`의 특정 섹션 인용
- ✅ 각 문서는 독립적으로 완결

plan.md는 구현 완료 후 아카이브한다
- 구현 완료 → `plan.md`를 `plan-YYYYMMDD.md`로 보관
- 다음 기능은 새 `plan.md`로 시작 (기존 파일에 추가 금지)

문서는 의도를 기록하고, 코드는 동작을 기록한다
- 문서가 outdated되어도 코드 동작에 영향 없는 구조 유지
- 문서 내 코드 스니펫은 예시용이며 소스 오브 트루스가 아님을 명시
