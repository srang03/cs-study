## 1. 설명

### 그래프란

**정점(Vertex)**과 **간선(Edge)**의 집합으로 이루어진 자료구조. 트리는 그래프의 특수한 형태이다.

### 용어 정리

| 용어 | 설명 |
|------|------|
| 정점(Vertex, Node) | 그래프의 각 원소 |
| 간선(Edge) | 정점을 연결하는 선 |
| 인접(Adjacent) | 간선으로 직접 연결된 관계 |
| 차수(Degree) | 한 정점에 연결된 간선 수 |
| 경로(Path) | 정점 A에서 B까지 간선을 따라 이동하는 순서 |
| 사이클(Cycle) | 시작과 끝이 같은 경로 |

### 그래프의 종류

| 종류 | 설명 |
|------|------|
| 무방향 그래프 | 간선에 방향 없음. A-B이면 양방향 |
| 방향 그래프(Digraph) | 간선에 방향 있음. A→B ≠ B→A |
| 가중치 그래프 | 간선에 비용(가중치)이 있음 |
| 연결 그래프 | 모든 정점이 경로로 연결됨 |

```
무방향:         방향:          가중치:
1 - 2          1 → 2         1 -3- 2
|   |          ↑   ↓         |     |
3 - 4          3 ← 4         5     2
                              |     |
                              3 -1- 4
```

### 인접 리스트 vs 인접 행렬

#### 인접 행렬 (Adjacency Matrix)

2차원 배열. `adj[i][j] = 1`이면 i에서 j로 간선 존재.

```java
int[][] adj = new int[n + 1][n + 1];

// 무방향 간선 추가
adj[1][2] = 1;
adj[2][1] = 1;
```

#### 인접 리스트 (Adjacency List)

각 정점별로 연결된 정점의 리스트를 저장.

```java
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i <= n; i++) {
    adj.add(new ArrayList<>());
}

// 무방향 간선 추가
adj.get(1).add(2);
adj.get(2).add(1);
```

#### 비교

| 기준 | 인접 행렬 | 인접 리스트 |
|------|----------|------------|
| 메모리 | O(V²) | O(V + E) |
| 간선 존재 확인 | O(1) | O(차수) |
| 인접 정점 순회 | O(V) | O(차수) |
| 적합한 경우 | 간선이 많은 밀집 그래프 | 간선이 적은 희소 그래프 |

> 코테에서는 **인접 리스트를 기본**으로 사용한다. 대부분 희소 그래프이고, 메모리 효율이 좋다.

### 입력 처리 패턴

코테에서 그래프 입력은 거의 항상 같은 패턴이다:

```
첫 줄: 정점 수 N, 간선 수 M
이후 M줄: 간선 정보 (u v) 또는 (u v w)
```

```java
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
StringTokenizer st = new StringTokenizer(br.readLine());
int n = Integer.parseInt(st.nextToken());
int m = Integer.parseInt(st.nextToken());

// 인접 리스트 초기화
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i <= n; i++) {
    adj.add(new ArrayList<>());
}

// 간선 입력
for (int i = 0; i < m; i++) {
    st = new StringTokenizer(br.readLine());
    int u = Integer.parseInt(st.nextToken());
    int v = Integer.parseInt(st.nextToken());
    adj.get(u).add(v);
    adj.get(v).add(u); // 무방향이면 양쪽 추가
}
```

### 가중치 그래프 입력

```java
// 가중치 저장용 클래스 또는 int[]
List<List<int[]>> adj = new ArrayList<>();
for (int i = 0; i <= n; i++) {
    adj.add(new ArrayList<>());
}

// 간선: u → v, 가중치 w
adj.get(u).add(new int[]{v, w});
adj.get(v).add(new int[]{u, w});
```

## 2. 예제

### 예제 1: 인접 리스트로 그래프 생성 후 출력

```java
// 그래프:
// 1 - 2
// 1 - 3
// 2 - 4
// 3 - 4

int n = 4, m = 4;
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i <= n; i++) adj.add(new ArrayList<>());

int[][] edges = {{1,2},{1,3},{2,4},{3,4}};
for (int[] e : edges) {
    adj.get(e[0]).add(e[1]);
    adj.get(e[1]).add(e[0]);
}

// 출력: 각 정점의 인접 리스트
for (int i = 1; i <= n; i++) {
    System.out.println(i + " → " + adj.get(i));
}
// 1 → [2, 3]
// 2 → [1, 4]
// 3 → [1, 4]
// 4 → [2, 3]
```

### 예제 2: 인접 행렬로 같은 그래프

```java
int[][] matrix = new int[5][5];
int[][] edges = {{1,2},{1,3},{2,4},{3,4}};
for (int[] e : edges) {
    matrix[e[0]][e[1]] = 1;
    matrix[e[1]][e[0]] = 1;
}
// matrix[1][2] = 1, matrix[2][4] = 1, ...
```

## 4. 이론 퀴즈

**Q1.** 정점 5개, 간선 4개인 무방향 그래프에서 인접 행렬의 크기와 인접 리스트의 총 원소 수는?

<details>
<summary>정답</summary>
인접 행렬: 5x5 = 25칸. 인접 리스트: 무방향이므로 간선당 2개씩 → 4 x 2 = 8개 원소.
</details>

**Q2.** 트리와 그래프의 차이는?

<details>
<summary>정답</summary>
트리는 사이클이 없고, 루트가 있으며, N개 노드에 N-1개 간선을 가진 연결 그래프이다. 그래프는 사이클이 있을 수 있고, 루트 개념이 없으며, 간선 수에 제한이 없다.
</details>

**Q3.** N=10000인 그래프에서 인접 행렬을 사용하면 메모리가 얼마나 필요한가?

<details>
<summary>정답</summary>
10000 x 10000 = 10^8 칸. int 배열이면 약 400MB로, 보통 256MB 메모리 제한을 초과한다. 인접 리스트를 써야 한다.
</details>

## 5. 함정 노트

### 1. 무방향 그래프에서 간선 한쪽만 추가

가장 흔한 실수. 무방향이면 **양쪽 모두** 추가해야 한다.

```java
// 실수: 한쪽만
adj.get(u).add(v);

// 정답: 양쪽
adj.get(u).add(v);
adj.get(v).add(u);
```

### 2. 1-indexed vs 0-indexed

문제에서 정점 번호가 1부터 시작하면 배열/리스트 크기를 `n + 1`로 잡아야 한다.

```java
// n+1로 초기화
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i <= n; i++) { // 0 ~ n
    adj.add(new ArrayList<>());
}
```

### 3. 자기 자신으로의 간선 (self-loop)

일부 문제에서 u = v인 간선이 주어질 수 있다. 문제 조건을 반드시 확인.
