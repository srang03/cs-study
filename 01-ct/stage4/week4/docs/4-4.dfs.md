## 1. 설명

### DFS란

**Depth-First Search(깊이 우선 탐색)**. 한 경로를 끝까지 탐색한 뒤 돌아와서 다른 경로를 탐색하는 방법.

```
시작 → 갈 수 있는 곳까지 깊이 진행 → 막히면 되돌아감(백트래킹)
```

### 핵심 아이디어

1. 현재 정점 방문 처리
2. 인접 정점 중 방문하지 않은 곳으로 이동
3. 더 이상 갈 곳이 없으면 되돌아감
4. 모든 정점을 방문할 때까지 반복

### 방문 배열 (visited)

**이미 방문한 정점을 다시 방문하지 않기 위한 장치.** 이것이 없으면 무한 루프에 빠진다.

```java
boolean[] visited = new boolean[n + 1];
```

### 재귀 구현

```java
static List<List<Integer>> adj;
static boolean[] visited;

static void dfs(int node) {
    visited[node] = true;
    System.out.print(node + " ");

    for (int next : adj.get(node)) {
        if (!visited[next]) {
            dfs(next);
        }
    }
}
```

### 스택 구현

재귀 대신 **명시적 스택**을 사용할 수 있다. 재귀 깊이가 깊어 스택오버플로우가 우려될 때 유용.

```java
static void dfsStack(int start) {
    Stack<Integer> stack = new Stack<>();
    stack.push(start);

    while (!stack.isEmpty()) {
        int node = stack.pop();
        if (visited[node]) continue;
        visited[node] = true;
        System.out.print(node + " ");

        // 인접 정점을 역순으로 넣어야 작은 번호부터 방문
        List<Integer> neighbors = adj.get(node);
        for (int i = neighbors.size() - 1; i >= 0; i--) {
            int next = neighbors.get(i);
            if (!visited[next]) {
                stack.push(next);
            }
        }
    }
}
```

### DFS의 시간 복잡도

- 인접 리스트: **O(V + E)** — 모든 정점과 간선을 한 번씩 확인
- 인접 행렬: **O(V²)** — 각 정점에서 모든 정점을 확인

### 연결 요소 (Connected Component) 찾기

그래프가 여러 조각으로 분리되어 있을 때, 각 조각이 하나의 연결 요소이다.

```java
int count = 0;
for (int i = 1; i <= n; i++) {
    if (!visited[i]) {
        dfs(i);
        count++;  // 새로운 연결 요소 발견
    }
}
System.out.println("연결 요소 수: " + count);
```

## 2. 예제

### 예제 1: DFS 탐색 순서

```
그래프:
1 - 2
1 - 3
2 - 4
2 - 5
3 - 6

DFS(1) → 방문: 1
  DFS(2) → 방문: 2
    DFS(4) → 방문: 4 (막힘, 돌아감)
    DFS(5) → 방문: 5 (막힘, 돌아감)
  DFS(3) → 방문: 3
    DFS(6) → 방문: 6 (막힘, 돌아감)

결과: 1 2 4 5 3 6
```

### 예제 2: 연결 요소 개수

```
그래프 (정점 6, 간선 3):
1 - 2
3 - 4
5 - 6

연결 요소: {1,2}, {3,4}, {5,6} → 3개
```

```java
// DFS를 1부터 순회
// i=1: visited[1]=false → dfs(1) → 1,2 방문 → count=1
// i=2: visited[2]=true → skip
// i=3: visited[3]=false → dfs(3) → 3,4 방문 → count=2
// i=4: visited[4]=true → skip
// i=5: visited[5]=false → dfs(5) → 5,6 방문 → count=3
```

## 4. 이론 퀴즈

**Q1.** DFS에서 visited 배열이 없으면 어떤 문제가 발생하는가?

<details>
<summary>정답</summary>
이미 방문한 정점을 다시 방문하여 무한 루프에 빠진다. 특히 사이클이 있는 그래프에서 발생.
</details>

**Q2.** DFS의 시간 복잡도가 O(V + E)인 이유는?

<details>
<summary>정답</summary>
각 정점은 visited 체크로 한 번만 방문(V), 각 간선도 한 번만 확인(E)하기 때문이다.
</details>

**Q3.** 재귀 DFS와 스택 DFS의 방문 순서가 다를 수 있는 이유는?

<details>
<summary>정답</summary>
스택은 LIFO이므로 인접 정점의 추가 순서에 따라 탐색 순서가 달라질 수 있다. 같은 결과를 얻으려면 인접 정점을 역순으로 push해야 한다.
</details>

## 5. 함정 노트

### 1. visited 체크 위치

```java
// 방법 1: 방문 시 체크 (재귀)
void dfs(int node) {
    visited[node] = true;  // 여기서 체크
    for (int next : adj.get(node)) {
        if (!visited[next]) dfs(next);
    }
}

// 방법 2: push 시 체크 (스택) — 중복 push 방지
// 또는 pop 시 체크 — 더 직관적이지만 중복 push 발생 가능
```

> 재귀는 진입 시 체크, 스택은 pop 시 체크가 일반적.

### 2. 재귀 깊이 제한 (StackOverflowError)

Java의 기본 스택 크기는 약 512KB~1MB. 정점이 수만 개이고 일렬로 연결되면 재귀 깊이가 한계를 넘을 수 있다.

해결: 명시적 스택 사용 또는 `-Xss` JVM 옵션으로 스택 크기 확대.

### 3. 방향 그래프에서의 연결 요소

무방향 그래프에서는 DFS로 연결 요소를 바로 구하지만, 방향 그래프에서는 **강연결 요소(SCC)**라는 별도 개념이 필요하다. 문제 조건을 확인할 것.
