## 1. 설명

### 트리란

**계층 구조**를 표현하는 비선형 자료구조. 하나의 루트에서 시작하여 자식 노드들로 뻗어나간다.

### 용어 정리

| 용어 | 설명 |
|------|------|
| 노드(Node) | 트리의 각 원소 |
| 엣지(Edge) | 노드와 노드를 연결하는 간선 |
| 루트(Root) | 최상위 노드 (부모 없음) |
| 리프(Leaf) | 자식이 없는 말단 노드 |
| 부모(Parent) | 바로 위 노드 |
| 자식(Child) | 바로 아래 노드 |
| 깊이(Depth) | 루트에서 해당 노드까지의 간선 수 |
| 높이(Height) | 해당 노드에서 가장 깊은 리프까지의 간선 수 |

```
        1          ← 루트 (깊이 0)
       / \
      2   3        ← 깊이 1
     / \   \
    4   5   6      ← 리프 (깊이 2)
```

### 이진트리 (Binary Tree)

각 노드가 **최대 2개의 자식**을 가지는 트리.

| 종류 | 설명 |
|------|------|
| 완전이진트리 | 마지막 레벨을 제외하고 모두 채워지고, 마지막 레벨은 왼쪽부터 채움 |
| 포화이진트리 | 모든 레벨이 꽉 찬 이진트리. 노드 수 = 2^h+1 - 1 |
| 편향이진트리 | 한쪽으로만 치우친 트리. 사실상 연결 리스트 |

### 트리 순회 (Traversal)

모든 노드를 한 번씩 방문하는 방법. **"루트를 언제 방문하느냐"** 로 구분한다.

```
        1
       / \
      2   3
     / \
    4   5
```

| 순회 | 순서 | 결과 |
|------|------|------|
| 전위(Preorder) | **루트** → 왼쪽 → 오른쪽 | 1 2 4 5 3 |
| 중위(Inorder) | 왼쪽 → **루트** → 오른쪽 | 4 2 5 1 3 |
| 후위(Postorder) | 왼쪽 → 오른쪽 → **루트** | 4 5 2 3 1 |
| 레벨순회(Level-order) | 위 → 아래, 왼 → 오 | 1 2 3 4 5 |

### 순회 구현 (재귀)

```java
class Node {
    int value;
    Node left, right;
    Node(int v) { this.value = v; }
}

// 전위 순회
void preorder(Node node) {
    if (node == null) return;
    System.out.print(node.value + " ");  // 루트 먼저
    preorder(node.left);
    preorder(node.right);
}

// 중위 순회
void inorder(Node node) {
    if (node == null) return;
    inorder(node.left);
    System.out.print(node.value + " ");  // 루트 중간
    inorder(node.right);
}

// 후위 순회
void postorder(Node node) {
    if (node == null) return;
    postorder(node.left);
    postorder(node.right);
    System.out.print(node.value + " ");  // 루트 마지막
}
```

### 레벨 순회 (BFS 활용)

```java
void levelOrder(Node root) {
    Queue<Node> queue = new LinkedList<>();
    queue.add(root);
    while (!queue.isEmpty()) {
        Node cur = queue.poll();
        System.out.print(cur.value + " ");
        if (cur.left != null) queue.add(cur.left);
        if (cur.right != null) queue.add(cur.right);
    }
}
```

// 배열: [_, A, B, C, D, E]  (인덱스 0은 비움)

## 5. 함정 노트

### 1. null 체크 누락

재귀 순회에서 `if (node == null) return;`을 빠뜨리면 NullPointerException 발생.

### 2. 배열 인덱스 혼동

배열로 트리를 표현할 때 인덱스 0부터 시작하면 수식이 달라진다:
- 0-indexed: 왼쪽 자식 = 2i + 1, 오른쪽 자식 = 2i + 2, 부모 = (i - 1) / 2
- 1-indexed: 왼쪽 자식 = 2i, 오른쪽 자식 = 2i + 1, 부모 = i / 2

> 코테에서는 **1-indexed**가 직관적이므로 권장.

### 3. 순회 순서 암기 혼동

"전중후"는 **루트 방문 시점**이다:
- 전위 = 루트가 **전**(먼저)
- 중위 = 루트가 **중**(가운데)
- 후위 = 루트가 **후**(마지막)
