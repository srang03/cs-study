## 1. 설명

### 힙이란

**완전이진트리** 기반의 자료구조로, 부모와 자식 사이에 **대소 관계**가 유지된다.

| 종류 | 조건 | 루트 |
|------|------|------|
| 최대힙(Max Heap) | 부모 >= 자식 | 최대값 |
| 최소힙(Min Heap) | 부모 <= 자식 | 최소값 |

```
최소힙 예시:         최대힙 예시:
      1                    9
     / \                  / \
    3   5                7   8
   / \                  / \
  7   4                3   5
```

### 배열로 표현

완전이진트리이므로 배열로 효율적으로 표현 가능 (인덱스 1부터 시작):

```
부모:      i / 2
왼쪽 자식: 2 * i
오른쪽 자식: 2 * i + 1
```

```
최소힙 [_, 1, 3, 5, 7, 4]
인덱스:  0  1  2  3  4  5

부모(4) = 4/2 = 2 → 값 3 (3 <= 7 ✓)
부모(5) = 5/2 = 2 → 값 3 (3 <= 4 ✓)
```

### 삽입과 삭제

**삽입 (O(log n))**:
1. 배열 끝에 추가 (완전이진트리 유지)
2. 부모와 비교하며 위로 이동 (**heapify up / sift up**)

**삭제 (O(log n))**:
1. 루트를 제거 (최대값 또는 최소값)
2. 마지막 원소를 루트로 이동
3. 자식과 비교하며 아래로 이동 (**heapify down / sift down**)

```java
// 최소힙 삽입 과정 예시: 2 삽입
// [_, 1, 3, 5, 7, 4] → 끝에 추가 → [_, 1, 3, 5, 7, 4, 2]
// 2와 부모(5) 비교 → 교환 → [_, 1, 3, 2, 7, 4, 5]
// 2와 부모(1) 비교 → 1 <= 2이므로 종료
```

### Java PriorityQueue

Java에서 힙은 `PriorityQueue`로 사용한다. **기본은 최소힙**이다.

```java
import java.util.PriorityQueue;
import java.util.Collections;

// 최소힙 (기본)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.add(5);
minHeap.add(1);
minHeap.add(3);
System.out.println(minHeap.poll()); // 1 (최소값)
System.out.println(minHeap.peek()); // 3 (다음 최소값, 제거 안 함)

// 최대힙
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
maxHeap.add(5);
maxHeap.add(1);
maxHeap.add(3);
System.out.println(maxHeap.poll()); // 5 (최대값)
```

### 주요 메서드와 시간 복잡도

| 메서드 | 동작 | 시간 복잡도 |
|--------|------|------------|
| `add(e)` / `offer(e)` | 삽입 | O(log n) |
| `poll()` | 최소(최대) 제거 후 반환 | O(log n) |
| `peek()` | 최소(최대) 조회 (제거 안 함) | O(1) |
| `size()` | 크기 | O(1) |
| `isEmpty()` | 비어있는지 | O(1) |

## 2. 예제

### 예제 1: 최소힙 동작 추적

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.add(4);   // [4]
pq.add(2);   // [2, 4]
pq.add(7);   // [2, 4, 7]
pq.add(1);   // [1, 2, 7, 4]
pq.poll();   // 1 반환 → [2, 4, 7]
pq.add(3);   // [2, 3, 7, 4]
pq.poll();   // 2 반환 → [3, 4, 7]
```

### 예제 2: K번째 최소값 찾기

```java
// 배열에서 K번째로 작은 수
int kthSmallest(int[] arr, int k) {
    PriorityQueue<Integer> pq = new PriorityQueue<>();
    for (int num : arr) {
        pq.add(num);
    }
    for (int i = 0; i < k - 1; i++) {
        pq.poll();
    }
    return pq.poll();
}
```

## 4. 이론 퀴즈

**Q1.** PriorityQueue의 기본 정렬 순서는?

<details>
<summary>정답</summary>
최소힙. 가장 작은 값이 먼저 나온다. 최대힙으로 바꾸려면 Collections.reverseOrder()를 생성자에 전달.
</details>

**Q2.** 힙에서 삽입과 삭제의 시간 복잡도가 O(log n)인 이유는?

<details>
<summary>정답</summary>
완전이진트리의 높이가 log n이므로, heapify up/down 과정에서 최대 log n번 비교·교환한다.
</details>

**Q3.** 힙은 정렬된 상태인가?

<details>
<summary>정답</summary>
아니다. 힙은 부모-자식 간 대소 관계만 보장한다. 같은 레벨의 형제 노드 간에는 순서가 없다. 전체 정렬이 아니라 "루트가 최소(최대)"만 보장.
</details>

## 5. 함정 노트

### 1. poll()과 peek() 혼동

- `poll()`: 꺼내고 **제거**
- `peek()`: 꺼내되 **제거 안 함**

빈 큐에서 `poll()`은 null 반환, `peek()`도 null 반환. int로 받으면 NullPointerException!

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
int val = pq.poll(); // NPE! null을 int로 언박싱
```

### 2. 최대힙 생성법 실수

```java
// 틀린 방법: 음수로 넣기 (값을 복원해야 해서 실수 발생)
pq.add(-num);
int result = -pq.poll();

// 권장 방법: Collections.reverseOrder()
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
```

### 3. PriorityQueue는 인덱스 접근 불가

`pq.get(i)` 같은 인덱스 접근은 없다. 순서대로 꺼내려면 `poll()`을 반복해야 한다.

## 7. 보스 문제

### [백준 11286 - 절댓값 힙](https://www.acmicpc.net/problem/11286)

절대값이 작은 것이 먼저 나오고, 절대값이 같으면 실제 값이 작은 것이 먼저 나오는 힙.

- **힌트**: PriorityQueue에 Comparator를 직접 지정

```java
PriorityQueue<Integer> pq = new PriorityQueue<>((a, b) -> {
    int absA = Math.abs(a), absB = Math.abs(b);
    if (absA != absB) return absA - absB;
    return a - b;
});
```
