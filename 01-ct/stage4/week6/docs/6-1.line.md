## 1. 설명

### LINE 인턴 코테 개요

- **형식**: 온라인, 보통 2~3시간, 4~5문제
- **난이도**: 쉬운 구현 1~2문제 + 중간 알고리즘 1~2문제 + 어려운 문제 1문제
- **언어**: Java, Python, C++ 등 자유 선택

### 자주 출제되는 유형

| 유형 | 빈도 | 핵심 역량 |
|------|------|----------|
| 문자열 처리 | 높음 | String 메서드, 파싱, 정규표현식 기초 |
| 구현/시뮬레이션 | 높음 | 조건 분기, 상태 관리, 꼼꼼한 구현 |
| 자료구조 활용 | 중간 | HashMap, Set, Queue, Stack |
| 그래프/탐색 | 중간 | BFS/DFS, 최단 경로 |
| DP | 낮음~중간 | 점화식 도출, 바텀업 |

### 문제 풀이 전략

1. **쉬운 문제부터**: 1~2번은 구현 문제일 확률이 높다. 빠르게 풀어 점수 확보
2. **문자열 메서드 숙달**: `split`, `substring`, `charAt`, `indexOf`, `replace` 등
3. **엣지 케이스 체크**: 빈 문자열, 길이 1, 최대값 등

### 구현 문제 접근법

```
1. 문제를 꼼꼼히 읽는다 (조건 하나하나 체크리스트로)
2. 예제 입출력으로 로직을 검증한다
3. 코드를 작성한다
4. 엣지 케이스를 테스트한다
```

## 2. 예제

### 예제 1: 문자열 압축 (구현)

연속된 같은 문자를 "문자+횟수"로 압축하라. 1개면 숫자 생략.

```java
public class Main {
    public static String compress(String s) {
        StringBuilder sb = new StringBuilder();
        int i = 0;
        while (i < s.length()) {
            char c = s.charAt(i);
            int count = 0;
            while (i < s.length() && s.charAt(i) == c) {
                count++;
                i++;
            }
            sb.append(c);
            if (count > 1) sb.append(count);
        }
        return sb.toString();
    }

    public static void main(String[] args) {
        System.out.println(compress("aaabbc"));    // a3b2c
        System.out.println(compress("abcd"));      // abcd
        System.out.println(compress("aaaaaa"));    // a6
    }
}
```

### 예제 2: 시뮬레이션 — 명령어 처리

커서 위치에서 문자열을 편집하는 시뮬레이션.

```java
import java.util.*;

public class Main {
    public static String simulate(String init, String[] commands) {
        LinkedList<Character> list = new LinkedList<>();
        for (char c : init.toCharArray()) list.add(c);
        ListIterator<Character> cursor = list.listIterator(list.size());

        for (String cmd : commands) {
            switch (cmd.charAt(0)) {
                case 'L': if (cursor.hasPrevious()) cursor.previous(); break;
                case 'R': if (cursor.hasNext()) cursor.next(); break;
                case 'D': if (cursor.hasNext()) { cursor.next(); cursor.remove(); } break;
                default:  cursor.add(cmd.charAt(1)); break;  // P x
            }
        }

        StringBuilder sb = new StringBuilder();
        for (char c : list) sb.append(c);
        return sb.toString();
    }
}
```

### 예제 3: HashMap 빈도수 패턴

가장 많이 등장하는 문자를 찾아라.

```java
public static char mostFrequent(String s) {
    Map<Character, Integer> map = new HashMap<>();
    for (char c : s.toCharArray()) {
        map.put(c, map.getOrDefault(c, 0) + 1);
    }

    char result = ' ';
    int maxCount = 0;
    for (Map.Entry<Character, Integer> entry : map.entrySet()) {
        if (entry.getValue() > maxCount) {
            maxCount = entry.getValue();
            result = entry.getKey();
        }
    }
    return result;
}
```

## 5. 함정 노트

### 1. 문자열 비교

```java
// 틀림: == 는 참조 비교
if (str == "hello") { }

// 맞음: equals()로 내용 비교
if (str.equals("hello")) { }
```

### 2. split의 정규식 특수문자

```java
// 틀림: .은 정규식에서 "아무 문자"
"a.b.c".split(".")    // 빈 배열!

// 맞음: 이스케이프
"a.b.c".split("\\.")  // ["a", "b", "c"]
```

### 3. Integer 범위 초과

큰 수의 합계나 곱을 구할 때 `int` 대신 `long`을 사용해야 하는지 항상 확인.

### 4. 시간 복잡도 확인 안 하고 구현부터 시작

구현이 복잡한 문제일수록 먼저 시간 복잡도를 계산하고, 제한 내에서 가능한지 확인한 후 코딩을 시작하라.
