## 1. 설명

### 카카오 코테 개요

- **형식**: 온라인, 보통 5시간, 7문제
- **난이도**: 점진적 상승 (1~2번 쉬움, 3~5번 중간, 6~7번 어려움)
- **특징**: 문제 지문이 길고 상황 설명이 복잡. 독해력이 중요

### 자주 출제되는 유형

| 유형 | 빈도 | 설명 |
|------|------|------|
| 문자열 처리 | 매우 높음 | 파싱, 변환, 정규표현식 |
| 구현 | 매우 높음 | 복잡한 조건, 시뮬레이션 |
| 해시(Map) | 높음 | 빈도수, 매핑, 캐싱 |
| 그래프 탐색 | 높음 | BFS/DFS, 최단 경로 |
| 이진 탐색 | 중간 | 매개변수 탐색 (Parametric Search) |
| DP | 중간 | 점화식 도출 |

### 카카오 문제의 특징

1. **지문이 길다**: 핵심 조건을 정리하는 연습 필요
2. **부분 점수**: 효율성 테스트와 정확성 테스트 분리
3. **문자열 파싱이 핵심**: JSON-like 구조, 특수 포맷 파싱

### 접근 전략

```
1. 문제 읽기 (5분): 핵심 조건을 메모
2. 예제 확인 (3분): 입출력으로 이해 검증
3. 알고리즘 선택 (2분): 제약 조건 → 시간 복잡도 → 방법

5. 테스트 (5분): 예제 + 엣지 케이스
```

## 2. 예제

### 예제 1: 해시 — 전화번호 접두어 판별

전화번호 목록에서 어떤 번호가 다른 번호의 접두어인지 확인.

```java
import java.util.*;

public class Main {
    public static boolean solution(String[] phoneBook) {
        Set<String> set = new HashSet<>(Arrays.asList(phoneBook));

        for (String phone : phoneBook) {
            // 자기 자신을 제외한 모든 접두어를 체크
            for (int i = 1; i < phone.length(); i++) {
                if (set.contains(phone.substring(0, i))) {
                    return false;
                }
            }
        }
        return true;
    }
}
```

### 예제 2: 문자열 파싱 — 괄호 매칭

중첩 괄호가 있는 문자열에서 가장 깊은 레벨을 구하라.

```java
public static int maxDepth(String s) {
    int depth = 0;
    int maxDepth = 0;
    for (char c : s.toCharArray()) {
        if (c == '(') {
            depth++;
            maxDepth = Math.max(maxDepth, depth);
        } else if (c == ')') {
            depth--;
        }
    }
    return maxDepth;
}
```

### 예제 3: BFS — 최단 거리

```java
import java.util.*;

public class Main {
    static int[] dx = {-1, 1, 0, 0};
    static int[] dy = {0, 0, -1, 1};

    public static int shortestPath(int[][] grid) {
        int n = grid.length, m = grid[0].length;
        int[][] dist = new int[n][m];
        for (int[] row : dist) Arrays.fill(row, -1);

        Queue<int[]> queue = new LinkedList<>();
        queue.offer(new int[]{0, 0});
        dist[0][0] = 1;

        while (!queue.isEmpty()) {
            int[] cur = queue.poll();
            for (int d = 0; d < 4; d++) {
                int nx = cur[0] + dx[d];
                int ny = cur[1] + dy[d];
                if (nx >= 0 && nx < n && ny >= 0 && ny < m
                    && grid[nx][ny] == 1 && dist[nx][ny] == -1) {
                    dist[nx][ny] = dist[cur[0]][cur[1]] + 1;
                    queue.offer(new int[]{nx, ny});
                }
            }
        }
        return dist[n - 1][m - 1];
    }
}
```

## 3. 직접 작성

### 과제 1: 문자열 변환

주어진 문자열에서 연속된 같은 문자를 제거하는 함수를 작성하라. 제거 후 다시 연속된 문자가 생기면 반복한다.

```
"aabbcc" → "" (aa 제거 → bbcc → cc 제거 → bb → 제거 → "")
"abcddcba" → "" (dd 제거 → abccba → cc 제거 → abba → bb 제거 → aa → 제거)
```

### 과제 2: 해시 활용

두 문자열 배열이 주어질 때, 첫 번째 배열에만 있고 두 번째 배열에는 없는 문자열을 찾아 반환하라. HashMap의 `containsKey`를 활용.

## 4. 이론 퀴즈

**Q1.** 카카오 문제에서 효율성 테스트를 통과하지 못했다면 어떤 순서로 접근하는가?

<details>
<summary>정답</summary>
1) 현재 알고리즘의 시간 복잡도 계산 → 2) 제약 조건에서 요구하는 복잡도 파악 → 3) 더 효율적인 자료구조/알고리즘으로 교체 (예: O(n^2) → HashMap O(n), 이진 탐색 O(n log n)).
</details>

**Q2.** 매개변수 탐색(Parametric Search)이란?

<details>
<summary>정답</summary>
"최적값을 구하라"를 "이 값이 가능한가?"로 바꿔 이진 탐색하는 기법. 가능/불가능의 경계를 이진 탐색으로 찾는다.
</details>

**Q3.** HashSet과 HashMap의 시간 복잡도 차이는?

<details>
<summary>정답</summary>
차이 없음. 둘 다 삽입/조회/삭제 평균 O(1). HashSet은 내부적으로 HashMap을 사용한다.
</details>

## 5. 함정 노트

### 1. 지문 오독

카카오 문제는 조건이 많다. "단, ..."으로 시작하는 예외 조건을 놓치면 정확성 테스트에서 탈락.

### 2. 부분 점수 전략

효율성을 못 풀겠으면 정확성만이라도 확보. 브루트포스로 정확성 점수를 먼저 얻고, 남은 시간에 최적화.

### 3. 정렬 후 이진 탐색 vs HashMap

```java
// 정렬 + 이진 탐색: O(n log n)
Arrays.sort(arr);
int idx = Arrays.binarySearch(arr, target);

// HashMap: O(n)
Map<Integer, Integer> map = new HashMap<>();
// 단, 메모리 사용이 더 많을 수 있음
```

문제 특성에 따라 선택. 범위 탐색이면 이진 탐색, 정확한 값 탐색이면 HashMap.

### 4. 카카오 프로그래머스 환경

- 클래스명이 `Solution`으로 고정
- 함수 시그니처가 주어짐
- `import`가 대부분 자동 포함
- 출력이 아닌 반환(`return`)으로 답을 제출
