## 1. 설명

### 삼성 SW 역량 테스트 개요

- **형식**: 온라인/오프라인, 3시간, 2문제
- **난이도**: Gold III ~ Gold I 수준
- **특징**: 문제 수가 적고 시간이 충분. 구현력과 정확성이 핵심

### 출제 유형 분석

| 유형 | 빈도 | 설명 |
|------|------|------|
| 구현/시뮬레이션 | 매우 높음 | 복잡한 규칙을 정확히 구현 |
| BFS/DFS | 높음 | 격자 탐색, 영역 나누기 |
| 브루트포스 + 백트래킹 | 높음 | 모든 경우 탐색 + 가지치기 |
| 조합/순열 | 중간 | 경우의 수 열거 |

### 삼성 문제의 특징

1. **2문제만 풀면 된다**: 시간 배분이 넉넉. 정확성에 집중
2. **구현이 무겁다**: 코드가 100~200줄 이상 되는 경우가 많음
3. **격자(2차원 배열)가 자주 등장**: 회전, 뒤집기, 이동 등
4. **여러 단계의 시뮬레이션**: 턴제 게임, 물리 시뮬레이션 등

### 접근 전략

```
1. 문제를 2~3번 정독한다 (15분)
2. 조건을 하나하나 체크리스트로 정리한다
3. 시뮬레이션 흐름을 의사코드로 작성한다
4. 함수를 분리하여 구현한다 (한 함수에 모든 로직 X)
5. 예제로 디버깅한다 (출력문 활용)
```

```java
// 90도 시계 방향 회전
int[][] rotate90(int[][] grid) {
    int n = grid.length, m = grid[0].length;
    int[][] result = new int[m][n];
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            result[j][n - 1 - i] = grid[i][j];
    return result;
}
```

    copy[i] = grid[i].clone();

## 2. 예제

### 예제 1: 격자 탐색 — 영역 개수 세기 (BFS)

```java
import java.util.*;

public class Main {
    static int[] dx = {-1, 1, 0, 0};
    static int[] dy = {0, 0, -1, 1};

    public static int countRegions(int[][] grid) {
        int n = grid.length, m = grid[0].length;
        boolean[][] visited = new boolean[n][m];
        int count = 0;

        for (int i = 0; i < n; i++) {
            for (int j = 0; j < m; j++) {
                if (grid[i][j] == 1 && !visited[i][j]) {
                    bfs(grid, visited, i, j, n, m);
                    count++;
                }
            }
        }
        return count;
    }

    static void bfs(int[][] grid, boolean[][] visited, int x, int y, int n, int m) {
        Queue<int[]> queue = new LinkedList<>();
        queue.offer(new int[]{x, y});
        visited[x][y] = true;

        while (!queue.isEmpty()) {
            int[] cur = queue.poll();
            for (int d = 0; d < 4; d++) {
                int nx = cur[0] + dx[d];
                int ny = cur[1] + dy[d];
                if (nx >= 0 && nx < n && ny >= 0 && ny < m
                    && grid[nx][ny] == 1 && !visited[nx][ny]) {
                    visited[nx][ny] = true;
                    queue.offer(new int[]{nx, ny});
                }
            }
        }
    }
}
```

### 예제 2: 브루트포스 + 백트래킹 — 순열 생성

```java
public class Main {
    static int n;
    static boolean[] used;
    static int[] result;

    public static void permutation(int depth) {
        if (depth == n) {
            // 순열 하나 완성 — 여기서 처리
            System.out.println(Arrays.toString(result));
            return;
        }
        for (int i = 1; i <= n; i++) {
            if (!used[i]) {
                used[i] = true;
                result[depth] = i;
                permutation(depth + 1);
                used[i] = false;  // 백트래킹
            }
        }
    }

    public static void main(String[] args) {
        n = 3;
        used = new boolean[n + 1];
        result = new int[n];
        permutation(0);
    }
}
```

### 예제 3: 시뮬레이션 — 배열 돌리기

```java
// 외곽 테두리를 시계 방향으로 한 칸씩 회전
public static void rotateEdge(int[][] grid, int n, int m) {
    int[] temp = new int[2 * (n + m) - 4];
    int idx = 0;

    // 테두리 원소를 순서대로 추출
    for (int j = 0; j < m; j++) temp[idx++] = grid[0][j];
    for (int i = 1; i < n; i++) temp[idx++] = grid[i][m - 1];
    for (int j = m - 2; j >= 0; j--) temp[idx++] = grid[n - 1][j];
    for (int i = n - 2; i >= 1; i--) temp[idx++] = grid[i][0];

    // 한 칸 시계 방향 이동 (마지막 원소를 맨 앞으로)
    int last = temp[idx - 1];
    for (int i = idx - 1; i > 0; i--) temp[i] = temp[i - 1];
    temp[0] = last;

    // 다시 격자에 채우기
    idx = 0;
    for (int j = 0; j < m; j++) grid[0][j] = temp[idx++];
    for (int i = 1; i < n; i++) grid[i][m - 1] = temp[idx++];
    for (int j = m - 2; j >= 0; j--) grid[n - 1][j] = temp[idx++];
    for (int i = n - 2; i >= 1; i--) grid[i][0] = temp[idx++];
}
```

## 4. 이론 퀴즈

**Q1.** 삼성 문제에서 함수를 분리하는 것이 왜 중요한가?

<details>
<summary>정답</summary>
코드가 100줄 이상이 되면 디버깅이 어렵다. 기능별로 함수를 분리하면 각 함수를 독립적으로 테스트할 수 있고, 버그 위치를 빠르게 찾을 수 있다.
</details>

**Q2.** 2차원 배열 깊은 복사에서 `grid.clone()`만 하면 안 되는 이유는?

<details>
<summary>정답</summary>
2차원 배열에서 clone()은 1차원 참조만 복사한다(얕은 복사). 내부 배열은 원본과 같은 참조를 공유하므로, 각 행을 개별적으로 clone()해야 한다.
</details>

**Q3.** 백트래킹에서 `used[i] = false`를 하는 이유는?

<details>
<summary>정답</summary>
현재 선택을 취소하고 다른 경우를 탐색하기 위해. 재귀 호출 후 상태를 복원(원복)하는 것이 백트래킹의 핵심.
</details>

## 5. 함정 노트

### 1. 배열 복사 안 하고 원본 수정

시뮬레이션에서 "이 턴의 상태"를 보고 "다음 턴의 상태"를 만들어야 할 때, 원본을 직접 수정하면 순서에 따라 결과가 달라진다.

```java
// 틀림: 원본을 직접 수정하면 순서 의존
for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++)
        grid[i][j] = /* 계산 */;

// 맞음: 복사본에 기록 후 교체
int[][] next = new int[n][m];
for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++)
        next[i][j] = /* grid 기반 계산 */;
grid = next;
```
