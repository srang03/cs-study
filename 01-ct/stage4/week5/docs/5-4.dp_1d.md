## 리뷰 미션 (5-3 바텀업)

**R1.** 바텀업에서 반복문을 `i = 2`부터 시작하는 이유는?

<details>
<summary>정답</summary>
dp[0]과 dp[1]은 base case로 직접 설정하고, i = 2부터 점화식으로 계산하기 때문. i = 0이나 1부터 시작하면 dp[i-1]이나 dp[i-2]에서 음수 인덱스 접근 에러가 발생한다.
</details>

**R2.** `dp[n] = dp[n-1] + 2 * dp[n-2]`가 되는 2×n 타일링 2의 점화식에서 2가 곱해지는 이유는?

<details>
<summary>정답</summary>
n-2 열에서 2칸을 채우는 방법이 2가지(1×2 가로 2개, 2×2 한 개)이기 때문. 기본 타일링에서는 1가지(1×2 가로 2개)뿐이었다.
</details>

## 1. 설명

### 1차원 DP란

DP 테이블이 **1차원 배열** `dp[i]`인 문제. 가장 기본적이고 자주 출제되는 DP 유형이다.

### 점화식 도출 과정

DP의 핵심은 **점화식**을 세우는 것이다.

```
1. dp[i]의 의미를 정의한다
2. dp[i]를 구하기 위해 어떤 이전 상태가 필요한지 파악한다
3. 점화식을 세운다: dp[i] = f(dp[i-1], dp[i-2], ...)
4. base case를 설정한다
5. 반복 방향과 범위를 결정한다
```

### 예시: 계단 오르기 (백준 2579)

**규칙:**
- 한 번에 1칸 또는 2칸
- 연속 3칸 밟기 불가
- 마지막 계단은 반드시 밟기

**점화식 도출:**

1. `dp[i]` = i번째 계단까지 올라갔을 때의 최대 점수
2. i번째 계단을 밟는 경우:
   - i-1도 밟음 → i-2는 못 밟음(연속 3칸 금지) → i-3에서 올라옴
   - i-1은 안 밟음 → i-2에서 올라옴

```
dp[i] = max(
    dp[i-2] + stair[i],              // i-2 → i (1칸 건너뜀)
    dp[i-3] + stair[i-1] + stair[i]  // i-3 → i-1 → i (연속 2칸)
)
```

### 1차원 DP 대표 유형

| 유형 | 점화식 패턴 | 대표 문제 |
|------|------------|-----------|
| 피보나치형 | `dp[i] = dp[i-1] + dp[i-2]` | 타일링, 계단 수 |
| 최솟값형 | `dp[i] = min(dp[이전]) + 비용` | 1로 만들기 |
| 최댓값형 | `dp[i] = max(dp[이전]) + 값` | 계단 오르기 |
| 경우의 수형 | `dp[i] = sum(dp[이전])` | 1,2,3 더하기 |

## 2. 예제

### 예제 1: 1로 만들기 (백준 1463)

정수 X에 사용할 수 있는 연산 3가지:
1. 3으로 나누어 떨어지면 3으로 나누기
2. 2로 나누어 떨어지면 2로 나누기
3. 1 빼기

X를 1로 만드는 최소 연산 횟수를 구하라.

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();

        int[] dp = new int[n + 1];
        // dp[i] = i를 1로 만드는 최소 연산 횟수
        // dp[1] = 0 (이미 1)

        for (int i = 2; i <= n; i++) {
            dp[i] = dp[i - 1] + 1;  // -1 연산
            if (i % 2 == 0) dp[i] = Math.min(dp[i], dp[i / 2] + 1);
            if (i % 3 == 0) dp[i] = Math.min(dp[i], dp[i / 3] + 1);
        }

        System.out.println(dp[n]);
    }
}
```

### 예제 2: 계단 오르기 (백준 2579)

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        int[] stair = new int[n + 1];
        for (int i = 1; i <= n; i++) {
            stair[i] = sc.nextInt();
        }

        if (n == 1) { System.out.println(stair[1]); return; }
        if (n == 2) { System.out.println(stair[1] + stair[2]); return; }

        int[] dp = new int[n + 1];
        dp[1] = stair[1];
        dp[2] = stair[1] + stair[2];
        dp[3] = Math.max(stair[1], stair[2]) + stair[3];

        for (int i = 4; i <= n; i++) {
            dp[i] = Math.max(
                dp[i - 2] + stair[i],
                dp[i - 3] + stair[i - 1] + stair[i]
            );
        }

        System.out.println(dp[n]);
    }
}
```

## 4. 이론 퀴즈

**Q1.** "1로 만들기"에서 `dp[i] = dp[i-1] + 1`을 먼저 계산하고, 나누기 가능하면 `min`을 취하는 이유는?

<details>
<summary>정답</summary>
-1 연산은 항상 가능하므로 기본값으로 설정하고, /2나 /3이 가능한 경우에만 더 작은 값이 있는지 비교한다.
</details>

**Q2.** 계단 오르기에서 `dp[i-3] + stair[i-1] + stair[i]`의 의미는?

<details>
<summary>정답</summary>
i-3까지의 최대 점수에서, i-1과 i를 연속으로 밟는 경우. i-2를 밟지 않으므로 연속 3칸 규칙을 위반하지 않는다.
</details>

**Q3.** dp[i]의 "정의"를 명확히 하는 것이 왜 중요한가?

<details>
<summary>정답</summary>
dp[i]의 의미가 모호하면 점화식을 세울 수 없고, 세우더라도 base case와 맞지 않는다. 정의가 곧 점화식의 기반이다.
</details>

## 5. 함정 노트

### 1. base case 부족

```java
// 틀림: n = 1일 때 dp[2], dp[3] 접근 에러
dp[1] = stair[1];
dp[2] = stair[1] + stair[2];
dp[3] = ...;

// 맞음: n이 작은 경우 미리 처리
if (n == 1) { System.out.println(stair[1]); return; }
```

### 2. 점화식에서 이전 상태 누락

"1로 만들기"에서 /3 연산만 고려하고 /2, -1을 빼먹으면 최소가 아닌 답이 나온다. **모든 가능한 이전 상태**를 고려해야 한다.

### 3. 최솟값 문제에서 초기값

```java
// 틀림: dp[i] = 0으로 시작하면 min이 항상 0
dp[i] = Math.min(dp[i], dp[i/2] + 1);

// 맞음: 먼저 하나의 유효한 값으로 설정
dp[i] = dp[i - 1] + 1;  // 기본값
dp[i] = Math.min(dp[i], dp[i/2] + 1);  // 비교
```

### 4. 인덱스 0-based vs 1-based

계단 오르기처럼 1번 계단부터 시작하는 문제는 1-based가 자연스럽다. 배열 크기를 `n+1`로 선언하는 것을 잊지 말라.
