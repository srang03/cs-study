## 1. 설명

### 탑다운이란

**큰 문제에서 시작하여 작은 문제로 내려가면서** 풀되, 이미 푼 문제는 메모에 저장하고 재사용하는 방식.

```
f(5) → f(4) → f(3) → f(2) → f(1) → f(0)  (처음 호출: 직접 계산)
         └→ f(3): 이미 계산됨 → 바로 반환!   (두 번째 호출: 메모 활용)
```

### 재귀 + 메모이제이션

탑다운 = **재귀** + **메모이제이션(Memoization)**.

- 재귀: week2에서 배운 base case + recursive case 구조 그대로
- 메모이제이션: 계산 결과를 배열(또는 Map)에 저장

```java
static int[] memo;

public static int solve(int n) {
    if (base case) return 값;
    if (memo[n] != -1) return memo[n];   // 이미 계산했으면 바로 반환
    memo[n] = /* 점화식 */;              // 처음 계산하면 저장
    return memo[n];
}
```

### week2 재귀와의 연결

week2에서 배운 재귀 패턴을 그대로 사용한다. 차이는 **메모 배열 추가** 하나뿐이다.

```
[week2 재귀]                     [탑다운 DP]
base case 확인        →     base case 확인
                            메모 확인 (이미 계산?)
재귀 호출             →     재귀 호출
결과 반환             →     결과 저장 + 반환
```

### 메모이제이션 배열 패턴

```java
// 1. 배열 선언 (전역 또는 클래스 필드)
static int[] memo;

// 2. 초기화: -1로 채우기 (0이 유효한 값일 수 있으므로)
memo = new int[n + 1];
Arrays.fill(memo, -1);

// 3. 재귀 함수에서 활용
public static int dp(int n) {
    if (n <= 1) return n;              // base case
    if (memo[n] != -1) return memo[n]; // 메모 확인
    memo[n] = dp(n - 1) + dp(n - 2);  // 계산 + 저장
    return memo[n];
}
```

### 시간 복잡도

각 상태를 **한 번만** 계산하므로:
- 상태 수 = N → O(N)
- 피보나치: O(2^n) → O(n)으로 개선

## 2. 예제

### 예제 1: 피보나치 탑다운

```java
import java.util.Arrays;

public class Main {
    static int[] memo;

    public static int fib(int n) {
        if (n <= 1) return n;
        if (memo[n] != -1) return memo[n];
        memo[n] = fib(n - 1) + fib(n - 2);
        return memo[n];
    }

    public static void main(String[] args) {
        int n = 10;
        memo = new int[n + 1];
        Arrays.fill(memo, -1);
        System.out.println(fib(n));  // 55
    }
}
```

### 예제 2: 계단 오르기 (1칸 또는 2칸)

N개의 계단을 1칸 또는 2칸씩 올라갈 때, 방법의 수를 구하라.

```java
import java.util.Arrays;

public class Main {
    static int[] memo;

    // 점화식: climb(n) = climb(n-1) + climb(n-2)
    // n번째 계단에 도달하려면 n-1에서 1칸 또는 n-2에서 2칸
    public static int climb(int n) {
        if (n <= 2) return n;
        if (memo[n] != -1) return memo[n];
        memo[n] = climb(n - 1) + climb(n - 2);
        return memo[n];
    }

    public static void main(String[] args) {
        int n = 5;
        memo = new int[n + 1];
        Arrays.fill(memo, -1);
        System.out.println(climb(n));  // 8
    }
}
```

### 예제 3: long 타입 메모이제이션

n이 크면 int 범위를 넘으므로 `long[]` 사용.

```java
static long[] memo;

public static long fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    memo[n] = fib(n - 1) + fib(n - 2);
    return memo[n];
}
```

### 과제 1: 탑다운 패턴 적용

순수 재귀로 작성된 아래 코드에 메모이제이션을 추가하라.

```java
// 순수 재귀: f(n) = f(n-1) + f(n-3), f(0)=0, f(1)=1, f(2)=1
public static int f(int n) {
    if (n <= 1) return n;
    if (n == 2) return 1;
    return f(n - 1) + f(n - 3);
}
```

## 4. 이론 퀴즈

**Q1.** 메모이제이션 배열을 -1로 초기화하는 이유는?

<details>
<summary>정답</summary>
0이 유효한 계산 결과일 수 있기 때문. -1은 "아직 계산하지 않음"을 의미하는 센티넬 값.
</details>

**Q2.** 탑다운의 시간 복잡도가 O(n)인 이유는?

<details>
<summary>정답</summary>
각 상태(0~n)가 최대 한 번만 계산되고, 이후에는 메모에서 O(1)로 반환되기 때문. 상태 수 n × 상태당 O(1) = O(n).
</details>

**Q3.** 탑다운 DP에서 재귀 깊이가 너무 깊으면 어떤 문제가 발생하는가?

<details>
<summary>정답</summary>
StackOverflowError. Java의 기본 스택 크기로는 약 수천~만 단위의 깊이가 한계. 이 경우 바텀업으로 전환하거나 스택 크기를 늘려야 한다.
</details>

## 5. 함정 노트

### 1. 메모 배열 크기

```java
// 틀림: n을 인덱스로 쓰는데 크기가 n
memo = new int[n];      // memo[n]에서 ArrayIndexOutOfBounds!

// 맞음
memo = new int[n + 1];  // 0~n까지 사용
```

### 2. 초기화 누락

```java
// 틀림: 0으로 초기화된 상태 — 0이 유효한 답이면 구분 불가
static int[] memo = new int[100];

// 맞음: -1로 초기화
Arrays.fill(memo, -1);
```

### 3. StackOverflowError

n이 10만 이상이면 재귀 깊이 한계에 도달할 수 있다. 이 경우 바텀업(5-3)으로 전환.

### 4. 전역 변수 초기화

여러 테스트 케이스를 처리할 때 `memo` 배열을 매번 초기화해야 한다. 이전 테스트의 값이 남아 있으면 오답.
