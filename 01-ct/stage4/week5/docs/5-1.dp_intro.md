## 1. 설명

### 동적 프로그래밍(DP)이란

**큰 문제를 작은 하위 문제로 나누고, 하위 문제의 결과를 저장(캐싱)하여 중복 계산을 피하는 기법**이다.

핵심 아이디어: "한 번 계산한 것은 다시 계산하지 않는다."

### DP를 적용할 수 있는 두 가지 조건

#### 1. 최적 부분 구조 (Optimal Substructure)

큰 문제의 최적 해가 **작은 문제의 최적 해로부터** 구해질 수 있는 구조.

```
f(n)의 최적 해 = f(n-1)의 최적 해 + 어떤 연산
```

예: 피보나치 — `fib(5) = fib(4) + fib(3)`

#### 2. 중복 부분 문제 (Overlapping Subproblems)

같은 하위 문제가 **여러 번 반복** 호출되는 구조.

```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2)  ← 중복!
│   │   └── fib(1)
│   └── fib(2)      ← 중복!
└── fib(3)           ← 중복!
    ├── fib(2)       ← 중복!
    └── fib(1)
```

fib(2)가 3번, fib(3)이 2번 중복 호출된다. 이런 중복을 제거하는 것이 DP의 핵심.

### 분할정복과의 차이

| 비교 | 분할정복 | DP |
|------|---------|------|
| 하위 문제 중복 | 없음 | 있음 |
| 결과 저장 | 안 함 | 저장 (메모이제이션/테이블) |
| 대표 예시 | 병합 정렬, 퀵 정렬 | 피보나치, 최단 경로 |

분할정복은 하위 문제가 서로 독립이라 중복이 없다. DP는 하위 문제가 겹치므로 결과를 저장해 효율을 높인다.

### DP 없이 순수 재귀의 문제

```java
// 피보나치: 순수 재귀 — O(2^n)
public static int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

n = 40이면 약 10억 번 호출. **시간 초과 확정.**

## 2. 예제

### 예제 1: 순수 재귀 피보나치의 호출 횟수

```java
public class Main {
    static int count = 0;

    public static int fib(int n) {
        count++;
        if (n <= 1) return n;
        return fib(n - 1) + fib(n - 2);
    }

    public static void main(String[] args) {
        System.out.println(fib(30));     // 832040
        System.out.println("호출 횟수: " + count);  // 2692537
    }
}
```

n = 30만 해도 약 270만 번 호출된다.

### 예제 2: DP로 피보나치 최적화 (미리보기)

```java
public class Main {
    static int count = 0;
    static int[] memo = new int[31];

    public static int fib(int n) {
        count++;
        if (n <= 1) return n;
        if (memo[n] != 0) return memo[n];  // 이미 계산했으면 재사용
        memo[n] = fib(n - 1) + fib(n - 2);
        return memo[n];
    }

    public static void main(String[] args) {
        System.out.println(fib(30));     // 832040
        System.out.println("호출 횟수: " + count);  // 59
    }
}
```

호출 횟수가 270만 → 59로 줄어든다. **O(2^n) → O(n)**.

<details>
<summary>정답</summary>

</details>

**Q3.** 분할정복과 DP의 핵심 차이는?

<details>
<summary>정답</summary>
하위 문제의 중복 여부. 분할정복은 하위 문제가 독립(중복 없음), DP는 하위 문제가 중복되어 결과를 저장한다.
</details>

**Q4.** 병합 정렬에 DP를 적용하지 않는 이유는?

<details>
<summary>정답</summary>
병합 정렬에서 나누어진 부분 배열은 서로 다르므로 중복 부분 문제가 없다. DP의 조건을 만족하지 않는다.
</details>

## 5. 함정 노트

### 1. "재귀 = DP"가 아니다

재귀는 단순히 함수 호출 방식이고, DP는 중복 계산을 제거하는 알고리즘 기법이다. 재귀 + 메모이제이션이어야 탑다운 DP다.

### 2. 모든 문제에 DP를 쓸 수 없다

DP의 두 조건(최적 부분 구조 + 중복 부분 문제)을 만족해야 한다. 그리디가 적합한 문제에 DP를 쓰면 오버킬이다.

### 3. "최적 부분 구조"의 오해

"작은 문제를 풀면 큰 문제가 풀린다"는 뜻이지, "작은 문제를 먼저 풀어야 한다"는 뜻이 아니다. 탑다운(큰→작은)도 DP다.

### 4. DP vs 그리디

DP는 모든 하위 문제를 고려. 그리디는 현재 최선만 선택. 문제에 따라 그리디가 최적해를 보장하지 못할 수 있다.
