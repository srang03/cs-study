## 1. 설명

### 바텀업이란

**작은 문제부터 순서대로 풀어 올라가는** 방식. 재귀 대신 **반복문**을 사용한다.

```
dp[0] → dp[1] → dp[2] → ... → dp[n]
(작은 문제부터 차례대로 테이블을 채워나간다)
```

### 반복문 + 테이블

바텀업 = **반복문** + **DP 테이블(배열)**.

```java
int[] dp = new int[n + 1];

// base case
dp[0] = 0;
dp[1] = 1;

// 점화식으로 테이블 채우기
for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}

System.out.println(dp[n]);  // 답
```

### 탑다운 vs 바텀업

| 비교 | 탑다운 | 바텀업 |
|------|--------|--------|
| 구현 방식 | 재귀 + 메모이제이션 | 반복문 + 테이블 |
| 방향 | 큰 문제 → 작은 문제 | 작은 문제 → 큰 문제 |
| 스택 오버플로우 | 가능 (n이 클 때) | 없음 |
| 필요한 상태만 계산 | O (호출된 것만) | X (모든 상태 계산) |
| 코테 추천 | 점화식이 복잡할 때 | **일반적으로 추천** |

### 바텀업이 일반적으로 유리한 이유

1. **스택 오버플로우 없음**: 반복문이므로 재귀 깊이 제한 없음
2. **함수 호출 오버헤드 없음**: 재귀 호출 비용이 없어 약간 빠름
3. **코드가 단순**: 배열 채우기 패턴으로 직관적

### 바텀업 구현 패턴

```java
// 1. DP 테이블 선언
int[] dp = new int[n + 1];

// 2. base case 설정
dp[0] = 초기값;
dp[1] = 초기값;

// 3. 점화식으로 채우기 (작은 → 큰)
for (int i = 시작; i <= n; i++) {
    dp[i] = 점화식(dp[i-1], dp[i-2], ...);
}

// 4. 답 출력
System.out.println(dp[n]);
```

## 2. 예제

### 예제 1: 피보나치 바텀업

```java
public class Main {
    public static void main(String[] args) {
        int n = 10;
        long[] dp = new long[n + 1];

        dp[0] = 0;
        dp[1] = 1;

        for (int i = 2; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }

        System.out.println(dp[n]);  // 55
    }
}
```

### 예제 2: 탑다운 → 바텀업 변환

```java
// [탑다운]
static int[] memo;
public static int fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    memo[n] = fib(n - 1) + fib(n - 2);
    return memo[n];
}

// [바텀업] — 같은 점화식, 다른 구현
int[] dp = new int[n + 1];
dp[0] = 0; dp[1] = 1;
for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}
```

변환 핵심:
1. base case → 배열 초기값
2. 점화식 → 반복문 안의 계산식
3. 메모 확인 코드 → 불필요 (순서대로 채우므로)

### 예제 3: 공간 최적화

피보나치는 직전 두 값만 필요하므로 배열 대신 변수 2개로 해결 가능.

```java
public class Main {
    public static void main(String[] args) {
        int n = 10;
        if (n <= 1) { System.out.println(n); return; }

        long prev2 = 0, prev1 = 1;
        for (int i = 2; i <= n; i++) {
            long curr = prev1 + prev2;
            prev2 = prev1;
            prev1 = curr;
        }
        System.out.println(prev1);  // 55
    }
}
```

O(n) 공간 → O(1) 공간으로 최적화.

## 4. 이론 퀴즈

**Q1.** 바텀업에서 반복문의 순회 방향이 중요한 이유는?

<details>
<summary>정답</summary>
dp[i]를 계산할 때 dp[i-1], dp[i-2] 등 이전 값이 이미 채워져 있어야 하므로, 작은 인덱스부터 큰 인덱스 순으로 채워야 한다.
</details>

**Q2.** 탑다운이 바텀업보다 유리한 경우는?

<details>
<summary>정답</summary>
전체 상태 중 일부만 필요할 때. 탑다운은 호출된 상태만 계산하지만 바텀업은 모든 상태를 채운다. 또한 점화식이 복잡하여 순회 순서를 정하기 어려울 때.
</details>

**Q3.** 피보나치의 공간 최적화가 가능한 이유는?

<details>
<summary>정답</summary>
dp[i]가 dp[i-1]과 dp[i-2]만 참조하므로, 전체 배열이 아닌 직전 두 값만 기억하면 된다.
</details>

## 5. 함정 노트

### 1. 반복 범위 오류

```java
// 틀림: i = 0부터 시작하면 dp[-1] 접근
for (int i = 0; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];  // i=0일 때 dp[-1]!
}

// 맞음: base case 이후부터 시작
dp[0] = 0; dp[1] = 1;
for (int i = 2; i <= n; i++) {
    dp[i] = dp[i - 1] + dp[i - 2];
}
```

### 2. base case 누락

바텀업에서 base case를 빠뜨리면 dp 배열이 전부 0으로 남는다. 반드시 초기값을 설정.

### 3. n = 0 또는 n = 1 예외 처리

```java
// n = 0일 때 dp[1]에 접근하면 ArrayIndexOutOfBounds
int[] dp = new int[n + 1];
dp[0] = 0;
dp[1] = 1;  // n = 0이면 dp 크기가 1이라 에러!

// 안전한 처리
if (n == 0) { System.out.println(0); return; }
```

### 4. 공간 최적화 시 역참조 불가

공간 최적화(변수 2~3개)를 하면 **경로 추적이 불가능**해진다. 최적 해의 "값"만 필요할 때 사용.

- **난이도**: Silver III
