## 리뷰 미션 (5-4 1차원 DP)

**R1.** "1로 만들기"에서 10을 1로 만드는 최소 연산 횟수는?

<details>
<summary>정답</summary>
3회. 10 → 9(/3이 안 되므로 -1) → 3(/3) → 1(/3). 또는 10 → 5(/2) → 4(-1) → 2(/2) → 1도 4회라 틀림. 정답은 10→9→3→1 = 3회.
</details>

**R2.** "1, 2, 3 더하기"에서 `dp[4]`의 값은?

<details>
<summary>정답</summary>
7. dp[4] = dp[3] + dp[2] + dp[1] = 4 + 2 + 1 = 7. {1+1+1+1, 1+1+2, 1+2+1, 2+1+1, 2+2, 1+3, 3+1}.
</details>

## 1. 설명

### 2차원 DP란

DP 테이블이 **2차원 배열** `dp[i][j]`인 문제. 상태를 표현하는 데 변수가 2개 필요할 때 사용한다.

### 격자 경로 문제

m×n 격자에서 좌상단(0,0) → 우하단(m-1,n-1)까지 **오른쪽 또는 아래로만** 이동할 때의 경로 수.

```
dp[i][j] = (i,j)까지 도달하는 경로의 수
dp[i][j] = dp[i-1][j] + dp[i][j-1]
           (위에서 온 것 + 왼쪽에서 온 것)
```

```
1  1  1  1
1  2  3  4
1  3  6  10
```

### 2차원 DP의 점화식 도출

```
1. dp[i][j]의 의미를 정의한다
2. dp[i][j]가 어떤 이전 상태에서 올 수 있는지 파악한다
3. 점화식을 세운다: dp[i][j] = f(dp[i-1][j], dp[i][j-1], ...)
4. base case를 설정한다 (보통 첫 행, 첫 열)
5. 순회 방향을 결정한다 (보통 행 우선, 위→아래, 왼→오른)
```

### 배낭 문제 입문 (0/1 Knapsack)

N개의 물건, 각각 무게 `w[i]`와 가치 `v[i]`가 있다. 배낭 용량 W일 때 최대 가치를 구하라.

**상태 정의:**
- `dp[i][j]` = i번째 물건까지 고려했을 때, 배낭 용량 j에서의 최대 가치

**점화식:**
```
dp[i][j] = max(
    dp[i-1][j],                    // i번째 물건을 안 넣는 경우
    dp[i-1][j - w[i]] + v[i]      // i번째 물건을 넣는 경우 (j >= w[i]일 때)
)
```

### 2차원 DP 대표 유형

| 유형 | 상태 정의 | 대표 문제 |
|------|----------|-----------|
| 격자 경로 | dp[i][j] = (i,j)까지의 경로/최대/최소 | RGB거리, 정수 삼각형 |
| 부분 수열 | dp[i] = i번째 원소까지의 최장 길이 | LIS (1차원이지만 2차원 확장 가능) |
| 배낭 | dp[i][j] = i번째까지 고려, 용량 j | 0/1 Knapsack |

## 2. 예제

### 예제 1: RGB거리 (백준 1149)

N개의 집을 빨강/초록/파랑으로 칠한다. 인접한 집은 같은 색 불가. 최소 비용을 구하라.

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        int[][] cost = new int[n][3];
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < 3; j++) {
                cost[i][j] = sc.nextInt();
            }
        }

        // dp[i][j] = i번째 집을 색 j로 칠할 때의 최소 누적 비용
        int[][] dp = new int[n][3];
        dp[0][0] = cost[0][0];
        dp[0][1] = cost[0][1];
        dp[0][2] = cost[0][2];

        for (int i = 1; i < n; i++) {
            dp[i][0] = Math.min(dp[i-1][1], dp[i-1][2]) + cost[i][0];
            dp[i][1] = Math.min(dp[i-1][0], dp[i-1][2]) + cost[i][1];
            dp[i][2] = Math.min(dp[i-1][0], dp[i-1][1]) + cost[i][2];
        }

        System.out.println(Math.min(dp[n-1][0], Math.min(dp[n-1][1], dp[n-1][2])));
    }
}
```

### 예제 2: LIS (가장 긴 증가하는 부분 수열)

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        int[] arr = new int[n];
        for (int i = 0; i < n; i++) arr[i] = sc.nextInt();

        // dp[i] = arr[i]를 마지막 원소로 하는 LIS의 길이
        int[] dp = new int[n];
        for (int i = 0; i < n; i++) {
            dp[i] = 1;  // 자기 자신만 포함
            for (int j = 0; j < i; j++) {
                if (arr[j] < arr[i]) {
                    dp[i] = Math.max(dp[i], dp[j] + 1);
                }
            }
        }

        int ans = 0;
        for (int i = 0; i < n; i++) ans = Math.max(ans, dp[i]);
        System.out.println(ans);
    }
}
```

시간 복잡도: O(n^2). (이진 탐색 활용 시 O(n log n)도 가능하지만 이 단계에서는 O(n^2)으로 충분)

### 예제 3: 0/1 배낭 문제

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();  // 물건 수
        int W = sc.nextInt();  // 배낭 용량

        int[] w = new int[n + 1];
        int[] v = new int[n + 1];
        for (int i = 1; i <= n; i++) {
            w[i] = sc.nextInt();
            v[i] = sc.nextInt();
        }

        // dp[i][j] = i번째 물건까지 고려, 용량 j일 때 최대 가치
        int[][] dp = new int[n + 1][W + 1];

        for (int i = 1; i <= n; i++) {
            for (int j = 0; j <= W; j++) {
                dp[i][j] = dp[i - 1][j];  // 안 넣는 경우
                if (j >= w[i]) {
                    dp[i][j] = Math.max(dp[i][j], dp[i - 1][j - w[i]] + v[i]);
                }
            }
        }

        System.out.println(dp[n][W]);
    }
}
```

## 4. 이론 퀴즈

**Q1.** RGB거리에서 dp 테이블의 두 번째 차원(색상)이 필요한 이유는?

<details>
<summary>정답</summary>
다음 집의 색을 결정할 때 이전 집의 색이 무엇이었는지 알아야 한다(같은 색 불가). 따라서 "몇 번째 집"과 "어떤 색"이라는 2개의 상태가 필요하다.
</details>

**Q2.** 0/1 배낭에서 `dp[i-1][j - w[i]]`의 의미는?

<details>
<summary>정답</summary>
i번째 물건을 넣기 위해 w[i]만큼의 공간을 확보한 상태에서, i-1번째까지의 물건으로 달성할 수 있는 최대 가치.
</details>

**Q3.** LIS의 O(n^2) 풀이에서 dp[i]의 초기값이 1인 이유는?

<details>
<summary>정답</summary>
자기 자신만으로도 길이 1의 증가하는 부분 수열이 되기 때문.
</details>

**Q4.** 2차원 DP에서 순회 순서가 중요한 이유는?

<details>
<summary>정답</summary>
dp[i][j]를 계산할 때 참조하는 dp[i-1][j], dp[i][j-1] 등이 이미 계산되어 있어야 하므로. 위→아래, 왼→오른 순서가 일반적.
</details>

## 5. 함정 노트

### 1. 배낭 문제에서 j < w[i] 체크 누락

```java
// 틀림: j - w[i]가 음수가 될 수 있음
dp[i][j] = Math.max(dp[i-1][j], dp[i-1][j - w[i]] + v[i]);

// 맞음: 무게 체크 필수
if (j >= w[i]) {
    dp[i][j] = Math.max(dp[i-1][j], dp[i-1][j - w[i]] + v[i]);
} else {
    dp[i][j] = dp[i-1][j];
}
```

### 2. RGB거리에서 마지막 답 처리

```java
// 틀림: dp[n-1][0]만 출력
System.out.println(dp[n-1][0]);

// 맞음: 마지막 집의 3가지 색 중 최솟값
Math.min(dp[n-1][0], Math.min(dp[n-1][1], dp[n-1][2]));
```

### 3. LIS에서 답이 dp[n-1]이 아님

```java
// 틀림: 마지막 원소가 LIS의 끝이 아닐 수 있음
System.out.println(dp[n - 1]);

// 맞음: 모든 dp[i] 중 최대값
int ans = 0;
for (int i = 0; i < n; i++) ans = Math.max(ans, dp[i]);
```

### 4. 0-based vs 1-based 혼동

배낭 문제는 1-based(물건 번호 1~n), 격자 경로는 0-based(좌표 (0,0)~(m-1,n-1))가 자연스러운 경우가 많다. 배열 크기와 인덱스를 일관되게 사용하라.

### 문제 1: [백준 1149 - RGB거리](https://www.acmicpc.net/problem/1149)

- **난이도**: Silver I
- **핵심**: 인접 집 같은 색 금지 조건의 최소 비용 DP

### 문제 2: [백준 11053 - 가장 긴 증가하는 부분 수열](https://www.acmicpc.net/problem/11053)

- **난이도**: Silver II
- **핵심**: LIS. dp[i] = arr[i]를 끝으로 하는 LIS 길이
- **힌트**: 이중 for문. j < i이고 arr[j] < arr[i]이면 dp[i] = max(dp[i], dp[j]+1)
