## 1. 설명

### 점근 표기법 (Asymptotic Notation)

입력 크기 N이 **충분히 클 때** 알고리즘의 성능 증가 추세를 나타내는 수학적 표기법.

### Big-O (상한)

**최악의 경우** 성능 상한을 나타낸다. 코테에서 가장 많이 사용.

- "이 알고리즘은 아무리 나빠도 이 정도 이내"
- f(n) = O(g(n)): 충분히 큰 n에서 f(n) ≤ c·g(n)

```
3n² + 5n + 10 = O(n²)
→ 최고차항만 남기고, 상수 계수 제거
```

### Big-Omega (하한)

**최선의 경우** 성능 하한을 나타낸다.

- "이 알고리즘은 최소한 이 정도는 걸린다"
- f(n) = Ω(g(n)): 충분히 큰 n에서 f(n) ≥ c·g(n)

### Big-Theta (딱 그만큼)

상한과 하한이 같을 때. 정확한 증가 추세.

- f(n) = Θ(g(n)): O(g(n))이면서 Ω(g(n))

### 코테에서의 실용

**Big-O만 알면 충분하다.** Omega, Theta는 이론적 이해를 위한 것.

| 다항식 | Big-O |
|--------|-------|
| 5n + 3 | O(n) |
| 2n² + 100n | O(n²) |
| 3n log n + n | O(n log n) |
| 7 | O(1) |

### 규칙

1. **상수 제거**: O(3n) → O(n)
2. **낮은 차수 제거**: O(n² + n) → O(n²)
3. **곱은 유지**: O(n × m) → O(nm)

## 2. 예제

### 예제 1: Big-O 판단

```java
// T(n) = 3n + 5 → O(n)
for (int i = 0; i < n; i++) {
    // 상수 시간 작업
}
// 반복 전후 상수 작업 5개 → 무시

// T(n) = n² + 2n + 1 → O(n²)
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        // 상수 시간 작업
// 2n + 1은 n²에 비해 무시 가능
```

### 예제 2: 순차적 코드 vs 중첩 코드

```java
// 순차: O(n) + O(n) = O(n)
for (int i = 0; i < n; i++) { }
for (int j = 0; j < n; j++) { }

// 중첩: O(n) × O(n) = O(n²)
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) { }
```

- 순차는 **더하기** → 큰 쪽이 지배
- 중첩은 **곱하기** → 전체가 곱

## 4. 이론 퀴즈

**Q1.** O(n² + n log n)을 간단히 하면?

<details>
<summary>정답</summary>
O(n²). n²이 n log n보다 빠르게 증가하므로 지배한다.
</details>

**Q2.** Big-O와 Big-Theta의 차이는?

<details>
<summary>정답</summary>
Big-O는 상한(최악), Big-Theta는 정확한 증가 추세(상한=하한). O(n)이라고 해서 항상 n번 수행하는 것은 아니지만, Θ(n)이면 항상 n에 비례한다.
</details>

**Q3.** O(1)이란?

<details>
<summary>정답</summary>
입력 크기와 무관하게 일정한 시간. 상수 시간. 배열 인덱스 접근, HashMap 조회 등.
</details>

## 5. 함정 노트

### 1. O(n)은 정확히 n번이 아니다

O(n)은 "n에 비례하는 상한"이지, 정확히 n번 실행을 뜻하지 않는다. 2n, 0.5n, 100n 모두 O(n).

### 2. O(log n)의 밑(base) 무시

```
log₂n과 log₁₀n은 상수 배 차이 → Big-O에서 같음
O(log₂n) = O(log₁₀n) = O(log n)
```

### 3. 재귀의 복잡도 판단

단순히 "재귀니까 O(n)"이 아니다. 호출 구조를 분석해야 한다.

```java
// O(n): 선형 재귀
f(n) → f(n-1) → f(n-2) → ... → f(0)

// O(2^n): 이진 재귀 (피보나치)
f(n) → f(n-1) + f(n-2) → ...
```
