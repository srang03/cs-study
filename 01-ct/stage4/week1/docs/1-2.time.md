## 1. 설명

### 시간 복잡도란

알고리즘이 **입력 크기 N에 따라 수행하는 연산 횟수의 증가 추세**를 표현한 것이다.

### 주요 시간 복잡도

| 표기 | 이름 | 예시 |
|------|------|------|
| O(1) | 상수 | 배열 인덱스 접근, HashMap get |
| O(log n) | 로그 | 이진 탐색 |
| O(n) | 선형 | 배열 순회, 합계 |
| O(n log n) | 선형 로그 | 정렬 (Arrays.sort) |
| O(n²) | 이차 | 이중 반복문 |
| O(n³) | 삼차 | 삼중 반복문 |
| O(2^n) | 지수 | 부분집합 열거 |
| O(n!) | 팩토리얼 | 순열 전체 탐색 |

### 시간 복잡도 판단법

**반복문의 수와 범위**를 분석한다.

```java
// O(n): 반복 1개, 범위 n
for (int i = 0; i < n; i++) { }

// O(n²): 이중 반복, 각각 n
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) { }

// O(n * m): 이중 반복, 범위 다름
for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++) { }

// O(log n): 반복마다 절반
for (int i = n; i > 0; i /= 2) { }

// O(n log n): 외부 n × 내부 log n
for (int i = 0; i < n; i++)
    for (int j = 1; j < n; j *= 2) { }
```

### 연산 횟수 추정

| N | O(n) | O(n log n) | O(n²) | O(n³) |
|---|------|-----------|-------|-------|
| 100 | 100 | ~700 | 10,000 | 1,000,000 |
| 1,000 | 1,000 | ~10,000 | 1,000,000 | 10^9 (초과) |
| 100,000 | 100,000 | ~1,700,000 | 10^10 (초과) | - |
| 10^6 | 10^6 | ~2×10^7 | 10^12 (초과) | - |

## 2. 예제

### 예제 1: 각 코드의 시간 복잡도

```java
// A: O(1)
int x = arr[0] + arr[n-1];

// B: O(n)
int max = arr[0];
for (int i = 1; i < n; i++) {
    if (arr[i] > max) max = arr[i];
}

// C: O(n²)
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        if (arr[i] + arr[j] == target) count++;
    }
}
```

### 예제 2: 같은 문제, 다른 복잡도

"배열에 특정 값이 존재하는가?"

```java
// O(n): 선형 탐색
for (int num : arr) {
    if (num == target) return true;
}

// O(log n): 이진 탐색 (정렬 전제)
int lo = 0, hi = arr.length - 1;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2;
    if (arr[mid] == target) return true;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid - 1;
}

// O(1): HashSet
Set<Integer> set = new HashSet<>(Arrays.asList(/* arr */));
return set.contains(target);
```

아래 코드 각각의 시간 복잡도를 종이에 먼저 적고, N=10000으로 실행하여 대략적 연산 횟수를 확인하라.

```java
// 코드 1
int count = 0;
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)

System.out.println(count);

int count = 0;
for (int i = 1; i <= n; i *= 2)
    count++;
System.out.println(count);

int count = 0;
for (int i = 0; i < n; i++)
    for (int j = 1; j <= n; j *= 2)
        count++;

```

## 4. 이론 퀴즈

**Q1.** O(n)과 O(2n)의 차이는?

<details>
<summary>정답</summary>
같다. Big-O에서 상수 계수는 무시한다. O(2n) = O(n).
</details>

**Q2.** `Arrays.sort()`의 시간 복잡도는?

<details>
<summary>정답</summary>
O(n log n). Java의 원시 타입 배열은 Dual-Pivot QuickSort, 객체 배열은 TimSort를 사용한다.
</details>

**Q3.** HashMap의 `get()`과 `put()`의 시간 복잡도는?

<details>
<summary>정답</summary>
평균 O(1). 최악의 경우(해시 충돌이 심할 때) O(n)이지만, 실무에서는 O(1)로 간주한다.
</details>

## 5. 함정 노트

### 1. 상수 계수 무시의 한계

O(n)이라도 내부에 무거운 연산(문자열 비교 등)이 있으면 실질적으로 느릴 수 있다.

### 2. 최선/평균/최악 혼동

- QuickSort: 평균 O(n log n), 최악 O(n²)
- HashMap: 평균 O(1), 최악 O(n)
- 코테에서는 **최악**을 기준으로 판단하는 것이 안전

### 3. 재귀의 복잡도

```java
// 피보나치 재귀: O(2^n) — 매우 느림
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}
```
