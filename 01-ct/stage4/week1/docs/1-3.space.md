# 1-3 공간 복잡도

> 랭크: Platinum 진행 중 (week1 3/4)

## 1. 설명

### 공간 복잡도란

알고리즘이 사용하는 **메모리 양**의 입력 크기 N에 대한 증가 추세.

### 메모리 사용 요소

| 요소 | 공간 |
|------|------|
| 변수 1개 (`int`, `long`) | O(1) |
| 크기 N 배열 | O(n) |
| N×M 2차원 배열 | O(nm) |
| 재귀 깊이 D | O(d) (스택 프레임) |

### 코테 메모리 제한

- 보통 **256MB**
- `int[10,000,000]` ≈ 40MB → 가능
- `int[100,000,000]` ≈ 400MB → 초과
- `int[1000][1000]` = 100만 × 4byte ≈ 4MB → 충분

### 메모리 추정법

| 타입 | 크기 |
|------|------|
| `int` | 4 byte |
| `long` | 8 byte |
| `double` | 8 byte |
| `boolean` | 1 byte (JVM 구현 의존) |
| `Integer` (Wrapper) | ~16 byte (객체 오버헤드) |

> ArrayList, HashMap 등 컬렉션은 객체 오버헤드가 추가되므로 배열보다 메모리를 더 사용한다.

## 2. 예제

### 예제 1: 공간 비교

```java
// O(1) 공간: 변수만 사용
int sum = 0;
for (int i = 1; i <= n; i++) sum += i;

// O(n) 공간: 배열 사용
int[] arr = new int[n];
for (int i = 0; i < n; i++) arr[i] = i + 1;

// O(n) 공간: 재귀 (스택 프레임 n개)
int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1);  // 스택에 n개 프레임
}
```

### 예제 2: 메모리 절약

```java
// 불필요한 배열 생성
int[] arr = new int[n];
// ... arr에 저장 후 합계 계산

// 개선: 배열 없이 바로 계산
int sum = 0;
// 입력받으면서 바로 합산
```

### 과제

## 4. 이론 퀴즈

**Q1.** 재귀 함수의 깊이가 N일 때 공간 복잡도는?

<details>
<summary>정답</summary>
O(n). 각 재귀 호출마다 스택 프레임이 쌓이므로.
</details>

**Q2.** 256MB에서 `int` 배열은 최대 몇 개까지 가능한가?

<details>
<summary>정답</summary>
약 6,400만 개. 256MB = 256 × 1024 × 1024 byte ≈ 2.68억 byte. int 1개 = 4 byte. 268,000,000 / 4 ≈ 67,000,000. 실제로는 JVM 오버헤드를 빼면 약 6,000만~6,400만.
</details>

## 5. 함정 노트

### 1. 재귀 깊이에 의한 StackOverflowError

```java
// n = 1,000,000이면 스택 오버플로우
int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1);
}
```

Java 기본 스택 크기에서 보통 10,000~50,000 깊이가 한계.

### 2. 불필요한 객체 생성

```java
// Integer[] 대신 int[] 사용 (메모리 4배 차이)
Integer[] arr = new Integer[n];  // ~16 byte × n
int[] arr = new int[n];          // 4 byte × n
```
