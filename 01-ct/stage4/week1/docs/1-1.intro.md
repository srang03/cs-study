## 1. 설명

### 알고리즘이란

**입력을 받아 원하는 출력을 만들어내는 유한한 단계의 절차**이다.

- **입력**: 문제에서 주어지는 데이터
- **출력**: 문제가 요구하는 결과
- **유한성**: 반드시 종료되어야 한다

### 좋은 알고리즘의 조건

1. **정확성**: 모든 입력에 대해 올바른 결과
2. **효율성**: 적은 시간과 메모리로 해결
3. **명확성**: 각 단계가 모호하지 않음

### 코테에서의 알고리즘

```
문제 → 제약 조건 확인 → 알고리즘 선택 → 구현 → 검증
```

| 제약 | 알고리즘 선택 기준 |
|------|------------------|
| N ≤ 10 | O(n!) 브루트포스 가능 |
| N ≤ 20 | O(2^n) 가능 |
| N ≤ 500 | O(n³) 가능 |
| N ≤ 5,000 | O(n²) 가능 |
| N ≤ 100,000 | O(n log n) 필요 |
| N ≤ 10,000,000 | O(n) 필요 |

> 1초에 약 1~2억 번의 단순 연산이 가능하다고 가정한다.

### 왜 복잡도 분석이 중요한가

코테에서 **시간 제한**(보통 1~2초)과 **메모리 제한**(보통 256MB)이 있다.
알고리즘을 선택하기 전에 "이 방법으로 시간 안에 풀 수 있는가?"를 먼저 판단해야 한다.

## 2. 예제

### 예제 1: 브루트포스 vs 효율적 접근

1부터 N까지의 합을 구하는 두 가지 방법:

```java
// 방법 1: 반복문 O(n)
int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;
}

// 방법 2: 공식 O(1)
int sum = n * (n + 1) / 2;
```

### 예제 2: 제약 조건에 따른 접근법 선택

"배열에서 합이 K인 두 수 찾기"

```java
// N ≤ 5000이면 O(n²) 가능
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        if (arr[i] + arr[j] == k) { /* found */ }
    }
}

// N ≤ 100000이면 O(n) 해시 필요
Set<Integer> seen = new HashSet<>();
for (int num : arr) {
    if (seen.contains(k - num)) { /* found */ }
    seen.add(num);
}
```

## 4. 이론 퀴즈

**Q1.** N = 1,000,000일 때 O(n²) 알고리즘이 1초 안에 실행 가능한가?

<details>
<summary>정답</summary>
불가. n² = 10^12 연산으로, 1초(약 10^8 연산)를 크게 초과한다.
</details>

**Q2.** O(n log n) 알고리즘이 O(n²)보다 항상 빠른가?

<details>
<summary>정답</summary>
충분히 큰 n에서는 항상 빠르다. 하지만 n이 매우 작으면(예: n < 10) 상수 계수에 따라 O(n²)이 더 빠를 수도 있다. 점근 표기법은 n → ∞ 에서의 성장 속도를 비교한다.
</details>

**Q3.** 코테에서 시간 제한이 2초이면 약 몇 번의 연산이 가능한가?

<details>
<summary>정답</summary>
약 2~4억 번. (1초에 약 1~2억 번 기준)
</details>

## 5. 함정 노트

### 1. 제약 조건을 안 읽고 시작

문제를 풀기 전에 반드시 N의 범위를 확인하라. N ≤ 100이면 O(n³)도 되고, N ≤ 100,000이면 O(n log n) 이상이 필요하다.

### 2. int 오버플로우

N이 크면 중간 계산에서 int 범위(약 21억)를 초과할 수 있다.
합계, 곱셈 등에서 `long`을 써야 하는지 확인.

### 3. 시간 초과 ≠ 코드 버그

답은 맞지만 시간이 부족한 경우. 알고리즘 자체를 바꿔야 한다.

## 7. 보스 문제

### 복잡도 분석 연습

아래 각 코드의 시간 복잡도를 분석하라 (종이에 풀기):

```java
// 코드 A
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++)
        sum++;

// 코드 B
for (int i = 0; i < n; i++)
    for (int j = i; j < n; j++)
        sum++;

// 코드 C
for (int i = 1; i < n; i *= 2)
    sum++;

// 코드 D
for (int i = 0; i < n; i++)
    for (int j = 0; j < m; j++)
        sum++;
```

<details>
<summary>정답</summary>

- A: O(n²)
- B: O(n²) — n + (n-1) + ... + 1 = n(n+1)/2
- C: O(log n) — i가 2배씩 증가
- D: O(nm) — 이중 반복이지만 범위가 다름

</details>
