</details>

**R2.** 다음 코드의 시간 복잡도는?

```java
for (int i = 0; i < n; i++)

```

<details>
<summary>정답</summary>

</details>

## 1. 설명

### 배열 (Array)

**연속된 메모리 공간**에 같은 타입의 요소를 저장하는 자료구조.

```
인덱스:  [0] [1] [2] [3] [4]
메모리:  |10 |20 |30 |40 |50 |  ← 연속 배치
```

- 인덱스로 **O(1)** 접근 (시작 주소 + offset 계산)
- 크기가 고정 (Java 기본 배열)

### 링크드리스트 (Linked List)

**노드(Node)**가 데이터와 다음 노드의 참조를 갖는 자료구조. 메모리가 연속일 필요 없다.

```
[10|→] → [20|→] → [30|→] → [40|→] → [50|null]
 head                                    tail
```

- 접근: **O(n)** — 처음부터 순회해야 함
- 삽입/삭제: **O(1)** — 포인터만 변경 (위치를 이미 알고 있을 때)

### 시간 복잡도 비교

| 연산 | 배열 | 링크드리스트 |
|------|------|-------------|
| 접근 (i번째) | **O(1)** | O(n) |
| 맨 앞 삽입 | O(n) — 전체 시프트 | **O(1)** |
| 맨 뒤 삽입 | O(1) — 공간 있을 때 | **O(1)** — tail 있을 때 |
| 중간 삽입 | O(n) — 시프트 | O(n) — 탐색 후 O(1) |
| 맨 앞 삭제 | O(n) — 시프트 | **O(1)** |
| 중간 삭제 | O(n) | O(n) — 탐색 후 O(1) |
| 탐색 (값 찾기) | O(n) | O(n) |

### Java: ArrayList vs LinkedList

둘 다 `List<E>` 인터페이스를 구현한다.

```java
import java.util.ArrayList;
import java.util.LinkedList;

List<Integer> arrayList = new ArrayList<>();
List<Integer> linkedList = new LinkedList<>();
```

| 항목 | ArrayList | LinkedList |
|------|-----------|------------|
| 내부 구조 | 동적 배열 (배열 복사로 확장) | 이중 연결 리스트 |
| `get(i)` | **O(1)** | O(n) |
| `add(e)` (맨 뒤) | **O(1)** amortized | O(1) |
| `add(0, e)` (맨 앞) | O(n) | **O(1)** |
| `remove(0)` | O(n) | **O(1)** |
| 메모리 | 적음 (데이터만) | 많음 (노드 + 포인터) |
| 캐시 친화성 | **높음** (연속 메모리) | 낮음 (분산 메모리) |

### 언제 무엇을 쓰는가

- **ArrayList**: 대부분의 경우. 랜덤 접근이 많거나, 순차 접근 위주일 때
- **LinkedList**: 맨 앞/뒤 삽입·삭제가 빈번할 때 (큐, 덱 구현)
- **실무와 코테 모두 ArrayList가 기본 선택**. LinkedList는 특수한 경우에만 사용

### ArrayList의 동적 확장

```
초기 용량: 10
→ 가득 차면 1.5배로 새 배열 생성 → 기존 데이터 복사
→ 복사 비용은 O(n)이지만, 전체적으로 amortized O(1)
```

## 2. 예제

### 예제 1: ArrayList 기본 사용

```java
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        ArrayList<Integer> list = new ArrayList<>();
        list.add(10);       // [10]
        list.add(20);       // [10, 20]
        list.add(1, 15);    // [10, 15, 20] — 인덱스 1에 삽입

        System.out.println(list.get(0));    // 10 — O(1)
        System.out.println(list.size());    // 3

        list.remove(0);     // [15, 20] — 맨 앞 삭제, O(n)
    }
}
```

### 예제 2: LinkedList 기본 사용

```java
import java.util.LinkedList;

public class Main {
    public static void main(String[] args) {
        LinkedList<Integer> list = new LinkedList<>();
        list.addFirst(10);  // [10]
        list.addLast(30);   // [10, 30]
        list.add(1, 20);    // [10, 20, 30]

        System.out.println(list.getFirst()); // 10 — O(1)
        System.out.println(list.getLast());   // 30 — O(1)

        list.removeFirst(); // [20, 30] — O(1)
    }
}
```

### 예제 3: 성능 비교

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int n = 100000;

        // ArrayList 맨 앞 삽입
        List<Integer> arr = new ArrayList<>();
        long start = System.currentTimeMillis();
        for (int i = 0; i < n; i++) arr.add(0, i);
        System.out.println("ArrayList 맨앞 삽입: " + (System.currentTimeMillis() - start) + "ms");

        // LinkedList 맨 앞 삽입
        List<Integer> linked = new LinkedList<>();
        start = System.currentTimeMillis();
        for (int i = 0; i < n; i++) linked.add(0, i);
        System.out.println("LinkedList 맨앞 삽입: " + (System.currentTimeMillis() - start) + "ms");
    }
}
```

## 4. 이론 퀴즈

**Q1.** 배열에서 인덱스 접근이 O(1)인 이유는?

<details>
<summary>정답</summary>
배열은 연속된 메모리에 저장되므로, 시작 주소 + (인덱스 × 요소 크기)로 바로 계산하여 접근할 수 있다.
</details>

**Q2.** LinkedList에서 중간 삽입이 O(1)이 아닌 O(n)인 이유는?

<details>
<summary>정답</summary>
삽입 자체는 O(1)이지만, 삽입할 위치까지 순회하는 데 O(n)이 걸리기 때문이다.
</details>

**Q3.** ArrayList의 `add(e)`가 amortized O(1)이란?

<details>
<summary>정답</summary>
보통은 O(1)이지만, 배열이 가득 차면 새 배열을 만들어 복사하는 O(n) 작업이 발생한다. 그러나 확장 빈도가 점점 줄어들어 전체 n번 삽입에 대한 평균은 O(1)이 된다.
</details>

**Q4.** 캐시 친화성(cache locality)이 배열에 유리한 이유는?

<details>
<summary>정답</summary>
배열은 연속된 메모리에 저장되므로, CPU 캐시 라인에 인접 요소가 함께 적재된다. 링크드리스트는 노드가 메모리 곳곳에 흩어져 있어 캐시 미스가 빈번하다.
</details>

**Q5.** Java에서 기본 배열(`int[]`)과 ArrayList의 가장 큰 차이는?

<details>
<summary>정답</summary>
기본 배열은 크기가 고정이고 기본형을 저장한다. ArrayList는 크기가 동적이고 객체(래퍼 클래스)를 저장한다.
</details>

## 5. 함정 노트

### 1. "LinkedList가 삽입에 빠르다"는 반쪽 진실

LinkedList의 삽입이 O(1)인 것은 **삽입 위치의 노드를 이미 알고 있을 때**다. `list.add(50000, value)`처럼 인덱스로 접근하면 탐색에 O(n)이 걸린다.

### 2. ArrayList의 remove(0)

```java
list.remove(0); // O(n) — 나머지 요소가 전부 한 칸씩 앞으로 이동
```

맨 앞 삭제가 빈번하면 LinkedList나 ArrayDeque를 고려하라.

### 3. 오토박싱 비용

```java
ArrayList<Integer> list = new ArrayList<>();
list.add(5); // int 5 → Integer.valueOf(5) 오토박싱 발생
```

대량 데이터에서는 `int[]` 배열이 메모리·속도 모두 유리하다.

### 4. ArrayList 초기 용량 설정

```java
// 크기를 알면 미리 지정 → 확장·복사 비용 절감
ArrayList<Integer> list = new ArrayList<>(100000);
```
