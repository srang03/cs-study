<details>
<summary>정답</summary>

</details>

<details>
<summary>정답</summary>

</details>

## 1. 설명

### 이진 탐색 (Binary Search)

**정렬된 배열**에서 탐색 범위를 절반씩 줄여가며 목표값을 찾는 알고리즘.

- **전제 조건**: 배열이 반드시 정렬되어 있어야 한다
- **시간 복잡도**: O(log n)
- **선형 탐색 O(n) vs 이진 탐색 O(log n)**: n = 1,000,000일 때 100만 번 vs 약 20번

### 동작 원리

```
배열: [1, 3, 5, 7, 9, 11, 13, 15]
목표: 7

1단계: left=0, right=7, mid=3 → arr[3]=7 → 발견!
```

```
목표: 11

1단계: left=0, right=7, mid=3 → arr[3]=7 < 11 → left=4
2단계: left=4, right=7, mid=5 → arr[5]=11 → 발견!
```

### 기본 구현

```java
int binarySearch(int[] arr, int target) {
    int left = 0;
    int right = arr.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;  // 오버플로우 방지

        if (arr[mid] == target) {
            return mid;        // 찾음
        } else if (arr[mid] < target) {
            left = mid + 1;    // 오른쪽 절반 탐색
        } else {
            right = mid - 1;   // 왼쪽 절반 탐색
        }
    }

    return -1; // 못 찾음
}
```

### mid 계산의 오버플로우 방지

```java
// 위험: left + right가 int 범위를 초과할 수 있다
int mid = (left + right) / 2;

// 안전: 차이를 먼저 구해서 더한다
int mid = left + (right - left) / 2;
```

`left = 1,500,000,000`, `right = 2,000,000,000`이면 합이 int 범위(약 21억) 초과.

### Java 표준 라이브러리

```java
import java.util.Arrays;

int[] arr = {1, 3, 5, 7, 9};
int index = Arrays.binarySearch(arr, 7);  // 3 반환
int index2 = Arrays.binarySearch(arr, 4); // 음수 반환 (-(삽입점) - 1)
```

- 찾으면: 해당 인덱스 반환
- 못 찾으면: `-(삽입 위치) - 1` 반환 (음수)

### Lower Bound / Upper Bound

중복된 값이 있을 때, 특정 값의 **범위**를 찾는 변형.

#### Lower Bound: target 이상인 첫 번째 위치

```java
int lowerBound(int[] arr, int target) {
    int left = 0, right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] < target) {
            left = mid + 1;
        } else {
            right = mid;   // arr[mid] >= target이면 right를 좁힘
        }
    }
    return left;
}
```

#### Upper Bound: target 초과인 첫 번째 위치

```java
int upperBound(int[] arr, int target) {
    int left = 0, right = arr.length;

    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] <= target) {
            left = mid + 1;
        } else {
            right = mid;   // arr[mid] > target이면 right를 좁힘
        }
    }
    return left;
}
```

#### 활용: 특정 값의 개수

```
배열: [1, 2, 2, 2, 3, 4]
target = 2

lowerBound(arr, 2) = 1  (첫 번째 2의 위치)
upperBound(arr, 2) = 4  (마지막 2 다음 위치)
개수 = upperBound - lowerBound = 3
```

### 기본 이진 탐색 vs Lower/Upper Bound 비교

| 항목 | 기본 이진 탐색 | Lower/Upper Bound |
|------|---------------|-------------------|
| 초기 right | `arr.length - 1` | `arr.length` |
| 반복 조건 | `left <= right` | `left < right` |
| 찾으면 | 즉시 반환 | 범위를 계속 좁힘 |
| 용도 | 값의 존재 여부 | 값의 범위·개수 |

## 2. 예제

### 예제 1: 기본 이진 탐색

```java
import java.util.*;
import java.io.*;

public class Main {
    public static void main(String[] args) throws Exception {
        int[] arr = {2, 5, 8, 12, 16, 23, 38, 56, 72, 91};

        System.out.println(binarySearch(arr, 23));  // 5
        System.out.println(binarySearch(arr, 10));  // -1
    }

    static int binarySearch(int[] arr, int target) {
        int left = 0, right = arr.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (arr[mid] == target) return mid;
            else if (arr[mid] < target) left = mid + 1;
            else right = mid - 1;
        }
        return -1;
    }
}
```

### 예제 2: Lower/Upper Bound로 개수 세기

```java
public class Main {
    public static void main(String[] args) {
        int[] arr = {1, 2, 2, 2, 3, 3, 4, 5};

        int count2 = upperBound(arr, 2) - lowerBound(arr, 2);
        int count3 = upperBound(arr, 3) - lowerBound(arr, 3);
        System.out.println("2의 개수: " + count2); // 3
        System.out.println("3의 개수: " + count3); // 2
    }

    static int lowerBound(int[] arr, int target) {
        int left = 0, right = arr.length;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (arr[mid] < target) left = mid + 1;
            else right = mid;
        }
        return left;
    }

    static int upperBound(int[] arr, int target) {
        int left = 0, right = arr.length;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (arr[mid] <= target) left = mid + 1;
            else right = mid;
        }
        return left;
    }
}
```

### 예제 3: Arrays.binarySearch 사용

```java
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        int[] arr = {3, 1, 4, 1, 5, 9};
        Arrays.sort(arr);   // 반드시 정렬 먼저!
        // arr = {1, 1, 3, 4, 5, 9}

        int idx = Arrays.binarySearch(arr, 4);
        System.out.println(idx);  // 3

        int idx2 = Arrays.binarySearch(arr, 6);
        System.out.println(idx2); // -6 (-(삽입점5) - 1)
    }
}
```

## 4. 이론 퀴즈

**Q1.** 이진 탐색의 전제 조건은?

<details>
<summary>정답</summary>
배열이 정렬되어 있어야 한다. 정렬되지 않은 배열에서는 사용할 수 없다.
</details>

**Q2.** `mid = (left + right) / 2`가 위험한 이유는?

<details>
<summary>정답</summary>
left + right의 합이 int 최대값(약 21억)을 초과하면 오버플로우가 발생한다. `left + (right - left) / 2`로 방지한다.
</details>

**Q3.** n = 1,000,000일 때 이진 탐색의 최대 비교 횟수는?

<details>
<summary>정답</summary>
log₂(1,000,000) ≈ 20. 최대 약 20번의 비교로 찾을 수 있다.
</details>

**Q4.** `Arrays.binarySearch`에서 값이 없으면 반환하는 값은?

<details>
<summary>정답</summary>
-(삽입 위치) - 1. 예를 들어 삽입 위치가 3이면 -4를 반환한다.
</details>

**Q5.** Lower Bound와 Upper Bound의 차이를 한 줄로 설명하라.

<details>
<summary>정답</summary>
Lower Bound는 target 이상인 첫 위치, Upper Bound는 target 초과인 첫 위치.
</details>

## 5. 함정 노트

### 1. 정렬 안 하고 이진 탐색

```java
int[] arr = {5, 3, 1, 4, 2};
Arrays.binarySearch(arr, 3); // 정렬 안 됨 → 결과 보장 안 됨!

Arrays.sort(arr);            // 반드시 정렬 먼저
Arrays.binarySearch(arr, 3); // 올바른 결과
```

### 2. left <= right vs left < right

```java
// 기본 이진 탐색: left <= right (등호 포함)
// → 등호가 없으면 원소가 1개일 때 검사하지 못함

// Lower/Upper Bound: left < right (등호 미포함)
// → right가 답의 후보이므로 left == right가 되면 답이 확정
```

이 차이를 혼동하면 무한 루프 또는 원소 누락이 발생한다.

### 3. mid + 1 / mid - 1 실수

```java
// 틀림: left = mid (무한 루프 가능)
if (arr[mid] < target) left = mid;

// 맞음: left = mid + 1
if (arr[mid] < target) left = mid + 1;
```

### 4. long 타입에서의 이진 탐색

값의 범위가 클 때(예: 답을 이진 탐색으로 찾기) `int` 대신 `long`을 써야 한다.

```java
long left = 0, right = 2_000_000_000L;
long mid = left + (right - left) / 2;
```
