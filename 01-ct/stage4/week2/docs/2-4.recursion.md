## 1. 설명

### 재귀 (Recursion)

함수가 **자기 자신을 호출**하여 문제를 해결하는 기법.

```java
void f(int n) {
    if (n == 0) return;  // base case
    f(n - 1);            // recursive case
}
```

### 두 가지 핵심 요소

| 요소 | 설명 | 없으면? |
|------|------|---------|
| **Base case** (기저 조건) | 재귀를 멈추는 조건 | 무한 루프 → StackOverflowError |
| **Recursive case** (재귀 단계) | 문제를 더 작은 부분으로 나누어 자기 호출 | 문제가 해결되지 않음 |

### 스택 프레임 (Stack Frame)

함수가 호출될 때마다 **콜 스택**에 프레임이 쌓인다. 재귀는 스택 프레임이 쌓이는 구조다.

```
f(3) 호출
  → f(2) 호출 (f(3) 대기)
    → f(1) 호출 (f(2) 대기)
      → f(0) 호출 (f(1) 대기)
        → base case, return
      ← f(1) 재개, return
    ← f(2) 재개, return
  ← f(3) 재개, return
```

```
스택:
| f(0) |  ← top (가장 최근)
| f(1) |
| f(2) |
| f(3) |  ← bottom (최초 호출)
```

- Java 기본 스택 크기: 약 512KB ~ 1MB
- 재귀 깊이가 너무 깊으면: `StackOverflowError`

### 팩토리얼

```
n! = n × (n-1) × (n-2) × ... × 1
0! = 1 (base case)
```

```java
int factorial(int n) {
    if (n <= 1) return 1;       // base case
    return n * factorial(n - 1); // recursive case
}
```

호출 흐름:
```
factorial(4) = 4 × factorial(3)
             = 4 × 3 × factorial(2)
             = 4 × 3 × 2 × factorial(1)
             = 4 × 3 × 2 × 1
             = 24
```

### 피보나치 수열

```
F(0) = 0, F(1) = 1
F(n) = F(n-1) + F(n-2)
```

```java
int fibonacci(int n) {
    if (n <= 0) return 0;   // base case
    if (n == 1) return 1;   // base case
    return fibonacci(n - 1) + fibonacci(n - 2); // recursive case
}
```

**문제점: 중복 계산**

```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2) ← 중복
│   │   └── fib(1)
│   └── fib(2) ← 중복
└── fib(3) ← 중복
    ├── fib(2) ← 중복
    └── fib(1)
```

- 시간 복잡도: **O(2^n)** — 지수적으로 폭발
- n=40이면 약 10억 번 호출 → 시간초과

### 재귀 → 반복 변환

모든 재귀는 반복문으로 변환 가능하다.

#### 팩토리얼: 재귀 → 반복

```java
// 재귀
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

// 반복
int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; i++) {
        result *= i;
    }
    return result;
}
```

#### 피보나치: 재귀 → 반복 (DP)

```java
// 재귀 O(2^n) — 느림
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

// 반복 O(n) — 빠름
int fib(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int cur = prev1 + prev2;
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
```

### 재귀를 써야 할 때

| 상황 | 적합성 |
|------|--------|
| 트리 순회 | 재귀가 자연스러움 |
| 분할 정복 (병합 정렬, 퀵 정렬) | 재귀 필수 |
| 백트래킹 (N-Queen, 순열) | 재귀 필수 |
| 단순 반복 (합계, 팩토리얼) | 반복문이 효율적 |
| 깊이가 매우 깊은 경우 | 반복문으로 변환 |

## 2. 예제

### 예제 1: 팩토리얼

```java
public class Main {
    public static void main(String[] args) {
        System.out.println(factorial(5)); // 120
        System.out.println(factorial(0)); // 1
    }

    static int factorial(int n) {
        if (n <= 1) return 1;
        return n * factorial(n - 1);
    }
}
```

### 예제 2: 1부터 N까지 합 (재귀)

```java
public class Main {
    public static void main(String[] args) {
        System.out.println(sum(10)); // 55
    }

    static int sum(int n) {
        if (n == 1) return 1;
        return n + sum(n - 1);
    }
}
```

### 예제 3: 문자열 뒤집기 (재귀)

```java
public class Main {
    public static void main(String[] args) {
        System.out.println(reverse("hello")); // "olleh"
    }

    static String reverse(String s) {
        if (s.length() <= 1) return s;
        return reverse(s.substring(1)) + s.charAt(0);
    }
}
```

### 예제 4: 피보나치 비교

```java
public class Main {
    static int callCount = 0;

    public static void main(String[] args) {
        callCount = 0;
        fibRecursive(30);
        System.out.println("재귀 호출 횟수: " + callCount); // 약 270만

        callCount = 0;
        fibIterative(30);
        System.out.println("반복 연산 횟수: " + callCount); // 29
    }

    static int fibRecursive(int n) {
        callCount++;
        if (n <= 1) return n;
        return fibRecursive(n - 1) + fibRecursive(n - 2);
    }

    static int fibIterative(int n) {
        if (n <= 1) return n;
        int a = 0, b = 1;
        for (int i = 2; i <= n; i++) {
            callCount++;
            int c = a + b;
            a = b;
            b = c;
        }
        return b;
    }
}
```

## 4. 이론 퀴즈

**Q1.** Base case가 없으면 어떤 에러가 발생하는가?

<details>
<summary>정답</summary>
StackOverflowError. 재귀가 무한히 호출되어 콜 스택이 가득 찬다.
</details>

**Q2.** 재귀 피보나치의 시간 복잡도가 O(2^n)인 이유는?

<details>
<summary>정답</summary>
각 호출이 2개의 하위 호출을 만들고, 중복 계산이 지수적으로 증가하기 때문이다. 호출 트리의 노드 수가 약 2^n개.
</details>

**Q3.** 재귀를 반복문으로 변환할 수 없는 경우가 있는가?

<details>
<summary>정답</summary>
없다. 모든 재귀는 이론적으로 반복문(+ 명시적 스택)으로 변환 가능하다. 다만 트리 순회나 백트래킹 등은 재귀가 훨씬 직관적이다.
</details>

**Q4.** 꼬리 재귀(tail recursion)란?

<details>
<summary>정답</summary>
재귀 호출이 함수의 마지막 연산인 경우. 일부 언어에서는 컴파일러가 반복문으로 최적화한다(TCO). Java는 꼬리 재귀 최적화를 지원하지 않는다.
</details>

**Q5.** Java에서 재귀 깊이의 실질적 한계는?

<details>
<summary>정답</summary>
기본 스택 크기(약 512KB~1MB)에서 약 수천~1만 번 정도. -Xss 옵션으로 스택 크기를 늘릴 수 있지만, 코테에서는 보통 기본값을 사용한다.
</details>

## 5. 함정 노트

### 1. Base case 누락

```java
// 틀림: base case 없음 → StackOverflowError
int factorial(int n) {
    return n * factorial(n - 1); // 영원히 호출
}

// 맞음
int factorial(int n) {
    if (n <= 1) return 1;       // base case 필수
    return n * factorial(n - 1);
}
```

### 2. 재귀 피보나치로 큰 수 계산

```java
fibonacci(50); // O(2^50) ≈ 10^15 → 절대 끝나지 않음
```

피보나치는 반드시 반복문 또는 메모이제이션으로 풀어야 한다.

### 3. 반환값 무시

```java
// 틀림: 재귀 호출의 반환값을 사용하지 않음
int sum(int n) {
    if (n == 1) return 1;
    sum(n - 1);       // 반환값 버림!
    return n;
}

// 맞음
int sum(int n) {
    if (n == 1) return 1;
    return n + sum(n - 1);  // 반환값 사용
}
```

### 4. int 오버플로우

```java
factorial(13); // 6,227,020,800 → int 범위 초과!
// 큰 팩토리얼은 long 또는 BigInteger 사용
```

### 5. 재귀 vs 스택 깊이

```java
// N = 100,000이면 재귀 깊이 10만 → StackOverflowError
// 반복문으로 바꾸면 스택 문제 없음
```

코테에서 N이 크면 재귀를 반복으로 변환하거나, 스택을 명시적으로 사용하라.

- **난이도**: Bronze I

- **난이도**: Silver I
