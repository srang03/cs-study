## 1. 설명

### 단방향 링크드리스트 (Singly Linked List)

각 노드가 **데이터**와 **다음 노드 참조(next)**를 갖는다.

```
head → [10|→] → [20|→] → [30|null]
```

### Node 클래스

```java
class Node {
    int data;
    Node next;

    Node(int data) {
        this.data = data;
        this.next = null;
    }
}
```

### 주요 연산

#### 1. 맨 앞 삽입 (addFirst) — O(1)

```java
void addFirst(int data) {
    Node newNode = new Node(data);
    newNode.next = head;
    head = newNode;
    size++;
}
```

```
기존:  head → [10|→] → [20|→] → [30|null]
삽입:  head → [5|→] → [10|→] → [20|→] → [30|null]
```

#### 2. 맨 뒤 삽입 (addLast) — O(n) (tail 없을 때)

```java
void addLast(int data) {
    Node newNode = new Node(data);
    if (head == null) {
        head = newNode;
    } else {
        Node cur = head;
        while (cur.next != null) {
            cur = cur.next;
        }
        cur.next = newNode;
    }
    size++;
}
```

tail 포인터를 유지하면 O(1)로 가능:

```java
void addLast(int data) {
    Node newNode = new Node(data);
    if (head == null) {
        head = newNode;
        tail = newNode;
    } else {
        tail.next = newNode;
        tail = newNode;
    }
    size++;
}
```

#### 3. 맨 앞 삭제 (removeFirst) — O(1)

```java
int removeFirst() {
    if (head == null) throw new NoSuchElementException();
    int data = head.data;
    head = head.next;
    size--;
    return data;
}
```

#### 4. 특정 값 삭제 (remove) — O(n)

```java
boolean remove(int data) {
    if (head == null) return false;

    // 헤드가 삭제 대상인 경우
    if (head.data == data) {
        head = head.next;
        size--;
        return true;
    }

    Node cur = head;
    while (cur.next != null) {
        if (cur.next.data == data) {
            cur.next = cur.next.next;
            size--;
            return true;
        }
        cur = cur.next;
    }
    return false;
}
```

```
기존:  [10|→] → [20|→] → [30|null]
20삭제: [10|→] →→→→→→→→→ [30|null]  (20 노드의 참조를 건너뜀)
```

#### 5. 탐색 (search) — O(n)

```java
boolean contains(int data) {
    Node cur = head;
    while (cur != null) {
        if (cur.data == data) return true;
        cur = cur.next;
    }
    return false;
}
```

#### 6. 출력 (display) — O(n)

```java
void display() {
    Node cur = head;
    while (cur != null) {
        System.out.print(cur.data + " → ");
        cur = cur.next;
    }
    System.out.println("null");
}
```

### 전체 구조

```java
class SinglyLinkedList {
    private Node head;
    private int size;

    SinglyLinkedList() {
        this.head = null;
        this.size = 0;
    }

    // addFirst, addLast, removeFirst, remove, contains, display ...

    int size() { return size; }
    boolean isEmpty() { return size == 0; }
}
```

## 2. 예제

### 예제 1: 직접 구현한 리스트 사용

```java
public class Main {
    public static void main(String[] args) {
        SinglyLinkedList list = new SinglyLinkedList();
        list.addFirst(30);   // 30 → null
        list.addFirst(20);   // 20 → 30 → null
        list.addFirst(10);   // 10 → 20 → 30 → null

        list.display();      // 10 → 20 → 30 → null
        System.out.println(list.contains(20)); // true
        System.out.println(list.size());       // 3

        list.remove(20);     // 10 → 30 → null
        list.display();
    }
}
```

### 예제 2: 특정 인덱스 접근

```java
int get(int index) {
    if (index < 0 || index >= size) throw new IndexOutOfBoundsException();
    Node cur = head;
    for (int i = 0; i < index; i++) {
        cur = cur.next;
    }
    return cur.data;
}
```

인덱스 0이면 바로 head, 인덱스 k이면 k번 이동 → **O(n)**.

### 예제 3: 리스트 뒤집기 (in-place)

```java
void reverse() {
    Node prev = null;
    Node cur = head;
    while (cur != null) {
        Node next = cur.next;  // 다음 노드 저장
        cur.next = prev;       // 방향 반전
        prev = cur;            // prev 전진
        cur = next;            // cur 전진
    }
    head = prev;
}
```

```
원래:   10 → 20 → 30 → null
반전:   null ← 10 ← 20 ← 30
결과:   30 → 20 → 10 → null
```

이 패턴은 면접과 코테에서 매우 자주 출제된다.

## 4. 이론 퀴즈

**Q1.** 단방향 링크드리스트에서 맨 뒤 삭제의 시간 복잡도는?

<details>
<summary>정답</summary>
O(n). 맨 뒤 노드의 이전 노드를 찾으려면 처음부터 순회해야 한다. (이중 연결 리스트라면 O(1))
</details>

**Q2.** head만 관리하는 리스트에서 `addLast`의 복잡도를 O(1)로 만들려면?

<details>
<summary>정답</summary>
tail 포인터를 추가로 유지하면 된다. 삽입 시 tail.next = newNode; tail = newNode;
</details>

**Q3.** 링크드리스트 뒤집기에서 `prev`, `cur`, `next` 세 변수가 모두 필요한 이유는?

<details>
<summary>정답</summary>
cur.next를 prev로 바꾸면 원래 다음 노드를 잃어버린다. 따라서 방향을 바꾸기 전에 next에 원래 다음 노드를 저장해야 한다.
</details>

**Q4.** Java의 `LinkedList`는 단방향인가 이중 연결인가?

<details>
<summary>정답</summary>
이중 연결 리스트(Doubly Linked List). 각 노드가 prev, next를 모두 갖고 있어 양방향 탐색이 가능하다.
</details>

## 5. 함정 노트

### 1. null 체크 누락

```java
// head가 null일 때 head.data 접근 → NullPointerException
void removeFirst() {
    int data = head.data; // head가 null이면 NPE!
    head = head.next;
}
```

**항상 빈 리스트(head == null)를 먼저 처리하라.**

### 2. 삭제 시 이전 노드 추적

```java
// 틀림: 현재 노드만으로는 이전 노드의 next를 바꿀 수 없다
Node cur = head;
while (cur != null) {
    if (cur.data == target) {
        // cur를 어떻게 제거? 이전 노드가 없다!
    }
    cur = cur.next;
}

// 맞음: cur.next를 검사하여 cur가 이전 노드 역할을 한다
while (cur.next != null) {
    if (cur.next.data == target) {
        cur.next = cur.next.next;
        break;
    }
    cur = cur.next;
}
```

### 3. size 관리 누락

삽입/삭제할 때 `size++`, `size--`를 빠뜨리면 `size()` 값이 틀어진다.

### 4. 순환 참조

```java
// 실수로 노드가 자기 자신을 가리키면 무한 루프
node.next = node; // 절대 하면 안 됨
```
