## 1. 설명

### 왜 정렬을 직접 구현하는가

코테에서는 `Arrays.sort()`를 쓰면 되지만, **정렬 알고리즘의 동작 원리와 시간 복잡도를 이해**해야 문제에 맞는 접근법을 선택할 수 있다. 또한 면접에서 자주 물어본다.

### 버블 정렬 (Bubble Sort)

인접한 두 원소를 비교하여 교환. 한 패스마다 가장 큰 값이 맨 뒤로 이동한다.

```java
void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int tmp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = tmp;
            }
        }
    }
}
```

- **시간 복잡도**: O(n^2) (최선/평균/최악 모두)
- **공간 복잡도**: O(1) (제자리 정렬)
- **안정 정렬**: O (같은 값의 상대 순서 유지)

### 선택 정렬 (Selection Sort)

매 패스에서 최소값을 찾아 앞으로 이동.

```java
void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) minIdx = j;
        }
        int tmp = arr[i];
        arr[i] = arr[minIdx];
        arr[minIdx] = tmp;
    }
}
```

- **시간 복잡도**: O(n^2) (최선/평균/최악 모두)
- **공간 복잡도**: O(1)
- **안정 정렬**: X (떨어진 원소를 교환하므로 순서가 바뀔 수 있음)

### 삽입 정렬 (Insertion Sort)

현재 원소를 이미 정렬된 부분의 올바른 위치에 삽입.

```java
void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
}
```

- **시간 복잡도**: 최선 O(n) (이미 정렬), 평균/최악 O(n^2)
- **공간 복잡도**: O(1)
- **안정 정렬**: O

### 정렬 비교 요약

| 정렬 | 최선 | 평균 | 최악 | 안정 |
|------|------|------|------|:----:|
| 버블 | O(n^2) | O(n^2) | O(n^2) | O |
| 선택 | O(n^2) | O(n^2) | O(n^2) | X |
| 삽입 | O(n) | O(n^2) | O(n^2) | O |

### Arrays.sort / Collections.sort

실전 코테에서는 직접 구현 대신 라이브러리를 사용한다.

```java
import java.util.Arrays;
import java.util.Collections;
import java.util.List;
import java.util.ArrayList;

// 원시 타입 배열: Dual-Pivot QuickSort, O(n log n)
int[] arr = {5, 3, 1, 4, 2};
Arrays.sort(arr);

// 객체 배열 / List: TimSort, O(n log n), 안정 정렬
Integer[] arr2 = {5, 3, 1, 4, 2};
Arrays.sort(arr2);

List<Integer> list = new ArrayList<>(List.of(5, 3, 1, 4, 2));
Collections.sort(list);
```

### Comparable / Comparator

객체를 정렬하려면 **비교 기준**을 정의해야 한다.

**Comparable** — 클래스 자체에 기본 정렬 기준 내장:

```java
class Student implements Comparable<Student> {
    String name;
    int age;

    @Override
    public int compareTo(Student o) {
        return this.age - o.age;  // 나이 오름차순
    }
}

List<Student> students = new ArrayList<>();
Collections.sort(students);  // compareTo 기준으로 정렬
```

**Comparator** — 외부에서 정렬 기준 제공 (람다 중심):

```java
// 람다식 (권장)
students.sort((a, b) -> a.age - b.age);               // 나이 오름차순
students.sort((a, b) -> a.name.compareTo(b.name));     // 이름 오름차순

// 2차원 배열 정렬 (코테 빈출)
int[][] points = {{3, 4}, {1, 1}, {1, 3}};
Arrays.sort(points, (a, b) -> {
    if (a[0] == b[0]) return a[1] - b[1];  // x 같으면 y 오름차순
    return a[0] - b[0];                     // x 오름차순
});
```

**익명 클래스** (참고용):

```java
// 람다 이전 방식 — 동작은 같지만 코드가 장황
students.sort(new Comparator<Student>() {
    @Override
    public int compare(Student a, Student b) {
        return a.age - b.age;
    }
});
```

> 코테에서는 거의 항상 **람다식**을 쓴다. 익명 클래스는 람다의 원리를 이해하는 용도로만 알아두면 된다.

## 2. 예제

### 예제 1: 버블 정렬 동작 확인

```java
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        int[] arr = {5, 3, 8, 1, 2};

        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    int tmp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = tmp;
                }
            }
            System.out.println("패스 " + (i + 1) + ": " + Arrays.toString(arr));
        }
        // 패스 1: [3, 5, 1, 2, 8]
        // 패스 2: [3, 1, 2, 5, 8]
        // 패스 3: [1, 2, 3, 5, 8]
        // 패스 4: [1, 2, 3, 5, 8]
    }
}
```

### 예제 2: Comparator 람다로 다중 조건 정렬

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        // (x, y) 좌표를 x 오름차순, x 같으면 y 오름차순 정렬
        int[][] points = {{3, 4}, {1, 1}, {1, 3}, {2, 2}};

        Arrays.sort(points, (a, b) -> {
            if (a[0] == b[0]) return a[1] - b[1];
            return a[0] - b[0];
        });

        for (int[] p : points) {
            System.out.println(p[0] + " " + p[1]);
        }
        // 1 1
        // 1 3
        // 2 2
        // 3 4
    }
}
```

### 예제 3: 문자열 배열을 길이순 정렬

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        String[] words = {"banana", "apple", "fig", "cherry"};

        Arrays.sort(words, (a, b) -> a.length() - b.length());

        System.out.println(Arrays.toString(words));
        // [fig, apple, banana, cherry]
    }
}
```

## 4. 이론 퀴즈

**Q1.** 버블·선택·삽입 정렬 중 이미 정렬된 배열에서 가장 빠른 것은?

<details>
<summary>정답</summary>
삽입 정렬. 이미 정렬된 경우 내부 while 루프를 돌지 않아 O(n)이다. 나머지는 여전히 O(n^2).
</details>

**Q2.** `Arrays.sort()`에 람다를 넘기려면 배열의 타입이 어떠해야 하는가?

<details>
<summary>정답</summary>
객체 배열(Integer[], String[], int[][] 등)이어야 한다. 원시 타입 배열(int[])에는 Comparator를 넘길 수 없다.
</details>

**Q3.** `(a, b) -> a - b`와 `(a, b) -> b - a`의 차이는?

<details>
<summary>정답</summary>
a - b는 오름차순, b - a는 내림차순이다. 비교 결과가 음수면 a가 앞, 양수면 b가 앞에 온다.
</details>

**Q4.** 안정 정렬(Stable Sort)이란?

<details>
<summary>정답</summary>
같은 값을 가진 원소의 상대적 순서가 정렬 후에도 유지되는 정렬. 예: 나이순 정렬 시, 같은 나이의 사람들은 입력 순서가 유지된다. TimSort(Arrays.sort 객체)는 안정, QuickSort(Arrays.sort 원시)는 불안정.
</details>

## 5. 함정 노트

### 1. int 뺄셈 비교에서 오버플로우

```java
// 위험: a = Integer.MIN_VALUE, b = 1이면 오버플로우
(a, b) -> a - b

// 안전한 대안
(a, b) -> Integer.compare(a, b)
```

코테 범위 내에서는 보통 문제없지만, 값이 매우 클 수 있는 경우 `Integer.compare` 사용.

### 2. 원시 배열에 Comparator 사용 불가

```java
int[] arr = {3, 1, 2};
// Arrays.sort(arr, (a, b) -> b - a);  // 컴파일 에러!

// 해결: Integer[]로 변환하거나 정렬 후 역순
Integer[] arr2 = {3, 1, 2};
Arrays.sort(arr2, (a, b) -> b - a);  // OK
```

### 3. 선택 정렬은 불안정

같은 값의 순서가 보장되지 않는다. 문제에서 "같은 값이면 입력 순서 유지"를 요구하면 안정 정렬을 써야 한다.

### 4. StringBuilder 출력 안 쓰면 시간 초과

정렬 문제에서 N이 크면 `System.out.println()`을 N번 호출하면 느리다.

```java
StringBuilder sb = new StringBuilder();
for (int num : arr) {
    sb.append(num).append('\n');
}
System.out.print(sb);
```
