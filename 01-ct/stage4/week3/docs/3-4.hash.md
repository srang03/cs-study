## 리뷰 미션 (3-3 큐)

**R1.** 큐에서 `poll()`과 `peek()`의 차이는?

<details>
<summary>정답</summary>
poll()은 앞의 원소를 제거하고 반환한다. peek()은 앞의 원소를 제거하지 않고 반환만 한다. 빈 큐에서 둘 다 null을 반환한다.
</details>

**R2.** 아래 코드 실행 후 출력은?

```java
Queue<Integer> q = new ArrayDeque<>();
q.offer(1);
q.offer(2);
q.offer(3);
q.poll();
q.offer(q.poll());
System.out.println(q.peek());
```

<details>
<summary>정답</summary>
3. offer(1,2,3) → poll()로 1 제거 → poll()로 2를 꺼내서 offer()로 다시 넣음. 큐: [3, 2]. peek()은 앞인 3.
</details>

## 1. 설명

### 해시란

**키(Key)를 고정 크기의 값(해시값)으로 변환하여, 데이터를 O(1)에 저장·조회하는 기법**이다.

stage3에서 `HashMap`과 `HashSet`을 사용법 중심으로 배웠다. 이 단원에서는 **내부 원리**를 이해한다.

### 해시 함수 (Hash Function)

임의의 입력을 고정 범위의 정수로 변환하는 함수.

```
"apple"  → hashCode() → 93029210  → % 16 → 10 (인덱스)
"banana" → hashCode() → 669541536 → % 16 → 0  (인덱스)
```

Java에서는 `Object.hashCode()` 메서드가 해시 함수 역할을 한다.

### 해시 테이블 (Hash Table)

배열 + 해시 함수로 구성. 키의 해시값을 배열 인덱스로 사용하여 O(1) 접근.

```
인덱스:  [0]       [1]  [2]  ...  [10]       ...  [15]
값:      "banana"  -    -    ...  "apple"    ...  -
```

### O(1) 조회의 원리

1. 키에 해시 함수를 적용하여 인덱스 계산
2. 해당 인덱스에 직접 접근 (배열 접근은 O(1))
3. 따라서 **검색, 삽입, 삭제 모두 평균 O(1)**

> 반면 배열에서 값을 찾으려면 O(n) 순회가 필요하고, 정렬된 배열에서 이진탐색을 해도 O(log n)이다.

### 해시 충돌 (Hash Collision)

서로 다른 키가 같은 인덱스에 매핑되는 경우. 반드시 발생할 수 있다 (비둘기집 원리).

```
"apple"  → % 16 → 10
"grape"  → % 16 → 10  ← 충돌!
```

### 충돌 처리 방법

**1. 체이닝 (Chaining)** — Java HashMap이 사용하는 방식

같은 인덱스에 연결 리스트(또는 트리)로 여러 값을 저장.

```
[10] → ("apple", 3) → ("grape", 7) → null
```

- 장점: 구현이 간단, 삽입 빠름
- 단점: 충돌이 많으면 리스트가 길어져 O(n)에 가까워짐
- Java 8+: 체인 길이가 8 이상이면 연결 리스트 → 레드블랙 트리로 변환 (O(log n))

**2. 오픈 어드레싱 (Open Addressing)**

충돌 시 다른 빈 인덱스를 찾아 저장.

```
"apple" → 인덱스 10에 저장
"grape" → 인덱스 10 충돌 → 11에 저장 (선형 탐사)
```

- 선형 탐사: 한 칸씩 이동
- 이차 탐사: 1, 4, 9, ... 칸씩 이동
- 이중 해싱: 두 번째 해시 함수로 이동 폭 결정

### HashMap의 내부 구조 (stage3 복습 + 심화)

```java
// stage3에서 배운 사용법
Map<String, Integer> map = new HashMap<>();
map.put("apple", 3);
map.get("apple");  // 3, O(1)
```

내부적으로:
1. `"apple".hashCode()` 호출 → 해시값 계산
2. 해시값을 배열 크기로 나눈 나머지 → 인덱스 결정
3. 해당 인덱스의 버킷에 (키, 값) 쌍 저장
4. 충돌 시 체이닝으로 처리

### HashSet의 내부 구조

`HashSet`은 내부적으로 `HashMap`을 사용한다. 값은 더미 객체, 키만 저장.

```java
// HashSet.add("apple") 내부:
// map.put("apple", PRESENT)  // PRESENT = 더미 상수
```

### 코테 핵심 패턴 1: 빈도수 세기

배열이나 문자열에서 각 원소의 출현 횟수를 세는 패턴.

```java
int[] arr = {1, 2, 2, 3, 3, 3};
Map<Integer, Integer> freq = new HashMap<>();
for (int num : arr) {
    freq.put(num, freq.getOrDefault(num, 0) + 1);
}
// {1=1, 2=2, 3=3}
```

### 코테 핵심 패턴 2: Two-Sum

배열에서 합이 target인 두 수를 찾는 패턴. O(n^2) 브루트포스를 O(n)으로 줄인다.

```java
int[] arr = {2, 7, 11, 15};
int target = 9;

Map<Integer, Integer> map = new HashMap<>();  // 값 → 인덱스
for (int i = 0; i < arr.length; i++) {
    int complement = target - arr[i];
    if (map.containsKey(complement)) {
        // map.get(complement)와 i가 답
    }
    map.put(arr[i], i);
}
```

핵심 아이디어: "target - 현재값"이 이미 map에 있는가?

## 2. 예제

### 예제 1: 빈도수 세기

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int[] cards = {6, 3, 2, 10, 10, 10, -10, -10, 7, 3};

        Map<Integer, Integer> freq = new HashMap<>();
        for (int card : cards) {
            freq.put(card, freq.getOrDefault(card, 0) + 1);
        }

        // 특정 수의 빈도 조회
        int[] queries = {10, 9, -10};
        StringBuilder sb = new StringBuilder();
        for (int q : queries) {
            sb.append(freq.getOrDefault(q, 0)).append(' ');
        }
        System.out.println(sb);  // 3 0 2
    }
}
```

### 예제 2: Two-Sum

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int[] nums = {2, 7, 11, 15};
        int target = 9;

        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement)) {
                System.out.println("인덱스: " + map.get(complement) + ", " + i);
                // 인덱스: 0, 1  (nums[0]=2, nums[1]=7, 합=9)
                break;
            }
            map.put(nums[i], i);
        }
    }
}
```

### 예제 3: 존재 여부 빠른 판별 (HashSet)

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        // 회사에 출근한 사람 목록에서 아직 퇴근하지 않은 사람 찾기
        String[] logs = {
            "enter kim", "enter lee", "leave kim",
            "enter park", "leave lee", "leave park"
        };

        Set<String> inOffice = new HashSet<>();
        for (String log : logs) {
            String[] parts = log.split(" ");
            if (parts[0].equals("enter")) {
                inOffice.add(parts[1]);
            } else {
                inOffice.remove(parts[1]);
            }
        }

        System.out.println(inOffice);  // [] (모두 퇴근)
    }
}
```

## 4. 이론 퀴즈

**Q1.** HashMap의 get(), put()의 평균/최악 시간 복잡도는?

<details>
<summary>정답</summary>
평균 O(1), 최악 O(n). 최악은 모든 키가 같은 인덱스에 충돌하는 경우. 실전에서는 O(1)로 간주한다.
</details>

**Q2.** 해시 충돌이란 무엇이며, Java HashMap은 어떻게 처리하는가?

<details>
<summary>정답</summary>
서로 다른 키가 같은 해시 인덱스에 매핑되는 현상. Java HashMap은 체이닝(같은 인덱스에 연결 리스트/트리로 연결)으로 처리한다. Java 8+에서는 체인 길이가 8 이상이면 레드블랙 트리로 변환한다.
</details>

**Q3.** HashSet은 내부적으로 어떤 자료구조를 사용하는가?

<details>
<summary>정답</summary>
HashMap을 사용한다. 키에 원소를 저장하고, 값은 더미 상수(PRESENT)를 넣는다.
</details>

**Q4.** Two-Sum 문제에서 HashMap을 사용하면 시간 복잡도가 어떻게 개선되는가?

<details>
<summary>정답</summary>
O(n^2) 이중 반복문 대신 O(n) 한 번 순회로 해결. 각 원소에 대해 "target - 현재값"이 map에 있는지 O(1)로 확인하기 때문이다.
</details>

## 5. 함정 노트

### 1. hashCode와 equals의 관계

HashMap/HashSet에서 같은 객체로 인식되려면 `hashCode()`와 `equals()` **둘 다** 일관되게 오버라이드해야 한다.

```java
// 커스텀 클래스를 Map의 키로 쓸 때
class Point {
    int x, y;
    // hashCode()만 오버라이드하고 equals() 안 하면 → 같은 좌표도 다른 키로 취급
    // equals()만 오버라이드하고 hashCode() 안 하면 → 같은 좌표가 다른 버킷에 저장
}
```

> 코테에서는 보통 기본 타입(Integer, String)을 키로 쓰므로 걱정할 필요 없다. 하지만 원리는 알아두자.

### 2. getOrDefault 안 쓰면 NPE

```java
Map<Integer, Integer> map = new HashMap<>();
// int count = map.get(5);  // NullPointerException!

// 올바른 사용
int count = map.getOrDefault(5, 0);
```

### 3. HashMap의 순서 보장 없음

```java
Map<String, Integer> map = new HashMap<>();
map.put("c", 3);
map.put("a", 1);
map.put("b", 2);
// 순회 순서 ≠ 삽입 순서 (보장 안 됨)

// 삽입 순서가 필요하면 LinkedHashMap
// 키 정렬이 필요하면 TreeMap (O(log n))
```

### 4. 빈도수 세기에서 merge 활용 (참고)

```java
// getOrDefault 패턴
freq.put(key, freq.getOrDefault(key, 0) + 1);

// merge 패턴 (같은 동작, 더 간결)
freq.merge(key, 1, Integer::sum);
```
