## 1. 설명

### 스택이란

**LIFO** (Last In, First Out) — 가장 나중에 넣은 것을 가장 먼저 꺼내는 자료구조.

접시 쌓기를 떠올리면 된다. 맨 위에 올리고, 맨 위에서 꺼낸다.

```
push(1) → [1]
push(2) → [1, 2]
push(3) → [1, 2, 3]
pop()   → 3 반환, [1, 2]
peek()  → 2 반환 (꺼내지 않고 확인만)
```

### 주요 연산

| 연산 | 동작 | 시간 복잡도 |
|------|------|:-----------:|
| `push(e)` | 맨 위에 추가 | O(1) |
| `pop()` | 맨 위 제거 및 반환 | O(1) |
| `peek()` | 맨 위 확인 (제거 안 함) | O(1) |
| `isEmpty()` | 비어 있는지 확인 | O(1) |
| `size()` | 원소 수 | O(1) |

### ArrayDeque를 스택으로 사용

Java의 `Stack` 클래스는 **쓰지 않는다**. `Vector` 기반으로 모든 메서드가 동기화(synchronized)되어 느리고, 레거시 클래스이다.

대신 `ArrayDeque`를 스택으로 사용한다.

```java
import java.util.ArrayDeque;
import java.util.Deque;

Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);       // 맨 위에 추가
stack.push(2);
stack.push(3);

stack.peek();        // 3 (제거 안 함)
stack.pop();         // 3 (제거)
stack.isEmpty();     // false
stack.size();        // 2
```

> `Deque` 인터페이스로 선언하고 `ArrayDeque`로 생성하는 것이 관례.

### Stack 클래스를 안 쓰는 이유

| 비교 | Stack | ArrayDeque |
|------|-------|-----------|
| 상속 | Vector (동기화 오버헤드) | 동기화 없음 (빠름) |
| 설계 | index 접근 가능 (스택 원칙 위반) | 양 끝 접근만 허용 |
| 성능 | 느림 | 빠름 |
| 권장 | X (레거시) | O (Java 공식 권장) |

### 괄호 매칭 패턴

스택의 대표적 활용. 여는 괄호를 push, 닫는 괄호가 나오면 pop하여 짝이 맞는지 확인.

```java
boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(') {
            stack.push(c);
        } else if (c == ')') {
            if (stack.isEmpty()) return false;
            stack.pop();
        }
    }
    return stack.isEmpty();
}
```

### 스택이 쓰이는 곳

- 괄호 검증
- 후위 표기식 계산
- DFS (깊이 우선 탐색) — week4에서 학습
- 재귀 호출의 내부 구조 (콜 스택)
- 문자열 뒤집기

## 2. 예제

### 예제 1: 기본 스택 사용

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Deque<Integer> stack = new ArrayDeque<>();

        stack.push(10);
        stack.push(20);
        stack.push(30);

        System.out.println(stack.peek());  // 30
        System.out.println(stack.pop());   // 30
        System.out.println(stack.pop());   // 20
        System.out.println(stack.size());  // 1
        System.out.println(stack.isEmpty()); // false
    }
}
```

### 예제 2: 괄호 매칭

```java
import java.util.*;

public class Main {
    static boolean isValid(String s) {
        Deque<Character> stack = new ArrayDeque<>();
        for (char c : s.toCharArray()) {
            if (c == '(') {
                stack.push(c);
            } else if (c == ')') {
                if (stack.isEmpty()) return false;
                stack.pop();
            }
        }
        return stack.isEmpty();
    }

    public static void main(String[] args) {
        System.out.println(isValid("(())()"));   // true
        System.out.println(isValid("(()"  ));    // false
        System.out.println(isValid(")("   ));    // false
    }
}
```

### 예제 3: 문자열 뒤집기

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        String input = "hello";
        Deque<Character> stack = new ArrayDeque<>();

        for (char c : input.toCharArray()) {
            stack.push(c);
        }

        StringBuilder sb = new StringBuilder();
        while (!stack.isEmpty()) {
            sb.append(stack.pop());
        }
        System.out.println(sb);  // olleh
    }
}
```

## 4. 이론 퀴즈

**Q1.** 스택에서 LIFO란 무엇인가?

<details>
<summary>정답</summary>
Last In, First Out. 가장 나중에 넣은 원소가 가장 먼저 나온다.
</details>

**Q2.** `Stack` 클래스 대신 `ArrayDeque`를 쓰는 이유는?

<details>
<summary>정답</summary>
Stack은 Vector 기반으로 모든 메서드가 synchronized(동기화)되어 불필요한 성능 오버헤드가 있다. 또한 인덱스 접근이 가능해 스택 원칙을 위반할 수 있다. ArrayDeque는 동기화 없이 빠르고, 양 끝 접근만 허용한다.
</details>

**Q3.** 빈 스택에서 `pop()`을 호출하면?

<details>
<summary>정답</summary>
ArrayDeque의 pop()은 NoSuchElementException을 던진다. 반드시 isEmpty()로 확인 후 호출해야 한다.
</details>

**Q4.** 괄호 매칭 알고리즘의 시간/공간 복잡도는?

<details>
<summary>정답</summary>
시간 O(n) — 문자열을 한 번 순회. 공간 O(n) — 최악의 경우 모든 문자가 여는 괄호.
</details>

## 5. 함정 노트

### 1. 빈 스택에서 pop/peek 호출

```java
Deque<Integer> stack = new ArrayDeque<>();
// stack.pop();   // NoSuchElementException!
// stack.peek();  // null 반환 (예외 아님!) — 주의

if (!stack.isEmpty()) {
    stack.pop();  // 안전
}
```

> `pop()`은 예외, `peek()`은 null 반환. 동작이 다르다.

### 2. Stack 클래스 사용

```java
// 쓰지 마라
Stack<Integer> stack = new Stack<>();  // 레거시, 느림

// 이렇게 쓸 것
Deque<Integer> stack = new ArrayDeque<>();
```

### 3. 괄호 매칭에서 닫는 괄호만 있는 경우

```java
// ")" → 첫 문자에서 pop 시도 → 빈 스택 체크 필수
if (stack.isEmpty()) return false;
```

### 4. 스택 순회 시 순서 주의

```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1);
stack.push(2);
stack.push(3);

// for-each는 top부터 순회 (3, 2, 1)
for (int n : stack) {
    System.out.print(n + " ");  // 3 2 1
}
```
