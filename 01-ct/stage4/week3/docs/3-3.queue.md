## 1. 설명

### 큐란

**FIFO** (First In, First Out) — 가장 먼저 넣은 것을 가장 먼저 꺼내는 자료구조.

줄서기(대기열)를 떠올리면 된다. 뒤에서 줄 서고, 앞에서 나간다.

```
offer(1) → [1]
offer(2) → [1, 2]
offer(3) → [1, 2, 3]
poll()   → 1 반환, [2, 3]
peek()   → 2 반환 (꺼내지 않고 확인만)
```

### 주요 연산

| 연산 | 동작 | 시간 복잡도 | 실패 시 |
|------|------|:-----------:|---------|
| `offer(e)` | 뒤에 추가 | O(1) | false 반환 |
| `poll()` | 앞에서 제거 및 반환 | O(1) | null 반환 |
| `peek()` | 앞의 원소 확인 | O(1) | null 반환 |
| `isEmpty()` | 비어 있는지 확인 | O(1) | - |
| `size()` | 원소 수 | O(1) | - |

> `add()`/`remove()`도 있지만, 실패 시 예외를 던진다. 코테에서는 **`offer()`/`poll()`** 쌍을 사용하는 것이 안전하다.

### ArrayDeque를 큐로 사용

스택과 마찬가지로 `ArrayDeque`를 사용한다. `LinkedList`도 Queue로 쓸 수 있지만, ArrayDeque가 더 빠르다.

```java
import java.util.ArrayDeque;
import java.util.Queue;

Queue<Integer> queue = new ArrayDeque<>();
queue.offer(1);      // 뒤에 추가
queue.offer(2);
queue.offer(3);

queue.peek();        // 1 (앞의 원소, 제거 안 함)
queue.poll();        // 1 (앞에서 제거)
queue.isEmpty();     // false
queue.size();        // 2
```

> 스택은 `Deque<>` 타입, 큐는 `Queue<>` 타입으로 선언. 둘 다 `ArrayDeque`로 생성.

### 스택 vs 큐 비교

| 비교 | 스택 (Stack) | 큐 (Queue) |
|------|:------------:|:----------:|
| 원칙 | LIFO | FIFO |
| 추가 | push (맨 위) | offer (맨 뒤) |
| 제거 | pop (맨 위) | poll (맨 앞) |
| 확인 | peek (맨 위) | peek (맨 앞) |
| 선언 | `Deque<>` | `Queue<>` |
| 활용 | DFS, 괄호 검증 | BFS, 시뮬레이션 |

### BFS 준비 관점

큐는 **BFS(너비 우선 탐색)**의 핵심 자료구조이다. week4에서 BFS를 배울 때 큐를 활용하게 된다.

```
BFS 기본 구조:
1. 시작 노드를 큐에 넣는다
2. 큐에서 하나 꺼낸다 (poll)
3. 인접 노드를 큐에 넣는다 (offer)
4. 큐가 빌 때까지 2~3 반복
```

지금은 큐의 동작 원리를 확실히 익히는 데 집중하자.

## 2. 예제

### 예제 1: 기본 큐 사용

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Queue<Integer> queue = new ArrayDeque<>();

        queue.offer(10);
        queue.offer(20);
        queue.offer(30);

        System.out.println(queue.peek());  // 10
        System.out.println(queue.poll());  // 10
        System.out.println(queue.poll());  // 20
        System.out.println(queue.size());  // 1
        System.out.println(queue.isEmpty()); // false
    }
}
```

### 예제 2: 카드 뒤집기 시뮬레이션

N장의 카드(1~N)가 큐에 있다. 맨 위 카드를 버리고, 그 다음 카드를 맨 아래로 보내는 것을 반복. 마지막 남은 카드는?

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int n = 6;
        Queue<Integer> queue = new ArrayDeque<>();
        for (int i = 1; i <= n; i++) {
            queue.offer(i);
        }

        while (queue.size() > 1) {
            queue.poll();                    // 맨 위 버림
            queue.offer(queue.poll());       // 그 다음을 맨 아래로
        }

        System.out.println(queue.poll());  // 마지막 남은 카드: 4
    }
}
```

### 예제 3: 큐로 요세푸스 문제 (간단 버전)

N명이 원형으로 앉아있고, K번째 사람을 제거하는 것을 반복.

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int n = 7, k = 3;
        Queue<Integer> queue = new ArrayDeque<>();
        for (int i = 1; i <= n; i++) queue.offer(i);

        StringBuilder sb = new StringBuilder("<");
        while (!queue.isEmpty()) {
            for (int i = 0; i < k - 1; i++) {
                queue.offer(queue.poll());  // k-1명을 뒤로 보냄
            }
            sb.append(queue.poll());        // k번째 제거
            if (!queue.isEmpty()) sb.append(", ");
        }
        sb.append(">");
        System.out.println(sb);  // <3, 6, 2, 7, 5, 1, 4>
    }
}
```

## 4. 이론 퀴즈

**Q1.** 큐에서 FIFO란 무엇인가?

<details>
<summary>정답</summary>
First In, First Out. 가장 먼저 넣은 원소가 가장 먼저 나온다.
</details>

**Q2.** `offer()`와 `add()`의 차이는?

<details>
<summary>정답</summary>
둘 다 큐에 원소를 추가한다. 차이는 실패 시: offer()는 false를 반환하고, add()는 IllegalStateException을 던진다. 코테에서는 offer()가 안전하다.
</details>

**Q3.** 빈 큐에서 `poll()`과 `remove()`를 각각 호출하면?

<details>
<summary>정답</summary>
poll()은 null을 반환한다. remove()는 NoSuchElementException을 던진다.
</details>

**Q4.** 큐를 선언할 때 `Queue<Integer> q = new ArrayDeque<>()`와 `Queue<Integer> q = new LinkedList<>()`의 차이는?

<details>
<summary>정답</summary>
둘 다 큐로 동작한다. ArrayDeque가 배열 기반이라 LinkedList(노드 할당 오버헤드)보다 빠르다. 대부분의 경우 ArrayDeque를 사용한다.
</details>

## 5. 함정 노트

### 1. poll()과 peek()의 null 반환

```java
Queue<Integer> queue = new ArrayDeque<>();
Integer val = queue.poll();   // null
Integer top = queue.peek();   // null

// int로 받으면 NullPointerException!
// int val = queue.poll();    // NPE!
```

> 빈 큐에서 poll/peek은 null을 반환한다. `int`로 받으면 언박싱 시 NPE.

### 2. 스택과 큐의 선언 혼동

```java
// 스택: Deque로 선언, push/pop 사용
Deque<Integer> stack = new ArrayDeque<>();

// 큐: Queue로 선언, offer/poll 사용
Queue<Integer> queue = new ArrayDeque<>();
```

둘 다 `ArrayDeque`지만 타입 선언이 다르다. 잘못된 타입으로 선언하면 의도치 않은 메서드 호출 가능.

### 3. Queue에서 인덱스 접근 불가

```java
Queue<Integer> queue = new ArrayDeque<>();
// queue.get(0);  // 컴파일 에러! Queue는 인덱스 접근 불가
```

앞의 원소만 `peek()`으로 확인 가능. 중간 접근이 필요하면 다른 자료구조를 써야 한다.

### 4. 출력 시 StringBuilder 필수

큐 문제도 N이 크면 `System.out.println()` 반복 호출 시 시간 초과.

```java
StringBuilder sb = new StringBuilder();
while (!queue.isEmpty()) {
    sb.append(queue.poll()).append('\n');
}
System.out.print(sb);
```
