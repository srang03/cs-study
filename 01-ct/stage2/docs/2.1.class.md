## 리뷰 미션 (prep p.3 String & StringBuilder)

**R1.** String이 불변(immutable)이란 무엇을 의미하는가?

<details>
<summary>정답</summary>
한 번 생성된 String 객체의 내용은 변경할 수 없다. 수정 메서드는 새 String을 반환한다.
</details>

**R2.** 반복문에서 `String +=` 대신 `StringBuilder`를 사용해야 하는 이유는?

<details>
<summary>정답</summary>
String +=는 매번 새 객체를 생성하여 O(n²). StringBuilder는 내부 버퍼에 추가하여 O(n).
</details>

## 1. 설명

### 클래스란

클래스는 **필드(데이터)와 메서드(동작)를 묶은 설계도**다. 이 설계도로 객체(인스턴스)를 생성한다.

```java
public class Student {
    // 필드
    private String name;
    private int age;

    // 생성자
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // 메서드
    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }
}
```

### 객체와 인스턴스

```java
Student s1 = new Student("Alice", 20);  // 인스턴스 생성
Student s2 = new Student("Bob", 22);    // 또 다른 인스턴스
```

- `s1`, `s2`는 같은 클래스에서 생성됐지만 각각 별도의 인스턴스
- `s1`과 `s2`는 스택에 참조가, 힙에 실제 객체가 존재

### 생성자

객체 생성 시 자동 호출되는 특수 메서드.

- 클래스 이름과 동일, 반환 타입 없음
- `new` 연산자와 함께만 호출
- 생성자를 하나도 안 만들면 컴파일러가 기본 생성자(인자 없는)를 자동 추가
- 오버로딩 가능

```java
public class Point {
    private int x, y;

    public Point() {            // 기본 생성자
        this(0, 0);             // 다른 생성자 호출
    }

    public Point(int x, int y) {  // 매개변수 있는 생성자
        this.x = x;
        this.y = y;
    }
}
```

### this

현재 인스턴스를 가리키는 참조.

- 필드와 매개변수 이름이 같을 때 구분: `this.name = name;`
- 같은 클래스의 다른 생성자 호출: `this(인자)`
- `static` 메서드에서는 사용 불가 (인스턴스가 없으므로)

### 접근 제어자

| 접근 제어자 | 같은 클래스 | 같은 패키지 | 자식 클래스 | 전체 |
|-------------|:-----------:|:-----------:|:-----------:|:----:|
| `private` | O | X | X | X |
| default | O | O | X | X |
| `protected` | O | O | O | X |
| `public` | O | O | O | O |

### getter / setter

`private` 필드에 대한 읽기/쓰기 메서드.

```java
public String getName() { return name; }       // getter
public void setName(String name) { this.name = name; } // setter
```

- 필드를 `private`으로 두고 getter/setter로만 접근 → **캡슐화**
- setter에 검증 로직을 넣어 잘못된 값 방지 가능

> 참조: [java/part2/docs/2.2.class.md](../../../java/part2/docs/2.2.class.md)

## 2. 예제

### 예제 1: 간단한 DTO 클래스

```java
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return name; }
    public int getAge() { return age; }

    public String toString() {
        return name + " (" + age + "세)";
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Person p = new Person("홍길동", 25);
        System.out.println(p);           // 홍길동 (25세)
        System.out.println(p.getName()); // 홍길동
    }
}
```

### 예제 2: 생성자 오버로딩

```java
public class Rectangle {
    private int width, height;

    public Rectangle() {
        this(1, 1);  // 기본값
    }

    public Rectangle(int side) {
        this(side, side);  // 정사각형
    }

    public Rectangle(int width, int height) {
        this.width = width;
        this.height = height;
    }

    public int area() { return width * height; }
}
```

### 예제 3: setter 검증

```java
public class Account {
    private int balance;

    public Account(int initial) {
        setBalance(initial);
    }

    public int getBalance() { return balance; }

    public void setBalance(int balance) {
        if (balance < 0) {
            throw new IllegalArgumentException("잔액은 음수일 수 없습니다");
        }
        this.balance = balance;
    }
}
```

## 4. 이론 퀴즈

**Q1.** 생성자를 하나도 정의하지 않으면 어떻게 되는가?

<details>
<summary>정답</summary>
컴파일러가 인자 없는 기본 생성자를 자동으로 추가한다. 단, 매개변수 있는 생성자를 하나라도 정의하면 기본 생성자는 자동 생성되지 않는다.
</details>

**Q2.** `this`를 `static` 메서드에서 사용할 수 없는 이유는?

<details>
<summary>정답</summary>
static 메서드는 클래스에 속하고 특정 인스턴스에 종속되지 않으므로, "현재 인스턴스"를 가리키는 this가 존재하지 않는다.
</details>

**Q3.** 필드를 `private`으로 하는 이유는?

<details>
<summary>정답</summary>
캡슐화. 외부에서 필드를 직접 수정하는 것을 막고, getter/setter를 통해 접근을 제어하여 검증, 불변식 유지 등이 가능하다.
</details>

## 5. 함정 노트

### 1. 기본 생성자 자동 생성 조건

```java
class Foo {
    Foo(int x) { }  // 매개변수 있는 생성자만 정의
}

Foo f = new Foo();   // 컴파일 에러! 기본 생성자가 없다
```

### 2. this()는 생성자 첫 줄에서만

```java
public Rectangle(int side) {
    System.out.println("test");  // 컴파일 에러!
    this(side, side);            // this()는 반드시 첫 줄
}
```

### 3. toString() 재정의 안 하면

```java
Person p = new Person("Alice", 20);
System.out.println(p);  // Person@1b6d3586 (해시코드 출력)
```

`toString()`을 재정의해야 의미 있는 출력이 나온다.
