## 리뷰 미션 (2.2 상속·다형성)

**R1.** `Animal a = new Dog();`에서 `a`는 `Dog`의 고유 메서드를 호출할 수 있는가?

<details>
<summary>정답</summary>
불가. a는 Animal 타입이므로 Animal에 정의된 멤버만 접근 가능. Dog 고유 메서드를 호출하려면 (Dog) a로 다운캐스팅해야 한다.
</details>

**R2.** `static` 필드와 인스턴스 필드의 차이를 한 줄로 설명하라.

<details>
<summary>정답</summary>
static 필드는 클래스당 하나로 모든 인스턴스가 공유하고, 인스턴스 필드는 객체마다 별도로 존재한다.
</details>

## 1. 설명

### 추상 클래스 (abstract class)

인스턴스를 직접 생성할 수 없는 클래스. 공통 구조를 정의하고 일부를 자식에게 위임.

```java
public abstract class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    // 추상 메서드: 본문 없음, 자식이 반드시 구현
    public abstract void speak();

    // 일반 메서드: 공통 동작
    public void sleep() {
        System.out.println(name + "이(가) 잔다");
    }
}
```

- `abstract` 메서드가 하나라도 있으면 클래스도 `abstract` 선언 필수
- 자식이 추상 메서드를 구현하지 않으면 자식도 `abstract`가 되어야 함
- 생성자, 필드, 일반 메서드를 가질 수 있음

### 인터페이스 (interface)

**"무엇을 할 수 있는가"만 정의**하고, "어떻게"는 구현 클래스에 맡긴다.

```java
public interface Drawable {
    void draw();  // public abstract 생략 가능
}

public interface Resizable {
    void resize(int factor);
}
```

- 메서드는 기본적으로 `public abstract`
- 필드는 `public static final` 상수만 가능
- 생성자 없음, 인스턴스 생성 불가
- **여러 인터페이스를 동시에 구현 가능** (다중 상속의 대안)

```java
class Circle implements Drawable, Resizable {
    int radius;

    @Override
    public void draw() {
        System.out.println("원 그리기");
    }

    @Override
    public void resize(int factor) {
        radius *= factor;
    }
}
```

### 추상 클래스 vs 인터페이스

| 구분 | 추상 클래스 | 인터페이스 |
|------|:-----------:|:---------:|
| 인스턴스 생성 | 불가 | 불가 |
| 필드 | 가능 (인스턴스/static) | 상수만 (`public static final`) |
| 생성자 | 가능 | 불가 |
| 메서드 구현 | 가능 (일반 메서드) | Java 8+ `default` 메서드 |
| 다중 구현 | 불가 (단일 상속) | **가능** |
| 용도 | "is-a" + 공통 코드 공유 | "can-do" 계약 |

### 인터페이스의 다형성

```java
Drawable d = new Circle();
d.draw();  // Circle의 draw() 실행
```

인터페이스 타입 변수로 구현체를 참조할 수 있다.
이를 통해 구현체를 바꿔도 호출 코드를 수정할 필요 없다.

> 참조: [java/part2/docs/2.6.polymorphism.md](../../../java/part2/docs/2.6.polymorphism.md)

## 2. 예제

### 예제 1: 추상 클래스

```java
abstract class Shape {
    abstract double area();

    void printArea() {
        System.out.println("넓이: " + area());
    }
}

class Circle extends Shape {
    double radius;

    Circle(double radius) { this.radius = radius; }

    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

class Square extends Shape {
    int side;

    Square(int side) { this.side = side; }

    @Override
    double area() {
        return side * side;
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Shape[] shapes = { new Circle(5), new Square(4) };
        for (Shape s : shapes) {
            s.printArea();  // 각각의 area() 호출
        }
    }
}
```

### 예제 2: 인터페이스와 다중 구현

```java
interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("오리가 난다");
    }

    @Override
    public void swim() {
        System.out.println("오리가 헤엄친다");
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Duck duck = new Duck();
        duck.fly();
        duck.swim();

        // 인터페이스 타입으로 참조
        Flyable f = duck;
        f.fly();
        // f.swim();  // 컴파일 에러: Flyable에는 swim()이 없음
    }
}
```

### 예제 3: 인터페이스를 메서드 매개변수로

```java
interface Printable {
    String toText();
}

class Report implements Printable {
    String title;
    Report(String title) { this.title = title; }

    @Override
    public String toText() {
        return "보고서: " + title;
    }
}

public class Main {
    // 인터페이스 타입을 받으므로 어떤 Printable 구현체든 사용 가능
    public static void print(Printable p) {
        System.out.println(p.toText());
    }

    public static void main(String[] args) {
        print(new Report("월간 매출"));
    }
}
```

## 3. 직접 작성

### 과제 1: 추상 클래스

`Animal` 추상 클래스 (`speak()` 추상 메서드, `sleep()` 일반 메서드)를 만들고, `Dog`, `Cat`, `Bird` 자식 클래스를 구현하라. `Animal[]` 배열에 담아 순회하며 `speak()`와 `sleep()`을 호출하라.

`Calculable` 인터페이스(`int calculate(int a, int b)`)를 만들고, `Adder`, `Multiplier` 클래스가 각각 구현하라. 메서드에서 `Calculable`을 받아 계산 결과를 출력하라.

## 4. 이론 퀴즈

**Q1.** 추상 클래스와 인터페이스의 가장 큰 차이는?

<details>
<summary>정답</summary>
추상 클래스는 필드, 생성자, 일반 메서드를 가질 수 있고 단일 상속만 가능. 인터페이스는 상수만 가지며 여러 개를 동시에 구현할 수 있다.
</details>

**Q2.** 추상 메서드를 가진 클래스에 `abstract`를 붙이지 않으면?

<details>
<summary>정답</summary>
컴파일 에러. 추상 메서드가 하나라도 있으면 클래스에 abstract를 반드시 선언해야 한다.
</details>

**Q3.** 인터페이스의 메서드에 `public abstract`를 생략할 수 있는 이유는?

<details>
<summary>정답</summary>
인터페이스의 메서드는 기본적으로 public abstract이므로, 명시하지 않아도 컴파일러가 자동으로 적용한다.
</details>

## 5. 함정 노트

### 1. 추상 메서드 구현 누락

```java
abstract class Animal { abstract void speak(); }

class Dog extends Animal { }  // 컴파일 에러! speak()를 구현하지 않음
```

### 2. 인터페이스 메서드에 public 누락

```java
interface Runnable { void run(); }

class Task implements Runnable {
    void run() { }  // 컴파일 에러! 인터페이스 메서드는 public인데 접근 범위를 줄임
    // 해결: public void run() { }
}
```

### 3. 인터페이스로 new 시도

```java
Drawable d = new Drawable();  // 컴파일 에러! 인터페이스는 인스턴스 생성 불가
```
