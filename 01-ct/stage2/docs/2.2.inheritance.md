## 1. 설명

### 상속 (extends)

부모 클래스의 필드와 메서드를 자식 클래스가 물려받는 것.

```java
class Animal {
    String name;

    void eat() {
        System.out.println(name + "이(가) 먹는다");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println(name + "이(가) 짖는다");
    }
}
```

- `extends`로 상속 선언
- Java는 **단일 상속**만 허용 (다중 상속 불가)
- 모든 클래스는 `Object`를 최상위 조상으로 가짐

### 메서드 재정의 (Override)

부모의 메서드를 자식이 같은 시그니처로 다시 정의.

```java
class Animal {
    void speak() {
        System.out.println("...");
    }
}

class Dog extends Animal {
    @Override
    void speak() {
        System.out.println("멍멍");
    }
}

class Cat extends Animal {
    @Override
    void speak() {
        System.out.println("야옹");
    }
}
```

- `@Override` 어노테이션으로 컴파일러 검증
- 호출 시 **실제 인스턴스 타입**의 메서드가 실행됨 (동적 바인딩)

### super

- `super.메서드()`: 부모의 메서드 호출
- `super()`: 부모의 생성자 호출 (자식 생성자 첫 줄에서만)
- 명시하지 않으면 컴파일러가 `super()`를 자동 삽입

```java
class Dog extends Animal {
    Dog(String name) {
        super();         // 부모 생성자 호출
        this.name = name;
    }

    @Override
    void eat() {
        super.eat();     // 부모의 eat() 먼저 실행
        System.out.println("사료를 먹는다");
    }
}
```

### 업캐스팅 / 다운캐스팅 (간단 언급)

```java
// 업캐스팅: 자식 → 부모 (자동)
Animal a = new Dog("멍이");
a.eat();     // OK
// a.bark(); // 컴파일 에러: Animal 타입으로는 Dog의 메서드 접근 불가

// 다운캐스팅: 부모 → 자식 (명시적, 주의 필요)
if (a instanceof Dog) {
    Dog d = (Dog) a;
    d.bark();  // OK
}
```

- 업캐스팅은 안전 (자식은 부모의 한 종류)
- 다운캐스팅은 `instanceof`로 확인 후 수행
- **실무에서 다운캐스팅은 가급적 피해야 할 설계 신호**

### 인스턴스 vs static 심화

| 구분 | 인스턴스 멤버 | static 멤버 |
|------|:------------:|:-----------:|
| 소속 | 객체(인스턴스) | 클래스 |
| 메모리 | 힙 (객체마다 별도) | Method Area (클래스당 하나) |
| 접근 | 객체 생성 후 `obj.멤버` | `클래스명.멤버` (객체 없이) |
| this 사용 | O | X |

```java
class Counter {
    static int totalCount = 0;  // 모든 인스턴스가 공유
    int id;                     // 각 인스턴스마다 별도

    Counter() {
        totalCount++;
        this.id = totalCount;
    }
}
```

```java
Counter c1 = new Counter();  // totalCount = 1, c1.id = 1
Counter c2 = new Counter();  // totalCount = 2, c2.id = 2
System.out.println(Counter.totalCount);  // 2 (클래스 이름으로 접근)
```

**코테에서의 활용:**
- `main`에서 호출하는 메서드는 모두 `static`
- 유틸 메서드(`Math.max` 등)는 `static`
- 인스턴스가 필요 없는 계산 로직은 `static`으로 작성

## 2. 예제

### 예제 1: 상속과 재정의

```java
class Shape {
    String name;

    Shape(String name) {
        this.name = name;
    }

    double area() {
        return 0;
    }

    @Override
    public String toString() {
        return name + " 넓이: " + area();
    }
}

class Circle extends Shape {
    double radius;

    Circle(double radius) {
        super("원");
        this.radius = radius;
    }

    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

class Rect extends Shape {
    int width, height;

    Rect(int width, int height) {
        super("사각형");
        this.width = width;
        this.height = height;
    }

    @Override
    double area() {
        return width * height;
    }
}
```

```java
public class Main {
    public static void main(String[] args) {
        Shape[] shapes = { new Circle(5), new Rect(3, 4) };
        for (Shape s : shapes) {
            System.out.println(s);  // 다형성: 각각의 area() 호출
        }
    }
}
```

### 예제 2: static vs 인스턴스

```java
public class Main {
    static int count = 0;  // static: 프로그램 전체에서 하나

    int id;  // 인스턴스: 객체마다 별도 (여기선 Main이 static 컨텍스트이므로 사용 안 함)

    // 코테에서는 이렇게 static 메서드로 로직 작성
    public static int add(int a, int b) {
        count++;  // static 필드는 static 메서드에서 접근 가능
        return a + b;
    }

    public static void main(String[] args) {
        System.out.println(add(3, 5));  // 8
        System.out.println(add(1, 2));  // 3
        System.out.println("호출 횟수: " + count);  // 2
    }
}
```

## 4. 이론 퀴즈

**Q1.** 오버라이딩과 오버로딩의 차이는?

<details>
<summary>정답</summary>
오버라이딩: 부모와 같은 시그니처(이름, 매개변수)로 자식에서 재정의. 오버로딩: 같은 이름, 다른 매개변수 목록으로 여러 메서드 정의.
</details>

**Q2.** `Animal a = new Dog();`에서 `a.speak()` 호출 시 어떤 메서드가 실행되는가?

<details>
<summary>정답</summary>
Dog의 speak(). 변수 타입(Animal)이 아닌 실제 인스턴스 타입(Dog)의 메서드가 실행된다 (동적 바인딩).
</details>

**Q3.** static 필드는 어디에 저장되는가?

<details>
<summary>정답</summary>
Method Area (메서드 영역). 클래스당 하나만 존재하며, 모든 인스턴스가 공유한다.
</details>

## 5. 함정 노트

### 1. super() 누락으로 인한 컴파일 에러

```java
class Parent {
    Parent(int x) { }  // 기본 생성자 없음
}

class Child extends Parent {
    Child() { }  // 컴파일 에러! super()가 자동 삽입되지만 Parent()가 없음
    // 해결: Child() { super(0); }
}
```

### 2. @Override 없이 오타

```java
class Dog extends Animal {
    void Speak() { }  // 대문자 S — 재정의가 아닌 새 메서드!
    // @Override를 붙였다면 컴파일러가 오류를 잡아줌
}
```

### 3. static 메서드에서 인스턴스 멤버 접근

```java
public class Main {
    int value = 10;  // 인스턴스 변수

    public static void main(String[] args) {
        // System.out.println(value);  // 컴파일 에러! static에서 인스턴스 접근 불가
        Main m = new Main();
        System.out.println(m.value);   // OK: 객체를 통해 접근
    }
}
```
