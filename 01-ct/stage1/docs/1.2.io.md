## 1. 설명

### Scanner

`java.util.Scanner`는 입력 데이터를 토큰 단위로 파싱하는 클래스다.

```java
import java.util.Scanner;

Scanner sc = new Scanner(System.in);
int n = sc.nextInt();       // 정수 한 개 읽기
String s = sc.nextLine();   // 한 줄 읽기
```

**주요 메서드:**

| 메서드 | 동작 |
|--------|------|
| `nextInt()` | 다음 토큰을 정수로 반환 |
| `nextLong()` | 다음 토큰을 long으로 반환 |
| `nextDouble()` | 다음 토큰을 실수로 반환 |
| `next()` | 공백 전까지 한 토큰을 문자열로 반환 |
| `nextLine()` | 개행 문자까지 한 줄 전체를 문자열로 반환 |

### Scanner의 한계

Scanner는 내부적으로 정규식 기반 파싱을 수행하여 **느리다**.
백준 등에서 입력이 많으면(10만 줄 이상) 시간초과의 원인이 된다.

### BufferedReader + StringTokenizer

코딩 테스트에서 사실상 표준인 빠른 입력 방식이다.

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
```

**BufferedReader**: 버퍼를 사용하여 I/O 호출 횟수를 줄인다. `readLine()`으로 한 줄씩 읽는다.

**StringTokenizer**: 읽은 문자열을 구분자(기본: 공백)로 분리한다.

```java
String line = br.readLine();                    // "10 20" 한 줄 읽기
StringTokenizer st = new StringTokenizer(line); // 공백으로 분리
int a = Integer.parseInt(st.nextToken());       // "10" → 10
int b = Integer.parseInt(st.nextToken());       // "20" → 20
```

### 출력 최적화

`System.out.println`을 반복 호출하면 매번 flush가 발생하여 느리다.
`StringBuilder`나 `BufferedWriter`로 모아서 한 번에 출력하면 빠르다.
(StringBuilder는 prep p.3에서 상세히 다룬다)

```java
import java.io.BufferedWriter;
import java.io.OutputStreamWriter;

BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
bw.write("Hello\n");
bw.flush();  // 반드시 flush 호출
bw.close();
```

### Scanner vs BufferedReader 비교

| 항목 | Scanner | BufferedReader |
|------|---------|----------------|
| 속도 | 느림 (정규식 파싱) | 빠름 (버퍼 I/O) |
| 사용 편의성 | 높음 (타입별 메서드) | 낮음 (문자열만 반환) |
| 타입 변환 | 자동 (`nextInt()`) | 수동 (`Integer.parseInt()`) |
| 코테 권장 | 소규모 입력 | **대규모 입력 (권장)** |

## 2. 예제

### 예제 1: Scanner로 정수 두 개 입력

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int a = sc.nextInt();
        int b = sc.nextInt();
        System.out.println(a + b);
    }
}
```

### 예제 2: BufferedReader로 같은 작업

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());
        int a = Integer.parseInt(st.nextToken());
        int b = Integer.parseInt(st.nextToken());
        System.out.println(a + b);
    }
}
```

### 예제 3: 여러 줄 입력 처리

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim()); // 첫 줄: 테스트 케이스 수

        for (int i = 0; i < n; i++) {
            String line = br.readLine(); // 각 줄 읽기
            System.out.println(line);
        }
    }
}
```

## 4. 이론 퀴즈

**Q1.** Scanner가 느린 이유는?

<details>
<summary>정답</summary>
내부적으로 정규식 기반 파싱을 수행하기 때문이다.
</details>

**Q2.** `BufferedReader`의 `readLine()`은 무엇을 반환하는가?

<details>
<summary>정답</summary>
개행 문자 전까지의 한 줄을 String으로 반환한다. 개행 문자 자체는 포함하지 않는다.
</details>

**Q3.** `nextInt()` 후 `nextLine()`을 호출하면 빈 문자열이 반환되는 이유는?

<details>
<summary>정답</summary>
nextInt()는 정수만 읽고 버퍼에 남은 개행 문자(\n)를 소비하지 않는다. 이어서 nextLine()을 호출하면 남은 개행 문자만 읽어 빈 문자열을 반환한다.
</details>

**Q4.** `StringTokenizer`의 기본 구분자는 무엇인가?

<details>
<summary>정답</summary>
공백(whitespace: 스페이스, 탭, 개행).
</details>

**Q5.** `BufferedWriter` 사용 후 반드시 호출해야 하는 메서드는?

<details>
<summary>정답</summary>
flush(). 버퍼에 남아 있는 데이터를 실제로 출력 스트림에 내보내기 위해 필요하다.
</details>

## 5. 함정 노트

### 1. nextInt() 후 nextLine() 문제

```java
Scanner sc = new Scanner(System.in);
int n = sc.nextInt();
String s = sc.nextLine(); // 빈 문자열! 개행 문자만 읽음
```

**해결**: `nextInt()` 후 `nextLine()`을 한 번 호출하여 버퍼를 비운다.

```java
int n = sc.nextInt();
sc.nextLine();             // 버퍼 비우기
String s = sc.nextLine();  // 실제 입력 읽기
```

### 2. BufferedReader에서 throws 누락

`BufferedReader`의 `readLine()`은 `IOException`을 던진다.
`main`에 `throws Exception`을 붙이지 않으면 컴파일 에러.

```java
// 틀림
public static void main(String[] args) { ... }

// 맞음
public static void main(String[] args) throws Exception { ... }
```

### 3. Integer.parseInt에 공백 포함

```java
int n = Integer.parseInt("10 ");  // NumberFormatException!
int n = Integer.parseInt("10 ".trim());  // OK: 10
```

`readLine()`으로 읽은 값에 불필요한 공백이 있을 수 있다. `trim()`으로 제거.

### 4. import 누락

Scanner: `import java.util.Scanner;`
BufferedReader: `import java.io.BufferedReader;` + `import java.io.InputStreamReader;`

IDE 없이 작성할 때 자주 빠뜨린다.
