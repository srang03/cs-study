## 1. 설명

### 산술 연산자

| 연산자 | 의미 | 예 |
|--------|------|-----|
| `+` | 덧셈 | `3 + 2` → 5 |
| `-` | 뺄셈 | `3 - 2` → 1 |
| `*` | 곱셈 | `3 * 2` → 6 |
| `/` | 나눗셈 | `7 / 2` → 3 (정수끼리면 정수) |
| `%` | 나머지 | `7 % 2` → 1 |

**정수 나눗셈**: `int / int`는 소수점을 버린다. 실수 결과가 필요하면 피연산자 중 하나를 `double`로 변환.

```java
System.out.println(7 / 2);        // 3
System.out.println(7.0 / 2);      // 3.5
System.out.println((double) 7 / 2); // 3.5
```

### 비교 연산자

| 연산자 | 의미 |
|--------|------|
| `==` | 같다 |
| `!=` | 다르다 |
| `>`, `<` | 크다, 작다 |
| `>=`, `<=` | 크거나 같다, 작거나 같다 |

결과는 `boolean` (`true` / `false`).

### 논리 연산자

| 연산자 | 의미 | 특성 |
|--------|------|------|
| `&&` | AND | 둘 다 true일 때 true |
| `\|\|` | OR | 하나라도 true이면 true |
| `!` | NOT | 반전 |

### 형승격 (Type Promotion)

연산 시 피연산자가 다른 타입이면, 더 넓은 타입으로 자동 승격된다.

```
byte, short, char  →  int  →  long  →  float  →  double
```

- `char + int` → `int`
- `int + double` → `double`
- `String + 숫자` → `String` (문자열 연결)

```java
System.out.println('A' + 1);         // 66 (char → int 승격)
System.out.println("값: " + 10);     // "값: 10" (문자열 연결)
System.out.println(1 + 2 + "abc");   // "3abc" (왼쪽부터 평가)
System.out.println("abc" + 1 + 2);   // "abc12" (왼쪽부터 문자열 연결)
```

### 단락 평가 (Short-Circuit Evaluation)

논리 연산에서 결과가 확정되면 나머지 항을 평가하지 않는다.

```java
// && : 왼쪽이 false이면 오른쪽 평가 안 함
if (arr != null && arr.length > 0) { ... }
// arr이 null이면 arr.length를 평가하지 않으므로 NullPointerException 방지

// || : 왼쪽이 true이면 오른쪽 평가 안 함
if (x == 0 || y / x > 10) { ... }
// x가 0이면 나눗셈을 평가하지 않으므로 ArithmeticException 방지
```

### 증감 연산자

```java
int a = 5;
a++;     // 후위: 현재 값 사용 후 증가
++a;     // 전위: 먼저 증가 후 값 사용
a--;     // 후위 감소
--a;     // 전위 감소
```

### 대입 연산자

```java
a += 5;   // a = a + 5
a -= 3;   // a = a - 3
a *= 2;   // a = a * 2
a /= 4;   // a = a / 4
a %= 3;   // a = a % 3
```

### 삼항 연산자

```java
int max = (a > b) ? a : b;  // a가 크면 a, 아니면 b
```

## 2. 예제

### 예제 1: 산술 연산과 정수 나눗셈

```java
public class Main {
    public static void main(String[] args) {
        int a = 10, b = 3;

        System.out.println(a + b);   // 13
        System.out.println(a - b);   // 7
        System.out.println(a * b);   // 30
        System.out.println(a / b);   // 3 (소수점 버림!)
        System.out.println(a % b);   // 1

        // 실수 나눗셈
        System.out.println((double) a / b);  // 3.3333...
    }
}
```

### 예제 2: 논리 연산과 단락 평가

```java
public class Main {
    public static void main(String[] args) {
        int x = 0;

        // 단락 평가: x == 0이 true이므로 오른쪽 평가 안 함
        if (x == 0 || 10 / x > 5) {
            System.out.println("단락 평가로 안전");
        }

        // 만약 단락 평가가 없었다면 10 / 0 → ArithmeticException
    }
}
```

### 예제 3: 문자열 연결과 형승격

```java
public class Main {
    public static void main(String[] args) {
        System.out.println(1 + 2 + "3");     // "33" (1+2=3, 3+"3"="33")
        System.out.println("1" + 2 + 3);     // "123" ("1"+2="12", "12"+3="123")
        System.out.println("1" + (2 + 3));   // "15" (괄호 먼저, "1"+5="15")
    }
}
```

## 4. 이론 퀴즈

**Q1.** `7 / 2`의 결과는?

<details>
<summary>정답</summary>
3. int끼리의 나눗셈은 소수점을 버린다.
</details>

**Q2.** `"abc" + 1 + 2`의 결과는?

<details>
<summary>정답</summary>
"abc12". 왼쪽부터 평가하므로 "abc"+1="abc1", "abc1"+2="abc12".
</details>

**Q3.** 단락 평가란 무엇인가?

<details>
<summary>정답</summary>
논리 연산에서 결과가 확정되면 나머지 항을 평가하지 않는 동작. &&는 왼쪽이 false이면 오른쪽을 평가하지 않고, ||는 왼쪽이 true이면 오른쪽을 평가하지 않는다.
</details>

**Q4.** `char c = 'A'; System.out.println(c + 1);`의 출력은?

<details>
<summary>정답</summary>
66. char는 int로 승격되어 연산된다. 'A'의 코드값 65 + 1 = 66.
</details>

## 5. 함정 노트

### 1. 정수 나눗셈에서 소수점 손실

```java
int score = 3;
int total = 5;
double rate = score / total;  // 0.0! (정수 나눗셈 후 double 대입)
double rate = (double) score / total;  // 0.6 (올바름)
```

### 2. 문자열 + 연산의 평가 순서

```java
System.out.println(1 + 2 + "3");   // "33" (의도: "123"이었다면?)
System.out.println("" + 1 + 2 + 3); // "123" (빈 문자열 먼저)
```

왼쪽부터 평가하므로, 숫자끼리 먼저 만나면 산술 연산이 된다.

### 3. 0으로 나누기

```java
int a = 10 / 0;     // ArithmeticException (정수)
double b = 10.0 / 0; // Infinity (실수 — 예외 없음!)
```

### 4. 증감 연산자 혼동

```java
int a = 5;
int b = a++;  // b = 5, a = 6 (후위: 대입 후 증가)
int c = ++a;  // c = 7, a = 7 (전위: 증가 후 대입)
```

코테에서는 헷갈림 방지를 위해 **단독으로만 사용**하는 것을 권장.
