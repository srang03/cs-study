## 1. 설명

### 메서드란

여러 구문을 `{ }`로 묶어 이름을 붙인 재사용 가능한 코드 단위다.

```java
반환타입 메서드이름(매개변수) {
    // 실행할 코드
    return 반환값;
}
```

### 메서드의 구성 요소

```java
public static int add(int a, int b) {
    return a + b;
}
```

| 요소 | 의미 |
|------|------|
| `public` | 접근 제어자: 어디서든 호출 가능 |
| `static` | 객체 생성 없이 호출 가능 |
| `int` | 반환 타입 |
| `add` | 메서드 이름 |
| `int a, int b` | 매개변수 (parameter) |
| `return a + b` | 반환값 |

### static 메서드

`static` 메서드는 클래스에 속하며, 객체를 생성하지 않고 호출할 수 있다.

```java
public class Main {
    public static void main(String[] args) {
        int result = add(3, 5);      // static 메서드이므로 직접 호출 가능
        System.out.println(result);
    }

    // main이 static이므로, main에서 호출하는 메서드도 static이어야 한다
    public static int add(int a, int b) {
        return a + b;
    }
}
```

**왜 코테에서 중요한가:**
- `main`이 `static`이므로, `main`에서 직접 호출하는 메서드도 `static`이어야 한다
- 인스턴스 메서드를 호출하려면 객체를 먼저 생성해야 하므로 번거롭다
- 코테에서는 대부분의 유틸 메서드를 `static`으로 선언한다

### void와 return

```java
// 반환값이 없는 메서드
public static void printHello() {
    System.out.println("Hello");
    // return 생략 가능
}

// 조건에 따라 일찍 반환
public static void printPositive(int n) {
    if (n <= 0) {
        return;  // void 메서드에서 일찍 종료
    }
    System.out.println(n);
}
```

### 매개변수 전달 (Pass by Value)

Java는 항상 **값에 의한 전달(pass by value)**이다.

- 원시 타입: 값이 복사됨 → 메서드 안에서 바꿔도 원본 불변
- 참조 타입: 참조(주소)가 복사됨 → 같은 객체를 가리키므로 객체 내용은 변경 가능

```java
public static void change(int x) {
    x = 100;  // 원본에 영향 없음
}

public static void main(String[] args) {
    int a = 10;
    change(a);
    System.out.println(a);  // 10 (변경 안 됨)
}
```

### 메서드 오버로딩

같은 이름, 다른 매개변수 목록(개수 또는 타입)으로 여러 메서드를 정의할 수 있다.

```java
public static int add(int a, int b) {
    return a + b;
}

public static double add(double a, double b) {
    return a + b;
}

public static int add(int a, int b, int c) {
    return a + b + c;
}
```

- 반환 타입만 다른 것은 오버로딩이 아니다
- 컴파일러가 호출 시 인자 타입/개수로 어떤 메서드인지 결정한다

### Math 클래스 — static 메서드의 실전 예시

`Math` 클래스는 수학 관련 `static` 메서드를 모아 놓은 유틸 클래스다.
객체 생성 없이 `Math.메서드이름()`으로 바로 호출한다.

| 메서드 | 동작 | 예 |
|--------|------|-----|
| `Math.max(a, b)` | 둘 중 큰 값 | `Math.max(3, 7)` → 7 |
| `Math.min(a, b)` | 둘 중 작은 값 | `Math.min(3, 7)` → 3 |
| `Math.abs(a)` | 절대값 | `Math.abs(-5)` → 5 |
| `Math.pow(a, b)` | 거듭제곱 | `Math.pow(2, 10)` → 1024.0 |
| `Math.sqrt(a)` | 제곱근 | `Math.sqrt(16)` → 4.0 |

```java
int bigger = Math.max(10, 20);      // 20
int smaller = Math.min(10, 20);     // 10
int positive = Math.abs(-42);       // 42
double power = Math.pow(2, 10);     // 1024.0
```

`Math.max`, `Math.min`은 코테에서 가장 자주 쓰는 메서드 중 하나다.

> 참조: [java/part1/docs/1.12.methods.md](../../../java/part1/docs/1.12.methods.md)

## 2. 예제

### 예제 1: 유틸 메서드 작성

```java
public class Main {
    // 두 수 중 큰 값을 반환
    public static int max(int a, int b) {
        if (a > b) {
            return a;
        }
        return b;
    }

    // 정수의 절대값 반환
    public static int abs(int n) {
        if (n < 0) {
            return -n;
        }
        return n;
    }

    public static void main(String[] args) {
        System.out.println(max(10, 20));   // 20
        System.out.println(abs(-15));      // 15
    }
}
```

### 예제 2: 오버로딩

```java
public class Main {
    public static int multiply(int a, int b) {
        return a * b;
    }

    public static double multiply(double a, double b) {
        return a * b;
    }

    public static void main(String[] args) {
        System.out.println(multiply(3, 4));      // 12 (int 버전)
        System.out.println(multiply(2.5, 3.0));  // 7.5 (double 버전)
    }
}
```

### 예제 3: Math 클래스 활용

```java
public class Main {
    public static void main(String[] args) {
        int[] nums = {3, -7, 15, -2, 9};

        // 최대값, 최소값 찾기
        int max = nums[0];
        int min = nums[0];
        for (int i = 1; i < nums.length; i++) {
            max = Math.max(max, nums[i]);
            min = Math.min(min, nums[i]);
        }

        System.out.println("최대: " + max);  // 15
        System.out.println("최소: " + min);  // -7
        System.out.println("차이: " + Math.abs(max - min));  // 22
    }
}
```

## 4. 이론 퀴즈

**Q1.** `main`에서 호출하는 메서드가 `static`이어야 하는 이유는?

<details>
<summary>정답</summary>
main 자체가 static이므로 인스턴스 없이 실행된다. static 컨텍스트에서는 인스턴스 멤버에 접근할 수 없으므로, 호출하는 메서드도 static이어야 한다. (또는 객체를 먼저 생성해야 한다)
</details>

**Q2.** 오버로딩의 조건은?

<details>
<summary>정답</summary>
같은 메서드 이름, 다른 매개변수 목록(개수 또는 타입). 반환 타입만 다른 것은 오버로딩이 아니다.
</details>

**Q3.** Java의 매개변수 전달 방식은?

<details>
<summary>정답</summary>
항상 값에 의한 전달(pass by value). 원시 타입은 값이 복사되고, 참조 타입은 참조(주소) 값이 복사된다.
</details>

**Q4.** `Math.pow(2, 10)`의 반환 타입은?

<details>
<summary>정답</summary>
double. Math.pow는 항상 double을 반환한다. (1024.0)
</details>

## 5. 함정 노트

### 1. static 빠뜨리기

```java
public class Main {
    // 틀림: main에서 직접 호출 불가
    public int add(int a, int b) { return a + b; }

    // 맞음
    public static int add(int a, int b) { return a + b; }

    public static void main(String[] args) {
        System.out.println(add(1, 2));
    }
}
```

### 2. return 누락

```java
// 컴파일 에러: 모든 경로에서 return이 있어야 함
public static int abs(int n) {
    if (n < 0) {
        return -n;
    }
    // else에서 return이 없음!
}
```

### 3. Math.pow의 반환 타입

```java
int result = Math.pow(2, 10);          // 컴파일 에러! double → int
int result = (int) Math.pow(2, 10);    // OK: 1024
```

### 4. 원시 타입 swap 불가

```java
public static void swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    // main의 원본 변수는 바뀌지 않는다! (값 복사)
}
```

Java에서 원시 타입 두 개를 메서드로 교환하는 것은 불가능하다.
배열이나 객체를 사용해야 한다.
