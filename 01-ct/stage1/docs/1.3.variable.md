## 1. 설명

### 원시 타입 (Primitive Type)

값 자체를 저장하며, 스택 메모리에 할당된다.

| 타입 | 크기 | 범위 | 기본값 |
|------|------|------|--------|
| `byte` | 1바이트 | -128 ~ 127 | 0 |
| `short` | 2바이트 | -32,768 ~ 32,767 | 0 |
| `int` | 4바이트 | -2^31 ~ 2^31-1 (약 21억) | 0 |
| `long` | 8바이트 | -2^63 ~ 2^63-1 | 0L |
| `float` | 4바이트 | IEEE 754 단정밀도 | 0.0f |
| `double` | 8바이트 | IEEE 754 배정밀도 | 0.0 |
| `char` | 2바이트 | 0 ~ 65,535 (UTF-16 코드 유닛) | '\u0000' |
| `boolean` | JVM 구현 의존 | true / false | false |

- `int`가 코테에서 가장 자주 사용하는 정수 타입
- 약 21억을 넘는 값은 `long` 사용 (`long n = 123456789012L;` — L 접미사 필수)
- 실수는 기본적으로 `double` (코테에서 `float`는 거의 안 씀)

### 참조 타입 (Reference Type)

객체의 메모리 주소(참조)를 저장한다. 실제 객체는 힙에 할당된다.

- `String`, 배열, 클래스 인스턴스 등
- 참조 변수 자체는 스택에, 가리키는 객체는 힙에 존재

```java
String s = "Hello";  // s는 스택에, "Hello" 객체는 힙에
int[] arr = {1, 2};  // arr은 스택에, 배열 객체는 힙에
```

### 원시 타입 vs 참조 타입

| 구분 | 원시 타입 | 참조 타입 |
|------|-----------|-----------|
| 저장 위치 | 스택 (값 자체) | 스택 (참조) + 힙 (객체) |
| 기본값 | 0, false 등 | null |
| 비교 | `==`로 값 비교 | `==`는 참조 비교, `.equals()`로 값 비교 |
| GC 대상 | 아님 | 참조가 없어지면 GC 대상 |

### Wrapper 클래스

원시 타입을 객체로 감싸는 클래스다.
컬렉션(`List`, `Set`, `Map`)은 원시 타입을 직접 담을 수 없으므로 Wrapper를 사용한다.

| 원시 타입 | Wrapper 클래스 |
|-----------|---------------|
| `int` | `Integer` |
| `long` | `Long` |
| `double` | `Double` |
| `char` | `Character` |
| `boolean` | `Boolean` |

### 오토박싱 / 언박싱

원시 타입 ↔ Wrapper 클래스 간 자동 변환이 일어난다.

```java
// 오토박싱: int → Integer
Integer num = 10;          // 컴파일러가 Integer.valueOf(10)으로 변환

// 언박싱: Integer → int
int val = num;             // 컴파일러가 num.intValue()로 변환
```

### 형변환 (Type Casting)

```java
// 자동 형변환 (작은 → 큰): 데이터 손실 없음
int i = 100;
double d = i;              // 100.0

// 강제 형변환 (큰 → 작은): 데이터 손실 가능
double pi = 3.14;
int n = (int) pi;          // 3 (소수부 버림)
```

## 2. 예제

### 예제 1: 원시 타입 선언과 초기화

```java
public class Main {
    public static void main(String[] args) {
        int age = 25;
        double height = 175.5;
        char grade = 'A';
        boolean isStudent = true;
        long population = 8000000000L; // 약 80억 — int 범위 초과

        System.out.println("나이: " + age);
        System.out.println("키: " + height);
        System.out.println("등급: " + grade);
        System.out.println("학생 여부: " + isStudent);
        System.out.println("인구: " + population);
    }
}
```

### 예제 2: 참조 타입과 null

```java
public class Main {
    public static void main(String[] args) {
        String name = "Java";
        String empty = null;

        System.out.println(name.length());   // 4
        // System.out.println(empty.length()); // NullPointerException!

        // null 체크
        if (empty != null) {
            System.out.println(empty.length());
        }
    }
}
```

### 예제 3: 오토박싱과 언박싱

```java
public class Main {
    public static void main(String[] args) {
        // 오토박싱
        Integer a = 10;
        Integer b = 20;

        // 언박싱 후 연산
        int sum = a + b;  // Integer → int 자동 변환 후 덧셈

        System.out.println("합: " + sum);   // 30

        // 주의: Integer끼리 == 비교
        Integer x = 128;
        Integer y = 128;
        System.out.println(x == y);       // false! (참조 비교)
        System.out.println(x.equals(y));  // true  (값 비교)
    }
}
```

## 4. 이론 퀴즈

**Q1.** `int`의 최대값은 약 얼마인가?

<details>
<summary>정답</summary>
약 21억 (2,147,483,647 = 2^31 - 1)
</details>

**Q2.** 다음 코드의 출력은?

```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b);
```

<details>
<summary>정답</summary>
true. Integer는 -128~127 범위의 값을 캐싱(Integer Cache)하므로 같은 객체를 참조한다. 128 이상은 false가 된다.
</details>

**Q3.** 원시 타입 변수는 어느 메모리 영역에 저장되는가?

<details>
<summary>정답</summary>
스택(Stack). 지역 변수의 경우 메서드 호출 시 스택 프레임에 할당되고, 메서드 반환 시 제거된다.
</details>

**Q4.** `String`끼리 값을 비교할 때 `==` 대신 무엇을 사용해야 하는가?

<details>
<summary>정답</summary>
.equals() 메서드. ==는 참조(주소) 비교이고, .equals()는 실제 문자열 내용 비교이다.
</details>

## 5. 함정 노트

### 1. int 오버플로우

```java
int a = 2000000000;  // 20억
int b = 2000000000;
int sum = a + b;     // 오버플로우! 음수가 됨

long sum2 = (long) a + b;  // 올바른 방법
```

코테에서 "두 수의 합"이 21억을 넘을 수 있으면 `long` 사용.

### 2. `==` vs `.equals()`

```java
String s1 = new String("hello");
String s2 = new String("hello");
System.out.println(s1 == s2);      // false (참조 비교)
System.out.println(s1.equals(s2)); // true  (값 비교)
```

### 3. Integer 캐시 범위

```java
Integer a = 127, b = 127;
System.out.println(a == b);  // true  (캐시 범위 내)

Integer c = 128, d = 128;
System.out.println(c == d);  // false (캐시 범위 밖)
```

Wrapper 타입은 항상 `.equals()`로 비교하라.

### 4. long 리터럴에 L 빠뜨리기

```java
long big = 3000000000;   // 컴파일 에러! int 범위 초과
long big = 3000000000L;  // OK
```

### 5. char는 정수 타입

```java
char c = 'A';
System.out.println(c + 1);       // 66 (int로 승격)
System.out.println((char)(c + 1)); // B
```
