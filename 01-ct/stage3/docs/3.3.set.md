**R1.** `List<Integer>`에서 `remove(1)`은 무엇을 삭제하는가?

<details>
<summary>정답</summary>
인덱스 1의 요소를 삭제한다. 값 1을 삭제하려면 remove(Integer.valueOf(1))을 사용해야 한다.
</details>

<details>
<summary>정답</summary>
O(n log n). Java의 정렬은 TimSort(Merge Sort + Insertion Sort 하이브리드)를 사용한다.
</details>

## 1. 설명

### Set이란

**중복을 허용하지 않는** 컬렉션. 순서를 보장하지 않는다.

```java
import java.util.HashSet;
import java.util.Set;

Set<String> set = new HashSet<>();
set.add("apple");
set.add("banana");
set.add("apple");  // 중복 → 추가되지 않음

System.out.println(set.size());  // 2
```

### HashSet

해시 테이블 기반. 삽입, 삭제, 조회가 평균 O(1).

### 주요 메서드

| 메서드 | 동작 | 시간 복잡도 |
|--------|------|:-----------:|
| `add(e)` | 추가 (이미 있으면 false 반환) | O(1) |
| `remove(e)` | 삭제 | O(1) |
| `contains(e)` | 포함 여부 | O(1) |
| `size()` | 요소 수 | O(1) |
| `isEmpty()` | 비었는지 | O(1) |
| `clear()` | 전체 삭제 | O(n) |

### Set 순회

```java
// for-each
for (String s : set) {
    System.out.println(s);
}
```

순서가 보장되지 않으므로 출력 순서는 예측 불가.

### 중복 제거 패턴

```java
// 배열에서 중복 제거
int[] arr = {1, 2, 2, 3, 3, 3};
Set<Integer> unique = new HashSet<>();
for (int num : arr) {
    unique.add(num);
}
System.out.println(unique.size());  // 3
```

### TreeSet (정렬된 Set)

```java
import java.util.TreeSet;

Set<Integer> sorted = new TreeSet<>();
sorted.add(5);
sorted.add(1);
sorted.add(3);
System.out.println(sorted);  // [1, 3, 5] — 자동 정렬
```

- 내부적으로 레드-블랙 트리 사용
- 삽입, 삭제, 조회 O(log n)
- 정렬된 순서로 순회 가능

### 집합 연산

```java
Set<Integer> a = new HashSet<>(Set.of(1, 2, 3, 4));
Set<Integer> b = new HashSet<>(Set.of(3, 4, 5, 6));

// 교집합
Set<Integer> inter = new HashSet<>(a);
inter.retainAll(b);  // {3, 4}

// 합집합
Set<Integer> union = new HashSet<>(a);
union.addAll(b);  // {1, 2, 3, 4, 5, 6}

// 차집합
Set<Integer> diff = new HashSet<>(a);
diff.removeAll(b);  // {1, 2}
```

## 2. 예제

### 예제 1: 중복 제거

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        String[] words = {"java", "python", "java", "c++", "python"};

        Set<String> unique = new HashSet<>(Arrays.asList(words));
        System.out.println("고유 언어 수: " + unique.size());  // 3
        System.out.println(unique);  // 순서 비보장
    }
}
```

### 예제 2: 두 배열의 교집합

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        int[] a = {1, 2, 3, 4, 5};
        int[] b = {3, 4, 5, 6, 7};

        Set<Integer> setA = new HashSet<>();
        for (int n : a) setA.add(n);

        Set<Integer> setB = new HashSet<>();
        for (int n : b) setB.add(n);

        setA.retainAll(setB);
        System.out.println("교집합: " + setA);  // [3, 4, 5]
    }
}
```

### 예제 3: 입력에서 중복 판별

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());

        Set<String> seen = new HashSet<>();
        int dupCount = 0;

        for (int i = 0; i < n; i++) {
            String name = br.readLine().trim();
            if (!seen.add(name)) {
                dupCount++;  // add()가 false면 이미 존재
            }
        }
        System.out.println("중복 수: " + dupCount);
    }
}
```

## 4. 이론 퀴즈

**Q1.** HashSet에 같은 값을 두 번 `add()`하면?

<details>
<summary>정답</summary>
두 번째 add()는 false를 반환하고, Set의 내용은 변하지 않는다. 중복을 허용하지 않기 때문.
</details>

**Q2.** HashSet과 TreeSet의 차이는?

<details>
<summary>정답</summary>
HashSet은 해시 기반으로 O(1) 접근이지만 순서 없음. TreeSet은 트리 기반으로 O(log n)이지만 정렬된 순서를 유지한다.
</details>

**Q3.** Set에서 인덱스로 요소를 접근할 수 있는가?

<details>
<summary>정답</summary>
불가. Set은 순서 개념이 없으므로 인덱스 접근(get(i))을 지원하지 않는다. 순회(for-each)로만 접근한다.
</details>

## 5. 함정 노트

### 1. HashSet의 순서를 기대하지 말 것

```java
Set<Integer> set = new HashSet<>();
set.add(3); set.add(1); set.add(2);
System.out.println(set);  // [1, 2, 3]? 아님! 순서 보장 안 됨
```

순서가 필요하면 `LinkedHashSet`(삽입 순서) 또는 `TreeSet`(정렬 순서) 사용.

### 2. equals()와 hashCode() 불일치

HashSet은 `hashCode()`로 버킷을 찾고, `equals()`로 동일 여부를 확인한다.
커스텀 클래스를 Set에 넣으려면 `equals()`와 `hashCode()`를 **둘 다** 재정의해야 한다.

### 3. Set을 정렬된 리스트로 변환

```java
Set<Integer> set = new HashSet<>(Set.of(5, 3, 1, 4, 2));
List<Integer> sorted = new ArrayList<>(set);
Collections.sort(sorted);
System.out.println(sorted);  // [1, 2, 3, 4, 5]
```
