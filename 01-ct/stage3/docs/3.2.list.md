## 1. 설명

### ArrayList

동적 크기 배열. 인덱스로 빠른 접근, 끝에 추가/삭제 효율적.

```java
import java.util.ArrayList;
import java.util.List;

List<String> list = new ArrayList<>();
```

### 주요 메서드

| 메서드 | 동작 | 시간 복잡도 |
|--------|------|:-----------:|
| `add(e)` | 끝에 추가 | O(1) 평균 |
| `add(i, e)` | i 위치에 삽입 | O(n) |
| `get(i)` | i번째 요소 반환 | O(1) |
| `set(i, e)` | i번째 요소 교체 | O(1) |
| `remove(i)` | i번째 요소 삭제 | O(n) |
| `remove(e)` | 첫 번째 일치 요소 삭제 | O(n) |
| `size()` | 요소 수 | O(1) |
| `isEmpty()` | 비었는지 확인 | O(1) |
| `contains(e)` | 포함 여부 | O(n) |
| `indexOf(e)` | 첫 번째 위치 (-1이면 없음) | O(n) |
| `clear()` | 전체 삭제 | O(n) |

### 배열 vs ArrayList

| 구분 | 배열 | ArrayList |
|------|------|-----------|
| 크기 | 고정 | 동적 |
| 타입 | 원시/참조 | 참조만 (Wrapper) |
| 접근 | `arr[i]` | `list.get(i)` |
| 추가 | 불가 (크기 고정) | `list.add(e)` |

### Collections 유틸 클래스

`java.util.Collections`는 컬렉션을 조작하는 static 메서드를 제공한다.

```java
import java.util.Collections;

List<Integer> list = new ArrayList<>(List.of(3, 1, 4, 1, 5));
```

| 메서드 | 동작 |
|--------|------|
| `Collections.sort(list)` | 오름차순 정렬 |
| `Collections.reverse(list)` | 순서 뒤집기 |
| `Collections.max(list)` | 최대값 |
| `Collections.min(list)` | 최소값 |
| `Collections.frequency(list, e)` | 특정 요소의 빈도 |
| `Collections.swap(list, i, j)` | i, j 위치 교환 |
| `Collections.fill(list, e)` | 모든 요소를 e로 채우기 |

### List 순회

```java
// for-each
for (String s : list) {
    System.out.println(s);
}

// 인덱스 기반
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
```

## 2. 예제

### 예제 1: ArrayList 기본 사용

```java
import java.util.ArrayList;
import java.util.List;

public class Main {
    public static void main(String[] args) {
        List<Integer> nums = new ArrayList<>();
        nums.add(10);
        nums.add(20);
        nums.add(30);

        System.out.println(nums.get(1));     // 20
        System.out.println(nums.size());     // 3
        System.out.println(nums.contains(20)); // true

        nums.remove(1);  // 인덱스 1 (값 20) 삭제
        System.out.println(nums);  // [10, 30]
    }
}
```

### 예제 2: Collections 유틸

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        List<Integer> nums = new ArrayList<>(Arrays.asList(5, 2, 8, 1, 9, 3));

        Collections.sort(nums);
        System.out.println("정렬: " + nums);        // [1, 2, 3, 5, 8, 9]

        Collections.reverse(nums);
        System.out.println("역순: " + nums);        // [9, 8, 5, 3, 2, 1]

        System.out.println("최대: " + Collections.max(nums)); // 9
        System.out.println("최소: " + Collections.min(nums)); // 1
    }
}
```

### 예제 3: List로 입력 처리

```java
import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());

        List<Integer> list = new ArrayList<>();
        StringTokenizer st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            list.add(Integer.parseInt(st.nextToken()));
        }

        Collections.sort(list);
        System.out.println(list);
    }
}
```

## 4. 이론 퀴즈

**Q1.** `ArrayList`의 `get(i)` 시간 복잡도는?

<details>
<summary>정답</summary>
O(1). 내부적으로 배열 인덱스 접근이므로 상수 시간.
</details>

**Q2.** `ArrayList`의 중간 삽입/삭제가 느린 이유는?

<details>
<summary>정답</summary>
삽입/삭제 위치 이후의 모든 요소를 한 칸씩 이동해야 하므로 O(n).
</details>

**Q3.** `Collections.sort(list)`의 정렬 순서는?

<details>
<summary>정답</summary>
오름차순 (작은 값 → 큰 값). 역순은 Collections.reverse() 또는 Comparator.reverseOrder() 사용.
</details>

## 5. 함정 노트

### 1. remove(int) vs remove(Object)

```java
List<Integer> list = new ArrayList<>(List.of(1, 2, 3));
list.remove(1);           // 인덱스 1 삭제 → [1, 3]
list.remove(Integer.valueOf(1)); // 값 1 삭제 → [3]
```

`Integer` 리스트에서 `remove(1)`은 **인덱스** 삭제! 값을 삭제하려면 `Integer.valueOf()` 사용.

### 2. 순회 중 삭제

```java
for (String s : list) {
    if (s.equals("remove")) {
        list.remove(s);  // ConcurrentModificationException!
    }
}

// 해결: Iterator 사용
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().equals("remove")) {
        it.remove();  // 안전한 삭제
    }
}
```

### 3. Arrays.asList()는 고정 크기

```java
List<Integer> list = Arrays.asList(1, 2, 3);
list.add(4);  // UnsupportedOperationException!

// 수정 가능하게 하려면
List<Integer> list = new ArrayList<>(Arrays.asList(1, 2, 3));
```

### 4. 빈 리스트 생성 방법

```java
List<String> list = new ArrayList<>();    // 가변
List<String> list = List.of();            // 불변 (add 불가)
List<String> list = Collections.emptyList(); // 불변
```
