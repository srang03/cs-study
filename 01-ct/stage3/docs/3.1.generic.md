## 1. 설명

### 제너릭이란

타입을 매개변수화하여 **하나의 코드로 여러 타입을 안전하게 다루는 문법**이다.

```java
// 제너릭 없이: Object로 받으면 꺼낼 때 형변환 필요
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0);  // 캐스팅 필요, 실수 시 런타임 에러

// 제너릭: 컴파일 시점에 타입 검사
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);  // 캐스팅 불필요, 타입 안전
```

### 제너릭 클래스

```java
public class Box<T> {
    private T content;

    public void put(T item) {
        this.content = item;
    }

    public T get() {
        return content;
    }
}
```

```java
Box<String> strBox = new Box<>();
strBox.put("hello");
String s = strBox.get();  // 캐스팅 불필요

Box<Integer> intBox = new Box<>();
intBox.put(42);
int n = intBox.get();  // 언박싱 자동
```

### 타입 파라미터 관례

| 문자 | 의미 |
|------|------|
| `T` | Type (일반) |
| `E` | Element (컬렉션 원소) |
| `K` | Key |
| `V` | Value |
| `N` | Number |

### 제너릭과 원시 타입

제너릭에는 **원시 타입을 사용할 수 없다**. Wrapper 클래스를 사용해야 한다.

```java
List<int> list;       // 컴파일 에러!
List<Integer> list;   // OK — Integer 사용
```

이것이 stage1.3에서 배운 Wrapper 클래스와 오토박싱이 필요한 이유다.

### 제너릭 메서드

클래스가 아닌 **특정 메서드에만** 타입 파라미터를 적용할 수 있다.

```java
public static <T> T firstElement(T[] arr) {
    return arr[0];
}

// 호출 시 타입 자동 추론
String s = firstElement(new String[]{"a", "b"});  // T = String
Integer n = firstElement(new Integer[]{1, 2});     // T = Integer
```

- 반환 타입 앞의 `<T>`가 타입 파라미터 선언
- static 메서드에서도 자신만의 타입 파라미터 사용 가능

### 타입 상한 (extends)

타입을 특정 클래스의 하위 타입으로 제한.

```java
// T는 Number 또는 그 하위 타입만 가능
public static <T extends Number> double sum(List<T> list) {
    double total = 0;
    for (T item : list) {
        total += item.doubleValue();  // Number의 메서드 사용 가능
    }
    return total;
}
```

### 타입 소거 (Type Erasure)

컴파일 후 제너릭 정보는 사라지고 `Object`(또는 상한 타입)로 대체된다.
JVM 바이트코드에는 제너릭이 없다.

- 컴파일 시점: 타입 안전성 검사
- 런타임: `List<String>`과 `List<Integer>`는 구분 불가 (같은 `List`)

> 참조: [java/part3a/section8generic/doc/8.1.generic.md](../../../java/part3a/section8generic/doc/8.1.generic.md)

## 2. 예제

### 예제 1: 제너릭 클래스

```java
public class Pair<K, V> {
    private K key;
    private V value;

    public Pair(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public K getKey() { return key; }
    public V getValue() { return value; }

    @Override
    public String toString() {
        return key + " = " + value;
    }
}
```

```java
Pair<String, Integer> p = new Pair<>("age", 25);
System.out.println(p);                    // age = 25
System.out.println(p.getKey());           // "age"
System.out.println(p.getValue());         // 25
```

### 예제 2: 제너릭 메서드

```java
public class Main {
    public static <T> void printArray(T[] arr) {
        for (T item : arr) {
            System.out.print(item + " ");
        }
        System.out.println();
    }

    public static void main(String[] args) {
        Integer[] nums = {1, 2, 3};
        String[] words = {"hello", "world"};

        printArray(nums);   // 1 2 3
        printArray(words);  // hello world
    }
}
```

### 예제 3: 타입 상한

```java
public class Main {
    // Number의 하위 타입만 받음
    public static <T extends Comparable<T>> T max(T a, T b) {
        return a.compareTo(b) >= 0 ? a : b;
    }

    public static void main(String[] args) {
        System.out.println(max(3, 7));         // 7
        System.out.println(max("abc", "xyz")); // xyz
    }
}
```

## 4. 이론 퀴즈

**Q1.** `List<int>`가 불가능한 이유는?

<details>
<summary>정답</summary>
제너릭에는 원시 타입을 사용할 수 없다. JVM의 타입 소거 후 Object로 대체되는데, 원시 타입은 Object가 아니기 때문. Integer 등 Wrapper 클래스를 사용해야 한다.
</details>

**Q2.** `<T extends Number>`의 의미는?

<details>
<summary>정답</summary>
T는 Number 또는 Number의 하위 타입(Integer, Double 등)만 가능하다는 타입 상한 제약이다.
</details>

**Q3.** 타입 소거(Type Erasure)란?

<details>
<summary>정답</summary>
컴파일 후 제너릭 타입 정보가 제거되고 Object(또는 상한 타입)로 대체되는 것. 런타임에는 제너릭 타입을 구분할 수 없다.
</details>

## 5. 함정 노트

### 1. 제너릭 배열 생성 불가

```java
T[] arr = new T[10];  // 컴파일 에러! 타입 소거 때문

// 우회 방법
Object[] arr = new Object[10];
T item = (T) arr[0];  // 비검사 경고
```

### 2. instanceof에 제너릭 타입 사용 불가

```java
if (obj instanceof List<String>) { }  // 컴파일 에러!
if (obj instanceof List<?>) { }       // OK (와일드카드)
```

### 3. raw type 사용 경고

```java
List list = new ArrayList();   // raw type — 타입 안전하지 않음
List<String> list = new ArrayList<>();  // 올바름
```

## 7. 보스 문제

### 제너릭 리스트 직접 구현

`SimpleList<T>` 클래스를 구현하라:
- 내부 `Object[]` 배열 사용
- `add(T item)`: 끝에 추가 (배열이 꽉 차면 2배로 확장)
- `get(int index)`: 해당 인덱스 반환
- `size()`: 현재 요소 수
- `remove(int index)`: 해당 인덱스 제거 후 뒤쪽 요소 이동
