# 도구 — AI가 덮지 않는 영역과 스택 매핑

## AI가 해주는 것과 해주지 않는 것

AI는 **코드·설정 초안 작성**까지 빠르게 돕는다. 다만 **빌드·실행·디버깅**, 로컬·클라우드에서의 **웹/API 서버·DB 기동**, **배포**, **버전 관리(Git)** 는 개발자가 도구로 직접 다루는 영역이다. IDE, 런타임, Git, 원격 접속 도구를 갖추는 것이 전제다.

## 스택별 권장 도구

### 프론트엔드 (Next.js)

- **에디터:** Visual Studio Code
- **런타임:** Next.js는 **Node.js**와 패키지 매니저(npm, pnpm, yarn 등)가 필요하다. 프로젝트 생성·의존성 설치는 터미널에서 수행한다.

### 백엔드 (Spring Boot)

- **IDE:** IntelliJ Community(또는 이에 준하는 Java IDE) — 빌드·실행·디버깅
- **JDK:** 사용하는 Spring Boot 버전에 맞는 **JDK** 설치

### 데이터베이스 (PostgreSQL)

- 서버로 **PostgreSQL**을 둔다. 스키마·데이터를 GUI로 볼 때는 **DBeaver** 등 클라이언트를 쓸 수 있다(선택).

### AWS 원격 작업

- **PuTTY:** SSH 접속
- **WinSCP:** SFTP 등 파일 전송

### AI 보조

- **Claude Desktop:** 대화·문서 작업 등
- **Claude Code:** 에디터·터미널과 연계한 코딩 보조

### Docker(선택)

이 트랙에서 컨테이너를 쓰지 않으면 당장 필수는 아니다. 인프라를 Docker 기준으로 맞출 때는 Docker Desktop 등을 별도로 준비한다.

## Git과 원격 저장소

**Git**은 소프트웨어 **버전 관리** 도구다. 별도 Git 서버를 직접 구축하지 않아도 **GitHub** 같은 호스팅으로 원격 저장소를 쓸 수 있다. **1인 개발**이라도 버전 관리는 필수다.

바이브 코딩의 산출물인 **소스 코드와 문서**는 Git으로 백업·이력을 남긴다. AI가 잘 짜인 코드를 **망가뜨릴 수도** 있으므로, 커밋 단위로 되돌릴 수 있는 상태를 유지하는 것이 안전하다. 커밋 메시지는 **나중에 읽어도 이해되는** 짧은 설명을 남긴다. 지금의 맥락은 빠르게 잊히므로, **6개월 뒤의 자신**을 돕는 기록이라고 보면 된다.

**Git Bash**는 Windows에서 Git과 함께 제공되는 **터미널(셸) 환경**일 뿐이며, Git 도구 자체와는 구분하는 것이 좋다. 명령줄이 익숙하지 않으면 **SourceTree** 같은 GUI 클라이언트를 쓸 수 있다([[development/vibe-coding/01.prologue|01. 프롤로그]]와 동일한 예시).

## Git 용어(최소)

- **Repository(저장소):** 프로젝트 소스와 변경 이력이 담기는 단위.
- **Branch:** 독립된 작업 줄기. `main` 또는 `master`가 기본 브랜치로 쓰이는 경우가 많다.
- **Commit:** 변경을 저장소에 기록하는 단위.
- **Clone:** 원격 저장소를 로컬에 복제한다.
- **Push:** 로컬 커밋을 원격 저장소에 올린다.
- **Pull:** 원격의 변경을 로컬로 가져온다.
- **병합 요청:** 브랜치를 합치기 위한 검토·요청 절차. **GitLab 등에서는 Merge Request(MR)**, **GitHub에서는 Pull Request(PR)** 로 부르는 것이 일반적이다.
