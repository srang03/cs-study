  - 미니 프로젝트 + AWS 환경에 적용

- 개발 환경
  - Claude for desktop + Claude code
  - IntelliJ Community

  - SourceTree

2. 학습 목표
- 바이브 코딩만으로 풀스택 개발자 되기 (나만의 서비스를 만들기)
  - 오직 AI 도구만을 사용하여 완성
- 서비스 기획을 근거로 신속하게 프로토타입으 개발할 수 있는 방법 학습
  - 기획은 협의이다.
  - 현실의 자신의 상태를 인지한 상태에서 나와 AI와 협의해서 최초의 기획을 도출하고 정확한 지시물을 생성해서 AI가 결과물을 생성하는 것이 목표이다.
- AWS 기반 배포 포함

과정에서 의사결정, 대부분의 오류나 설정에 대한 문제가 발생한다. 이를 해결하는 과정의 시간이 많이 소요된다.

3. 경력자의 현재
나의 상태를 파악하고 대책을 세우는 것이 중요하다.
과정상에서 겪에될 문제로 인하여 진행이 안되고 원인을 파악하는 것이 이슈이다.
해결하지 못할 때 나오는 증상이 있다.

대부분은 플랫폼을 활용해서 수익을 내는 형태였는데, 남들 다 하니까 나도 한 번 해보자는 가벼운 생각, 유행이며 나도 관심이 생겼음

약간의 비용과 시간만 들 뿐이지 큰손해가 나지 않는다.
이제는 내 서비스르 가지고 싶다는 마음
저와 비슷한 생각을 가진 분들에게 도움이 되기를 희망

비용이 발생하는것 AWS, Claude code

5. 목표 시스템 구성과 필수 배경지식
서비스를 만들기 위한 서비스 구성을 도출

미리 알아야할 기초 용어
- CPU, RAM, SSD 등 컴퓨터 스팩
- Process와 Thread 및 동기화
  - JVM은 WAS가 작동하기 위한 기반

- 관계형 DB (트랜잭션, 롤백과 커밋, 백업)
  - 대부분의 경우 MySQL, PostgreSQL 같은 관계형 DB
  
- File 입/출력
- Framework (React.js, Next.js, Spring Boot)
- Spring Boot 기반의 API 서버

REST API
API는 함수로 이해할 수 있음
- 함수가 반환하는 정보는 보통 JSON 형식
- AI가 생성한 백앤드 서버의 REST API에 대해서는 반드시 별도 문서로 관리하는 것이 바람직함
  - 최초 설계에서 벗어나는 일은 불가피 (기존 기능의 구조적 오류, 새기능 추가 등)
  - 일련의 과정에서 원인, 대응에 관한 내용이 포함된 문서를 프론트/백앤드 각각 생성
  - SQL 스크립트 관리 필수
- IP 주소 (Public, Private), 포트 번호
  - 웹 서버, WAS가 작동 중인 컴퓨터의 IP 주소를 알아야 한다.
- URL, 도메인 이름, DNS, HTTP, SSL
  - Localhost(127.0.0.1)
  - SSL 인증서
- HTML(tag, id, class), CSS, Javascript
- 웹 서버, WAS
  
AI에 의존하고 다 해주면, 기술 부채가 생긴다.
프로그래밍을 하게 되면 최초 설계에서 벗어나는 일은 불가피하다. 최소 설계 단계에서 생각해보지 못하거나 몰랐던 부분에 대해 맞닥뜨리게 되면 설계 변경을 하게 된다.
그 이유는 구조적으로 내가 생각하지 못해서 생긴 오류와 새로운 기능을 추가해야하는데 이 기능을 추가하려고 하니까 그점이 고려되지 않은 채로 이미 만들어진 것이다. 그러면 이거를 바꾸는 일과 뜯어 고치는 일은 늘상 있는 일이다.
