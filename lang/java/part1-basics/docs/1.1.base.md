# 기본 개념

## 1. 인코딩

문자·문자열을 비트로 표현하는 방식을 인코딩이라 한다. 컴퓨터는 문자를 다룰 때 이 인코딩을 사용한다.

- ASCII 인코딩: 영문 한 글자를 1바이트로 표현하는 기본 인코딩이다.
- 한계: 동아시아 등 많은 문자를 ASCII만으로는 표현할 수 없어, 최소 2바이트 이상의 유니코드 체계를 사용한다.
- UTF-8: 유니코드 기반 인코딩으로, 웹에서 기본적으로 널리 사용된다.
- 자바: 문자열 표현에 UTF-8 계열을 사용한다. 표준 UTF-8이 아니라 modified UTF-8을 사용한다는 점이 특징이다.

## 2. Windows OS 관련 용어

### 2.1 디렉토리와 파일 식별

- 디렉토리(폴더): 파일을 묶어서 관리하는 단위이다.
- 파일 식별: 경로, 이름, 확장자로 파일을 구분한다.

### 2.2 경로

- 절대 경로: 루트부터 이어지는 전체 경로이다.
- 상대 경로: 현재 위치를 기준으로 한 경로이다.
- 상대 경로 기호: `..`는 상위 디렉토리, `.`(또는 `./`)는 현재 디렉토리를 가리킨다.

## 3. 파일

파일은 바이너리와 텍스트로 구분해서 이해하는 것이 좋다.

- 바이너리 파일: 0과 1의 비트 스트림으로 이루어진 데이터이다. 중간에 정해진 길이 단위로 잘라 해석하는 방식이 기본이다.
- 텍스트 파일: 특정 문자 인코딩(예: UTF-8)으로 해석되는 바이트 스트림이다.

### 3.1 실행 파일

- 실행 파일: 바이너리 스트림을 해석하면 CPU가 실행할 수 있는 기계어로 구성된 파일이다.
- 확장자 예: `.exe`, `.dll`, `.sys`
- PE(Portable Executable): Windows에서 사용하는 실행 파일 형식이다.
