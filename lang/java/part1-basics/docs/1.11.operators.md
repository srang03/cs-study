# 연산자

## 1. 형승격(Type promotion)

연산 시 임시 결과의 타입은 피연산자보다 넓은 표현 범위를 가질 수 있어야 한다.
그래야 오버플로우 없이 연산 결과를 담을 수 있다.

- `char + int`의 결과 타입은 `int`이다. `char`는 연산 전에 `int`로 승격된다.
- `double * int`의 결과 타입은 `double`이다. `int`는 연산 전에 `double`로 승격된다.
- `4 + 3`은 정수 연산이고, `4.0 + 3`은 실수 연산이다. 리터럴 타입이 다르므로 수행되는 연산이 다르다.
- `String`과 숫자를 `+`로 연산하면 문자열 연결이 이루어지며, 결과 타입은 항상 `String`이다.

## 2. 형변환 연산자

`(타입)`은 피연산자의 형식을 강제로 바꾸는 단항 연산자이다.

```java
double a = 3.3;
int b = (int) a;
```

부적절한 변환(예: `double`을 `int`로 캐스트)을 하면 소수 이하가 버려지는 등 정보가 유실될 수 있다.

## 3. 비트 연산자

`&`, `|`, `^`, `~`, `<<`, `>>`, `>>>`는 자료를 비트 단위로 연산하는 연산자이다.
동작을 판단할 때는 값을 2진수로 생각하면 이해하기 쉽다.

- `&`: AND. 두 비트가 모두 1일 때 1.
- `|`: OR. 한 비트라도 1이면 1.
- `^`: XOR. 두 비트가 다를 때 1.
- `~`: NOT. 비트를 반전한다. 단항 연산자이다.
- `<<`: 왼쪽 시프트.
- `>>`: 부호 있는 오른쪽 시프트.
- `>>>`: 부호 없는 오른쪽 시프트.

`~`는 단항 연산자이고, 나머지는 모두 이항 연산자이다.

## 4. 비트 마스크

비트 마스크는 데이터에서 특정 비트 영역을 0으로 만드는 연산이다.
AND의 성질(0과 AND하면 결과는 0)을 이용한다.

```java
int data = 0x11223344;
int lower = data & 0x0000FFFF;  // 하위 16비트만 남기고 상위 16비트는 0
```

JVM은 빅엔디안을 사용한다. 비트·바이트 단위 해석이 필요할 때는 이 점을 고려한다.

## 5. 단락 평가(Short circuit)

논리식은 왼쪽부터 평가한다. 피연산자가 식이면 그 식을 먼저 평가한 뒤 논리 연산을 적용한다.

- 논리합(`||`): 왼쪽이 `true`이면 결과가 이미 `true`이므로 오른쪽은 평가하지 않는다.
- 논리곱(`&&`): 왼쪽이 `false`이면 결과가 이미 `false`이므로 오른쪽은 평가하지 않는다. 왼쪽이 `true`일 때만 오른쪽을 평가해 모든 항이 참인지 확인한다.

이처럼 결과가 확정되면 나머지 항을 평가하지 않는 동작을 단락 평가(short circuit evaluation)라 한다.
