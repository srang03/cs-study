# 문자열과 인코딩 (Java 중심)

## 1. 문자, 문자열, 문자 집합

- 문자(character): 사람이 의미를 부여한 기호(예: `A`, `가`)
- 문자열(string): 문자들의 순서 있는 나열
- 문자 집합(character set): 문자를 숫자(코드 포인트) 로 매핑한 규칙의 집합

ASCII는 문자 집합(및 그에 대한 코드 규칙)의 한 예이며, 주로 영문·기호 영역을 다룬다. 더 넓은 문자 지원이 필요해지면서 Unicode(유니코드)라는 문자 집합이 등장했다.

## 2. 유니코드와 인코딩(encoding)

- 유니코드(Unicode): 문자를 코드 포인트로 정의하는 문자 집합
- 인코딩(encoding): 코드 포인트를 바이트열로 표현하는 규칙(저장/전송을 위해 필요)

대표적인 유니코드 인코딩은 UTF-8, UTF-16 등이 있다.

## 3. Java에서의 `char`와 `String`

### 3.1 `char`

Java의 `char`는 16비트(2바이트) 정수 타입이며, UTF-16의 코드 유닛(code unit) 1개를 담는다.

- `'A' + 1` 같은 연산이 가능한 이유는 `char`가 정수 타입이기 때문이다.
  - 예: `'A'`는 코드값 65, `65 + 1 = 66`이며 이를 문자로 해석하면 `B`가 된다.
- 모든 문자가 `char` 1개로 표현되는 것은 아니다.
  - UTF-16에서 일부 문자는 2개의 코드 유닛(서로게이트 페어)로 표현될 수 있다.

### 3.2 `String`

`String`은 불변(immutable)이다. 한 번 생성된 문자열의 내용은 변경되지 않는다.

## 4. 문자열 표현 방식: C의 NUL 종료 vs Java의 길이 기반

- C/C++: 문자열이 바이트 배열로 표현되며, 끝을 `\\0`(NUL)로 표시하는 방식이 흔하다.
- Java: 문자열은 객체로 표현되며, `\\0`로 끝을 표시하는 방식이 아니다.

## 5. 줄바꿈(CR/LF)과 텍스트 데이터

텍스트에서 줄바꿈은 제어 문자로 표현된다.

- CR(carriage return): `0x0D` 또는 `\\r`
- LF(line feed): `0x0A` 또는 `\\n`

파일/네트워크 I/O에서는 결국 바이트를 읽고 쓰므로, 문자열을 저장·전송할 때는 어떤 인코딩(예: UTF-8)을 사용할지 명확히 해야 한다.
