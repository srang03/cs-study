# 변수와 GC

## 1. 변수의 종류 및 사용

### 지역 변수

스코프 내에서만 접근 가능한 변수이다.
`static`이 없으면 자동 변수이며 스택 메모리를 사용한다. 스택 영역의 할당·해제는 메서드 호출·반환에 따라 자동으로 이루어진다.
스택 자료구조 LIFO에 대한 이해가 필요하다.

### 매개 변수

메서드의 매개변수이다. 접근성은 지역 변수와 같고, 메서드 내부에서만 사용된다.

### 인스턴스 변수

클래스의 non-static 필드이다. 각 객체(인스턴스)마다 하나씩 가지며, `new`로 객체를 생성하면 그 객체는 heap에 할당되고 인스턴스 변수는 그 객체의 일부로 heap에 둔다.

### 클래스 변수

클래스의 static 필드이다. 클래스당 하나이며, 인스턴스화 없이 클래스 이름으로 접근한다. 클래스 변수는 Method area에 로드된다.

## 2. 자동 변수와 스택

지역 변수는 `static`이 없을 때 자동 변수로, 스택에 할당된다.
`push`하면 스택 공간이 확보되고 `pop`하면 스택 공간에서 제거된다.
메서드 호출 시 콜 스택이 쌓이고, 순서에 따라 메서드 내부의 지역 변수·매개 변수가 아래부터 쌓인다. 메서드가 반환하면 스택에서 위에서 아래로 순차적으로 제거된다.

참조 변수인 경우, 실제 객체(인스턴스)는 heap에 할당되고 스택에는 그 참조(주소)만 쌓인다. 스택의 참조 변수가 참조하는 객체와 연결되어 있다가, 메서드 반환으로 스택에서 참조 변수가 모두 `pop`되면 해당 객체를 가리키는 참조가 없어져 GC 대상이 되고, GC가 회수한다.
