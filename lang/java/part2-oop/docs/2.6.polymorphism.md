# 다형성

## 1. 다형성 개념

다형성(Polymorphism)은 "같은 타입으로 보이지만 실제로는 다양한 형태로 동작하는" 성질이다. 예를 들어 포유류라는 타입 아래 인간, 원숭이 등 다양한 구체 타입이 있듯이, 프로그래밍에서는 부모 타입 변수가 자식 타입 인스턴스를 참조할 수 있고, 호출 시 실제 인스턴스 타입에 맞는 메서드가 실행된다.

다형성을 쓰면 추상적인 타입(부모, 인터페이스)만으로 코드를 작성할 수 있어, 구체 타입에 덜 의존하게 된다. 프레임워크가 "추상 타입"을 정의하고, 사용자가 "구체 구현"을 넣는 방식이 가능한 이유이다.

## 2. 업 캐스팅

부모 타입 변수에 자식 인스턴스를 대입하는 것을 업 캐스팅(upcasting)이라 한다. 자식은 부모의 한 종류이므로, 자동으로 변환되며 별도 문법이 필요 없다.

```java
class Parent { }
class Child extends Parent { }

Parent p = new Child();  // 업 캐스팅: Child를 Parent 타입으로 참조
```

`p`는 `Parent` 타입이므로 `Parent`에 정의된 멤버만 접근할 수 있다. 하지만 실제 인스턴스는 `Child`이므로, 재정의된 메서드를 호출하면 `Child`의 구현이 실행된다(동적 바인딩).

## 3. 다운 캐스팅

부모 타입 변수가 가리키는 인스턴스를 자식 타입으로 다시 참조하려면 다운 캐스팅(downcasting)을 한다. `(자식타입)`을 붙여 명시적으로 변환한다.

```java
Parent p = new Child();
Child c = (Child) p;  // 다운 캐스팅: p가 실제로 Child를 가리킬 때만 안전
```

실제 인스턴스가 해당 자식 타입이 아닌데 다운 캐스팅하면 `ClassCastException`이 발생한다. 부모가 여러 자식 타입을 가질 수 있으므로, 다운 캐스팅 전에 타입을 확인해야 한다.

부모 클래스 코드에서 자식 타입으로 다운 캐스팅하는 것은 의존성 역전을 일으킬 수 있다. 부모는 자식의 존재를 몰라야 하고, 다운 캐스팅이 필요하다면 설계를 다시 검토하는 것이 좋다.

## 4. instanceof 연산자

다운 캐스팅이 안전한지 확인하려면 `instanceof` 연산자를 사용한다. 실행 시점에 실제 인스턴스 타입을 조사하는 RTTI(Run-Time Type Information)에 해당한다.

```java
if (obj instanceof Child) {
    Child c = (Child) obj;  // 안전한 다운 캐스팅
}
```

`instanceof`는 `obj`가 `null`이 아니고, `Child` 타입이거나 `Child`의 하위 타입일 때 `true`를 반환한다.

## 5. 추상 클래스

`abstract` 클래스는 인스턴스를 만들 수 없다. `new`로 생성할 수 없고, 오직 파생 클래스를 정의하기 위한 베이스로만 사용한다. 공통 구조는 두고, 일부만 파생 클래스에서 채우게 할 때 쓴다.

```java
public abstract class Animal {
    public abstract void speak();  // 추상 메서드: 본문 없음, 파생에서 반드시 재정의
    public void sleep() {
        System.out.println("zzz");
    }
}
class Dog extends Animal {
    @Override
    public void speak() {
        System.out.println("멍멍");
    }
}
```

추상 메서드는 본문이 없고, 파생 클래스에서 반드시 재정의해야 한다. 추상 메서드가 하나라도 있으면 그 클래스는 `abstract`로 선언해야 한다. C++의 순수 가상 함수와 비슷한 개념이다.

## 6. 인터페이스

인터페이스는 메서드 시그니처(원형)만 선언하고, 구현은 구현 클래스에 맡긴다. `implements`로 구현한다.

```java
public interface Drawable {
    void draw();  // 본문 없음, public abstract가 기본
}
class Circle implements Drawable {
    @Override
    public void draw() {
        System.out.println("원 그리기");
    }
}
```

- 인터페이스는 인스턴스 필드를 가질 수 없고, `public static final` 상수만 가질 수 있다.
- 메서드는 기본적으로 `public abstract`이다.
- 클래스는 하나만 상속할 수 있지만, 인터페이스는 여러 개 구현할 수 있다.

인터페이스는 "무엇을 할 수 있는가"만 정의하고, "어떻게 할 것인가"는 구현 클래스에 맡긴다. 이 때문에 다형성과 느슨한 결합을 위한 수단으로 자주 사용된다.

## 7. 열거형(enum)

`enum`은 고정된 상수 집합을 타입으로 정의한다. `int` 상수 대신 `enum`을 쓰면 타입 안전성이 보장되고, 허용된 값만 사용할 수 있다.

```java
public enum Color {
    RED, GREEN, BLUE
}
Color c = Color.RED;
```

`enum`은 내부적으로 `java.lang.Enum`을 상속한 클래스이다. `values()`, `valueOf(String)`, `name()`, `ordinal()` 등의 메서드를 가진다. `ordinal()`은 선언 순서(0부터)를 반환하므로, 로직에 의존하지 말고 `name()`이나 별도 필드를 사용하는 것이 좋다.
