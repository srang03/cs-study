# 자바 언어 기본

## 1. 메모리 관리

Java에는 C/C++의 소멸자(destructor)가 없다. Heap 메모리는 GC(Garbage Collector)가 자동으로 회수한다.
Java는 managed 언어이므로 개발자가 메모리를 직접 반환할 수 없고, 사용이 끝난 객체는 GC가 참조 여부를 추적하여 회수한다. 이 때문에 메모리 누수·이중 해제 같은 C/C++에서 흔한 오류를 언어 수준에서 피할 수 있다.

## 2. C/C++와의 차이

### 2.1 상수형(const) 없음

Java에는 C/C++의 `const`처럼 "상수형 메서드" 또는 "상수형 참조"가 없다.
대신 `final` 키워드로 "한 번만 할당 가능하고 이후 재할당 불가"를 표현한다. Java는 C++의 `const`가 가진 여러 의미(상수 메서드, 상수 참조, 상수 포인터 등)를 하나의 `final`로 단순화했고, 재할당 방지와 의도 표현에 집중한다.

### 2.2 연산자 오버로딩 없음

Java에는 C++의 연산자 함수(operator overloading)가 없다.
연산자 동작은 언어에 정의된 대로만 사용한다. Java 설계 당시 C++에서 연산자 오버로딩이 가독성을 해친 사례를 참고하여, 의도를 메서드 이름으로 드러내는 방식을 선택했다.

## 3. 불변 객체 활용

Java에는 상수형 메서드·상수형 참조가 없으므로, 읽기 전용을 보완하기 위해 불변(immutable) 객체를 설계한다.

- 수정이 필요하면 기존 객체를 버리고 수정된 새 객체를 사용한다.
- 불변 객체는 생성 후 내부 상태가 바뀌지 않으므로, GC가 회수할 때까지 여러 곳에서 안전하게 공유할 수 있다. 또한 풀링·캐싱과 함께 사용하면 메모리 효율이 좋다.
- 멀티스레드 환경에서는 상태 변경이 없어 동기화 없이 공유할 수 있어, 동시성 오류를 줄이는 데 유리하다.

## 4. 문자열 상수와 String constant pool

문자열 리터럴은 `String` 클래스 객체이다. `String`은 불변(immutable)이므로 한 번 생성된 내용은 바뀌지 않는다.
JVM은 성능과 메모리 효율을 위해 Heap의 String constant pool에 리터럴을 저장하고, 동일한 리터럴은 pool에서 재사용한다. 불변이기 때문에 여러 참조가 같은 객체를 가리켜도 내용이 바뀌지 않아 안전하다.

## 5. void와 null

### 5.1 void

`void`는 변수 타입으로 선언할 수 없고, 메서드 반환 타입으로만 사용한다.
`void`는 "반환값이 없다"는 의미이므로, 값을 담는 변수에는 쓸 수 없다. 반환할 값이 없는 메서드의 반환 타입으로만 의미가 있다.

### 5.2 null

`null`은 참조형에만 적용 가능하다. 원시 타입에는 대입할 수 없다.

```java
int a = null;  // 컴파일 오류
String s = null;  // 가능
```

원시 타입은 값을 직접 담고, 참조형은 객체를 가리키는 참조를 담는다. `null`은 "참조 대상(인스턴스)이 없다"는 의미이므로, 참조를 담는 참조형에만 적용된다. 반환 타입이 참조형인 메서드는 `null`을 반환할 수 있다.

## 6. 참조형과 참조 비교

### 6.1 참조형의 성격

Java에서 클래스 타입은 모두 참조(reference)이다. C/C++의 포인터처럼 "객체를 가리키는 값"으로 이해할 수 있다.
다만 주소 연산(포인터 산술)은 불가능하다. Java는 메모리 안전을 위해 임의 주소 접근을 막았고, 참조는 객체를 가리키거나 `null`만 가질 수 있다.

### 6.2 참조 비교

참조형에 대한 `==`, `!=`는 객체의 필드 값이 아니라 참조(메모리 주소) 비교이다.

```java
String a = "hello";
String b = new String("hello");
// a == b는 false (서로 다른 객체)
// 내용 비교는 equals()를 사용한다
```

참조는 다른 객체를 가리키도록 재할당될 수 있다.

### 6.3 NullPointerException

`null`을 가리키는 참조로 멤버 접근 연산(`.`)을 수행하면 `NullPointerException`이 발생한다.
참조가 `null`이면 대상 객체가 없으므로, 필드나 메서드에 접근할 수 없기 때문이다.
