# 클래스 고급

## 1. 생성자 다중 정의(오버로딩)

일반 메서드처럼 생성자도 매개변수 구성(개수·타입)이 다르면 여러 개를 정의할 수 있다. `new` 연산 시에는 매개변수에 맞는 생성자 하나만 호출된다.

한 생성자에서 다른 생성자를 `this(인자)`로 호출할 수 있다. 공통 초기화 로직이 있을 때 중복을 줄이기 위함이다. `this()` 호출은 생성자 본문의 첫 문장이어야 한다.

## 2. 얕은 복사와 깊은 복사

### 2.1 얕은 복사(Shallow copy)

참조만 복사하고, 참조가 가리키는 객체는 새로 만들지 않는다. 복사된 참조와 원본 참조가 같은 인스턴스를 가리키게 된다.

한 참조를 통해 객체를 수정하면 다른 참조를 통해서도 같은 변경이 보인다. 이 때문에 의도치 않은 부수 효과(side effect)가 발생할 수 있어, 공유가 의도된 경우가 아니면 주의가 필요하다.

### 2.2 깊은 복사(Deep copy)

원본 객체와 그 안의 참조가 가리키는 객체까지 새로 할당하여 복사한다. 복사본과 원본이 서로 독립된 인스턴스를 가지므로, 한쪽을 수정해도 다른 쪽에 영향을 주지 않는다.

필드가 참조형을 포함할 때는, 참조형 필드에 대해서도 새 인스턴스를 만들어 복사해야 진정한 깊은 복사가 된다. 그렇지 않으면 참조형 필드만 복사되어 얕은 복사와 같은 문제가 생긴다.

## 3. 복사 생성자와 clone()

Java에는 C++의 복사 생성자처럼 언어가 자동으로 제공하는 복사 문법이 없다. 대신 같은 클래스 타입을 매개변수로 받는 생성자를 두어 사본을 만드는 방식을 사용한다. `rhs`(Right Hand Side)는 복사의 원본 인스턴스를 가리키는 참조를 의미하는 관례적인 이름이다.

`clone()` 메서드(`Cloneable` 인터페이스)를 사용할 수도 있으나, 규약이 모호하고 예외 처리·깊은 복사 구현이 복잡해 권장되지 않는다. 복사가 필요하면 복사 생성자나 정적 팩토리 메서드를 사용하는 편이 낫다.

```java
public class MyTest {
    private int[] array;

    public MyTest(int size) {
        array = new int[size];
    }

    public MyTest(MyTest rhs) {
        this(rhs.array.length);
        for (int i = 0; i < array.length; i++) {
            array[i] = rhs.array[i];
        }
    }
}
```

## 4. 임시 객체

메서드의 반환 타입이 클래스일 때, `return` 문에서 `new`로 생성한 객체는 이름 없는 임시 객체로 반환된다. 호출자가 그 참조를 받아 사용한 뒤, 더 이상 참조가 없으면 GC가 회수한다.

```java
public Point createPoint(int x, int y) {
    return new Point(x, y);  // 임시 객체 생성 후 참조 반환
}
// 호출: Point p = createPoint(1, 2);  → p가 참조를 잃으면 GC 대상
```

`String`의 `+` 연산은 매 연산마다 새 `String` 객체를 만든다. `"a" + "b"`는 `"a"`와 `"b"`를 연결한 새 `String`을 만들고, 반복문 안에서는 그 수만큼 임시 객체가 쌓인다.

```java
// 비효율적: 반복마다 새 String 객체 생성
String result = "";
for (int i = 0; i < 100; i++) {
    result = result + i;  // 매번 새 String 생성, 이전 String은 GC 대상
}

// 효율적: StringBuilder 하나로 연결
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100; i++) {
    sb.append(i);
}
String result = sb.toString();
```

`StringBuilder`는 내부 버퍼에 문자를 이어 붙이고, `toString()` 호출 시 한 번만 `String`을 만든다. 이 때문에 반복 연결 시 임시 객체가 훨씬 적게 생성된다.

## 5. 정적 멤버

`static`이 붙은 필드와 메서드는 클래스에 속하며, 인스턴스를 만들지 않아도 존재한다.

- 정적 필드: 클래스당 하나만 존재하며, 모든 인스턴스가 공유한다.
- 정적 메서드: 인스턴스 없이 클래스 이름으로 호출할 수 있다. 정적 메서드 안에서는 `this`를 쓸 수 없다. `this`는 인스턴스를 가리키는데, 정적 메서드는 인스턴스에 속하지 않기 때문이다.
- `static final` 필드는 한 번만 할당되고 변경되지 않으므로, 상수로 자주 사용한다(예: `Math.PI`, 색상 코드).

```java
public class Counter {
    private static int count = 0;           // 정적 필드: 모든 인스턴스가 공유
    private static final int MAX = 100;    // 정적 상수

    public Counter() {
        count++;  // 생성될 때마다 같은 count 증가
    }

    public static int getCount() {         // 정적 메서드: 인스턴스 없이 호출
        return count;
        // return this.count;  // 컴파일 오류: this 사용 불가
    }

    public static boolean isValid(int value) {
        return value >= 0 && value <= MAX;  // 유틸리티 성격의 정적 메서드
    }
}

// 사용
Counter c1 = new Counter();
Counter c2 = new Counter();
System.out.println(Counter.getCount());    // 2 (인스턴스 없이 호출)
System.out.println(Counter.isValid(50));   // true
```

정적 필드 `count`는 인스턴스마다 따로 두지 않고 클래스당 하나만 있어, `c1`과 `c2`가 같은 값을 본다. 정적 메서드는 `Counter.getCount()`처럼 클래스 이름으로 호출하며, 인스턴스 변수에 접근할 수 없어 `this`를 쓸 수 없다.

## 6. 정적 필드와 인스턴스의 메모리 배치

정적 필드는 인스턴스와 별도로 Method area(또는 JDK 8 이후 Metaspace의 클래스 메타데이터 영역)에 로드된다. 클래스가 로딩될 때 한 번 할당되고, 인스턴스 생성·소멸과 무관하게 유지된다.

인스턴스 필드는 `new`로 객체를 생성할 때마다 Heap에 할당된다. 메서드 코드는 인스턴스마다 따로 존재하지 않고, 클래스당 하나만 Method area에 있다. 인스턴스 메서드를 호출할 때 `this` 참조로 어떤 인스턴스의 필드를 사용할지 구분한다.
